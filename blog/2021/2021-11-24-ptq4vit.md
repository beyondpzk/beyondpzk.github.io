# PTQ4ViT 论文详解：让 ViT 量化不掉点的 Twin Uniform 与 Hessian 度量

> **论文**: PTQ4ViT: Post-Training Quantization for Vision Transformers with Twin Uniform Quantization
> **作者**: Zhihang Yuan, Chenhao Xue, Yiqi Chen, Qiang Wu, Guangyu Sun (Peking University)
> **发表**: ECCV 2022
> **arXiv**: https://arxiv.org/abs/2111.12293

## 一、动机：CNN 量化那一套在 ViT 上为什么不好使

在 PTQ4ViT 之前，CNN 的 PTQ 方法（MSE / cosine distance 做 scale 搜索）已经相对成熟。但是当研究人员把这套方法直接搬到 ViT 上时，**8-bit 量化掉点普遍超过 1%，某些配置甚至崩溃**。

原因在于 ViT 和 CNN 的激活分布存在本质差异。CNN 依赖 BatchNorm + ReLU 这对组合：BN 把激活分布规整到近似高斯（推理时折叠进卷积），ReLU 把激活截断到 `[0, +∞)`。而 ViT 用 LayerNorm、GELU、Softmax，三者各制造一种极端分布。PTQ4ViT 是第一个系统性地诊断并解决这三个问题的论文。

## 二、三大结构性问题

### 2.1 post-Softmax：幂律长尾

Softmax 输出被压在 `[0,1]`，但分布极度不均——绝大多数值接近 0，极少数关键 attention 值接近 1。论文的观察（Fig. 2）：

- 用大 scale → 小值全部量化为 0，丢失 token 间弱关联
- 用小 scale → 大值被压小，破坏注意力机制的核心功能

这个问题在 6-bit 尤其致命：Base PTQ 下 ViT-S 从 81.39% 掉到 **70.24%（-11.15 pt）**，Swin-T 相对好一些，只掉 2.92 pt——因为 Swin 用窗口内 attention，参与 Softmax 的 token 少得多，长尾效应自然弱。

### 2.2 post-GELU：正负极端不对称

GELU 输出的正值无界且范围大，负值被压在约 `[-0.17, 0]` 的极窄区间。对称 uniform 量化要么浪费一半码位在几乎无信息的负区间，要么 clip 正值破坏 MLP 表达。

### 2.3 常见 scale 搜索指标不可靠

论文将 MSE、cosine distance、Pearson 相关系数等常用指标与任务 loss（cross-entropy）做对比（Fig. 5）。结论非常直白：**这些局部指标在 ViT 上选不出最优 scale。** 例如在 ViT-S 的第 6 个 block 的 `mlp.fc1` 激活上，MSE/cosine/Pearson 都指向 ~0.4× 是"最优" scale，但 cross-entropy 显示 ~0.75× 才是正确的——按前者的 scale，任务精度多掉了近 1 个点。

## 三、Twin Uniform Quantization：一个数据格式解决两类分布

PTQ4ViT 的核心创新之一是 **twin uniform quantization**（双范围均匀量化）。思路非常直觉化：既然 post-Softmax 和 post-GELU 各有一对被严重分割的值域，那就给每类值各自一个 scale。

**数据格式设计：**

- k-bit 量化：最高位是 range flag（0=R1, 1=R2），剩余 k-1 位是 unsigned 数值
- 不需要额外的符号位——因为在同一个 range 内数值符号一致（post-Softmax 全正，post-GELU 的 R1 全负、R2 全正）
- 相比之前的多 range 量化方案（需要额外 1-bit 表示 range + FP32 对齐两个 range），这个方案通过约束 ∆_R2 = 2^m · ∆_R1，用 **bit-shift 完成 range 对齐**，避免了 FP32 乘法和加法

**两种场景的具体应用：**

| 场景 | R1 | R2 | 固定参数 | 搜索参数 |
|------|-----|-----|---------|---------|
| post-Softmax | 小值区间 `[0, 2^(k-1)·∆s_R1)` | 全域 `[0,1]` | ∆s_R2 = 1/2^(k-1)（覆盖最大值） | ∆s_R1（精细量化小值） |
| post-GELU | 负值区间 `[-2^(k-1)·∆g_R1, 0]` | 正值区间 `[0, 2^(k-1)·∆g_R2]` | ∆g_R1（刚好覆盖全部负值） | ∆g_R2（适应各层正值范围） |

**硬件加速如何实现：** 假设 a_q 在 R1、b_q 在 R2，且 ∆_R2 = 2^m · ∆_R1：

```
a_q · ∆_R1 + b_q · ∆_R2 = (a_q + b_q · 2^m) · ∆_R1
```

把 b_q 左移 m 位即可对齐到 R1 的 scale。相比使用 FP32 乘法做对齐，bit-shift 在 CPU/GPU 上的开销低了一个数量级。

**Ablation 效果：** 仅加 Softmax twin（不加 GELU twin 和 Hessian），ViT-S/224 的 8-bit 精度从 80.47（base PTQ）提升到 80.93（+0.46）；再加 GELU twin 到 81.11（+0.64）。6-bit 的效果更显著——从 70.24 提升到 78.63（+8.39），说明 Twin Quantization 在低位宽下是救命级技术。

## 四、Hessian Guided Metric：用损失函数曲率指导 scale 搜索

Twin quantization 解决了"用什么格式量化"，下一个问题是"怎么选最优 scale"。论文用 Taylor 展开分析量化扰动对任务 loss 的影响：

```
E[L(Ŵ)] - E[L(W)] ≈ ε^T · g̅^(W) + (1/2) · ε^T · H̅^(W) · ε
```

其中 ε 是量化引入的权重扰动，H̅^(W) 是 Hessian 矩阵。经过推导（假设预训练网络已收敛、忽略一阶项、层间解耦近似），优化目标简化为：

```
min_∆ E[(Ô_l - O_l)^T · diag((∂L/∂O_l)^2) · (Ô_l - O_l)]
```

**通俗理解**：不只是看当前层的输出误差 `Ô_l - O_l` 有多大，还要看"这个误差沿着损失函数曲面的陡峭方向有多大的影响"。Hessian 对角项 `(∂L/∂O_l)^2` 起到了逐元素加权的作用——对 loss 影响大的输出维度，量化误差被放大惩罚。

对比 Fig. 5 中的各指标曲线：Hessian guided metric 的极小值点最接近 cross-entropy 的极小值点。从 ablation 数据看，在已启用 twin quantization 的基础上，Hessian guided metric 额外贡献约 0.2~0.5 pt（例如 ViT-B/224 8-bit：81.11 → 81.25）。

计算成本：一次前向（收集每层输出 O_l）+ 一次反向（收集梯度 ∂L/∂O_l），不需要传播到全部参数。单张 V100 上 ViT-S 约 2 分钟、ViT-B 约 6 分钟，相比 QAT 的天级训练是数量级的节省。

## 五、PTQ4ViT 完整流程

```
Algorithm: PTQ4ViT

阶段一：统计收集
  for l = 1 to L:
      前向传播，记录第 l 层输出 O_l
  for l = L to 1:
      反向传播，计算梯度 ∂L/∂O_l

阶段二：逐层搜索最优 scale
  for l = 1 to L:
      初始化 ∆_B ← B_max / 2^(k-1)
      生成 ∆_A 和 ∆_B 的搜索空间（n 个候选，α, β 控制范围）
      for r = 1 to #Round:
          固定 ∆_B，搜索 ∆_A 最小化 Hessian 距离
          固定 ∆_A，搜索 ∆_B 最小化 Hessian 距离
      应用 found ∆_A, ∆_B 量化该层
      校准下一层时使用当前层的量化输出
```

关键参数：校准图像 32-128 张即可饱和（论文证明 20 次随机抽样 std 仅 0.035%-0.094%）；不需要训练数据增强（RandAugment/RandomResizedCrop 会拉宽激活统计）；不需要标签。

### 6.2 与同期方法对比（Table 2）

| 方法 | 校准图数 | DeiT-S W8A8 | DeiT-S W6A6 |
|------|---------|------------|------------|
| EasyQuant (MSE-based) | 1024 | 76.59 | 73.26 |
| Liu et al. (Pearson-based) | 1024 | 77.47 | 74.58 |

| **PTQ4ViT** | **32** | **79.47** | **76.28** |

PTQ4ViT 用 **1/32 的校准数据量**，精度远超同期方法，且不需要混合精度（MP）和 bias correction（BC）。

### 6.3 Ablation：每项贡献

以 ViT-S/224 W8A8 为例：

| 配置 | W8A8 |
|------|------|
| Base PTQ | 80.47 |
| + Softmax Twin | 80.93 (+0.46) |
| + Softmax Twin + GELU Twin | 81.11 (+0.64) |
| + Hessian Guided Metric | 80.84 (+0.37) |
| Softmax + GELU Twin + Hessian | **81.00 (+0.53)** |

**结论：三项改进各自独立有效，叠加后累计 +0.53 pt。**

### 6.4 关键观察

- **模型越大越耐量化**：ViT-S/224 8-bit 掉 0.38，ViT-B/224 掉 0.29，ViT-B/384 只掉 0.17
- **Swin 天然耐量化**：Swin 的窗口内 attention 减少了 post-Softmax 的长尾效应，Base PTQ 下 8-bit 就只掉 0.26（ViT-B 掉 0.64）
- **6-bit 是 Twin Quantization 的主场**：Base PTQ 在 6-bit 下平均掉 9.8 pt，PTQ4ViT 把这一数字压缩到 2.1 pt

## 七、为什么 PTQ4ViT 重要：工程与学术影响力

PTQ4ViT 是 ViT 量化领域的**里程碑论文**。它做了三件至今仍在主导该方向的事：

1. **定义了 ViT 量化的三大问题**：post-Softmax power-law、post-GELU asymmetry、scale search metric failure。后续所有 ViT 专用 PTQ 方法（FQ-ViT、RepQ-ViT、APQ-ViT 等）都在解同一组问题，只是换了更工程化的方法。
2. **确立了两条技术路线**：专用量化格式（twin uniform → FQ-ViT 的 LIS → RepQ-ViT 的 log2）和面向任务 loss 的 scale 搜索（Hessian → APQ-ViT 的 BBC → ERQ 的显式误差消减）。
3. **证明了 PTQ 可以逼近无损**：在 W8A8 上全线 <0.5% 掉点（Swin-B/384 到 -0.04%），意味着 ViT 量化在 8-bit 已经工程可用，不需要 QAT。

**工程落地注意事项：**

- Twin uniform 需要自定义内核做 bit-shift 对齐，在 TensorRT 标准 Q/DQ 范式中不是开箱即用的。实际部署通常要映射为：post-Softmax → FP8（指数位天然适配）或 log2 查表；post-GELU → RepQ-ViT 的 scale 重参数化
- Hessian guided metric 只在 PTQ 阶段发挥，推理时无需额外开销
- 校准图像 32-128 张，按类均匀抽样，**不做训练式数据增强**（会把激活统计拉宽！）
- 校准复杂度：单张 V100 上 ViT-S 约 2-7 分钟，Swin-B/384 约 25-69 分钟

## 八、核心 Takeaway

1. **ViT 量化的三大杀手**：post-Softmax 幂律长尾、post-GELU 正负不对称、局部 scale 搜索指标失准。这三个问题是结构性的，与模型大小无关，Swin 因窗口 attention 天然部分规避了 Softmax 问题。
2. **Twin uniform quantization 是解决不对称分布的经济方案**：用最高位做 range flag、bit-shift 做对齐，在通用硬件上高效，6-bit 下相对 base PTQ 拯救了 8+ 个点。
3. **Hessian guided metric 让 scale 搜索不再依赖不可靠的局部指标**：一次额外的反向传播，换来 0.2~0.5 pt 的稳定提升。
4. **8-bit 已经工程可用，6-bit 在专用格式下也可接受**：PTQ4ViT 全线 W8A8 ≤0.5% 掉点，W6A6 平均掉 2.1%。后续方法（RepQ-ViT、FQ-ViT、ERQ）在 6-bit 和 4-bit 上持续改进，但 PTQ4ViT 的基本诊断框架一直在被沿用。
