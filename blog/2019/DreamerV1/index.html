DreamerV1

paper link

研讨的是一篇在基于模型的强化学习（Model-Based Reinforcement Learning, MBRL）领域具有里程碑意义的论文：“Dream to Control: Learning Behaviors by Latent Imagination”，也就是大家熟知的 Dreamer。这篇论文由 Danijar Hafner 等人发表于 ICLR 2020。

为什么这篇论文如此重要？因为它优雅地解决了长期以来困扰 MBRL 的核心矛盾：如何在保持高样本效率（Sample Efficiency）的同时，获得不仅能通过渐进性能测试，甚至超越无模型（Model-Free）方法的策略性能。

第一部分：背景与动机

1.1 强化学习的双重困境

深度强化学习（Deep RL）在过去几年取得了惊人的成就（如 Atari, AlphaGo）。主要分为两派：

  • Model-Free RL (MFRL)：如 DQN, A3C, D4PG。
    • 优点：渐进性能（Asymptotic Performance）通常很高，能学会复杂的策略。
    • 缺点：样本效率极低。需要与环境交互数百万甚至数亿次。就像一个人学骑自行车，必须摔倒一万次才能学会，而不能通过思考来学习。
  • Model-Based RL (MBRL)：试图建立环境模型（Dynamics Model）。
    • 优点：样本效率高。可以通过模型进行规划（Planning）。
    • 缺点：模型误差（Model Bias/Compounding Error）容易导致策略崩溃；且在高维输入（如像素图像）下建模极难。

1.2 世界模型（World Models）

人类是如何学习的？我们不仅通过试错，更通过潜意识的想象（Mental Rehearsal）。我们的大脑中有一个“世界模型”，可以预测行为的后果。

Hafner 等人在 2019 年提出了 PlaNet (Deep Planning Network)。PlaNet 学习了一个隐空间动力学模型，并使用交叉熵方法（CEM）进行在线规划（Online Planning）。

  • PlaNet 的问题：它在做决策时需要进行大量的采样搜索，计算昂贵；而且它只看未来有限的步数（Horizon），是“短视”的。

1.3 Dreamer 的核心洞见

Dreamer 的核心思想可以用一句话概括：不要只在测试时做规划，要在训练时通过“想象”来训练一个策略网络。

Dreamer 依然沿用了世界模型的架构，但在行为学习上做出了本质改变：

  1. 不再仅仅依赖在线规划：而是训练一个 Actor 网络（策略）和一个 Critic 网络（价值）。
  2. 利用解析梯度（Analytic Gradients）：传统的 MBRL 往往把模型当成黑盒做采样（如 CEM），而 Dreamer 利用神经网络的可微性，直接将价值函数的梯度通过动力学模型反向传播给策略网络。

第二部分：世界模型架构 (Latent Dynamics Model)

Dreamer 使用的世界模型结构与 PlaNet 一致，称为 RSSM (Recurrent State Space Model)。这是一个结合了确定性路径和随机路径的模型，非常关键。

2.1 为什么要隐空间（Latent Space）？

如果直接在像素空间（Pixel Space）预测下一帧图像（例如 $64 \times 64 \times 3$），计算量巨大且难以捕捉高层语义。我们需要将高维观测 $o_t$ 压缩到低维隐状态 $s_t$。

2.2 RSSM 详细结构

RSSM 将状态分解为两部分：

  1. 确定性状态 (Deterministic State, $h_t$)：由 RNN (GRU) 建模，负责携带长期的历史信息。
  2. 随机状态 (Stochastic State, $z_t$)：由后验分布采样得出，负责捕捉环境的不确定性。

在论文中，作者统称隐状态为 $s_t$，但在具体实现和理解上，我们要把 $s_t$ 看作 ${h_t, z_t}$ 的结合。

模型由以下三个主要组件构成（请大家注意公式中的依赖关系）：

  1. 表示模型 (Representation Model / Encoder):
    • $p_\theta(s_t s_{t-1}, a_{t-1}, o_t)$
    • 作用：融合历史信息和当前的真实图像观测，推断当前的后验状态。这相当于卡尔曼滤波中的“更新”步骤。
  2. 转移模型 (Transition Model / Prior):
    • $q_\theta(s_t s_{t-1}, a_{t-1})$
    • 作用：在没有看到真实图像 $o_t$ 的情况下，仅根据历史和动作预测下一个状态。这是我们在“做梦”（想象）时使用的核心组件。
  3. 观测模型 (Observation Model / Decoder):
    • $q_\theta(o_t s_t)$
    • 作用：从隐状态重构图像。这是为了给隐状态提供训练信号（让 $s_t$ 包含图像的语义信息）。
  4. 奖励模型 (Reward Model):
    • $q_\theta(r_t s_t)$
    • 作用：预测当前状态的即时奖励。这是后续强化学习的基础。

2.3 训练目标：变分下界 (ELBO)

我们如何训练这个世界模型？目标是最大化观测数据的对数似然。根据变分推断（Variational Inference），我们要最大化证据下界（ELBO）。

损失函数包含三个部分：

\[J_{REC} = \mathbb{E} \left[ \sum_t \underbrace{\ln q(o_t | s_t)}_{图像重构} + \underbrace{\ln q(r_t | s_t)}_{奖励预测} - \underbrace{\beta KL(p(s_t | \dots) || q(s_t | \dots))}_{KL散度正则化} \right]\]
  • 重构项：迫使隐状态 $s_t$ 包含能够还原图像的信息。
  • 奖励预测项：迫使隐状态包含能够预测奖励的信息。
  • KL 散度项：这是关键。它拉近“后验分布”（Representation，看到图片后的状态）和“先验分布”（Transition，没看图片预测的状态）的距离。
    • 思考题：为什么这一项如此重要？
    • 答案：因为在“做梦”时，我们没有真实图片，只能靠 Transition Model。如果 Transition Model 预测得不准（和 Representation Model 差别大），想象出来的轨迹就是胡说八道。

第三部分：在隐空间中学习行为 (Learning Behaviors by Latent Imagination)

这是 Dreamer 最核心的创新点。有了训练好的世界模型，我们现在可以关闭“眼睛”（切断真实环境交互），完全在“大脑”（隐空间）中通过想象来训练策略。

3.1 想象的轨迹 (Imagined Trajectories)

想象过程如下：

  1. 从重放缓冲区（Replay Buffer）中抽取真实的起始状态 $s_\tau$。
  2. 使用学习到的 Actor 网络 $a_\tau \sim q_\phi(a_\tau s_\tau)$ 选择动作。
  3. 使用学习到的 Transition Model 预测下一个状态 $s_{\tau+1}$。
  4. 使用 Reward Model 预测奖励 $r_{\tau+1}$。
  5. 重复 $H$ 步（Imagination Horizon，通常 $H=15$）。

这产生了一条轨迹：${s_\tau, a_\tau, r_\tau, s_{\tau+1}, \dots}_{t}^{t+H}$。

3.2 价值估计：$\lambda$-return (The Critic)

如果我们只看 $H$ 步内的奖励和，策略会变得非常“短视”（Short-sighted）。为了解决这个问题，Dreamer 引入了一个价值网络（Value Network / Critic）： \(v_\psi(s_\tau) \approx \mathbb{E} \left[ \sum_{n=\tau}^{t+H} \gamma^{n-\tau} r_n \right]\)

为了平衡偏差（Bias）和方差（Variance），Dreamer 使用了 $\lambda$-return（类似于 TD($\lambda$)）：

\(V_N^k(s_\tau) = \sum_{n=\tau}^{h-1} \gamma^{n-\tau}r_n + \gamma^{h-\tau} v_\psi(s_h)\) \(V_\lambda(s_\tau) = (1-\lambda) \sum_{n=1}^{H-1} \lambda^{n-1} V_N^n(s_\tau) + \lambda^{H-1} V_N^H(s_\tau)\)

  • 简单来说，这就是在不同步数的截断回报之间做加权平均。
  • Critic 的更新目标是回归这个 $V_\lambda(s_\tau)$。

3.3 策略优化：解析梯度 (The Actor)

这是最精彩的部分。请大家注意，Dreamer 的 Actor 不是像 REINFORCE 或 PPO 那样通过似然比（Likelihood Ratio）来更新的。

因为整个世界模型（Transition, Reward, Value）都是可微的神经网络，我们可以直接计算回报关于动作的梯度！

目标函数是最大化想象轨迹的期望价值： \(\max_\phi \mathbb{E}_{q_\theta, q_\phi} \left[ \sum_{\tau=t}^{t+H} V_\lambda(s_\tau) \right]\)

为了让梯度通过随机采样节点（Action sampling），Dreamer 使用了 重参数化技巧 (Reparameterization Trick)： \(a_\tau = \tanh(\mu_\phi(s_\tau) + \sigma_\phi(s_\tau) \cdot \epsilon), \quad \epsilon \sim \mathcal{N}(0, I)\)

这样，梯度 $\nabla_\phi$ 可以从 Value 网络出发，流经 Reward 网络，流经 Transition 网络（State），最终流回 Action 网络（Actor）。

  • 对比：
    • PlaNet (CEM): 盲目尝试一组动作序列，选好的，不计算梯度。低效。
    • Dreamer: 计算链式法则 $\frac{\partial V}{\partial s} \cdot \frac{\partial s}{\partial a} \cdot \frac{\partial a}{\partial \phi}$。高效，指引性强。

第四部分：算法流程与实现细节

4.1 算法总览 (Algorithm 1)

整个过程分为三个循环，通常并行执行：

  1. 数据收集 (Environment Interaction):
    • 在真实环境中使用当前的 Actor 执行动作。
    • 积累数据到 Replay Buffer。
    • 注意：每采集 1 个 episode，可能进行 100 次训练迭代。
  2. 动态学习 (Dynamics Learning):
    • 从 Buffer 采样序列（Sequence）。
    • 计算重构损失、KL 损失、奖励预测损失。
    • 更新 $\theta$ (Encoder, Decoder, Transition, Reward)。
  3. 行为学习 (Behavior Learning):
    • 在隐空间展开 $H$ 步想象。
    • 计算 Value Target ($V_\lambda$)。
    • 更新 Critic ($\psi$) 最小化价值预测误差。
    • 更新 Actor ($\phi$) 最大化价值（通过反向传播）。

4.2 关键超参数

  • Action Repeat: 2 (在连续控制任务中，每做一次决策重复执行 2 帧，提高效率)。
  • Horizon $H$: 15 (想象 15 步)。
  • $\lambda$: 0.95 (权衡长短期回报)。
  • KL Free Nats: 为了防止 KL 散度过大导致后验坍塌，通常会设置一个最小阈值（Clamping）。

第五部分：实验结果与讨论

5.1 DeepMind Control Suite 结果

请参考论文中的图表（Figure 6）。

  • 对比 D4PG (SOTA Model-Free): Dreamer 在大多数任务上达到了相当或更好的最终性能，但样本量（Environment Steps）仅为 D4PG 的 1/20 (5M vs 100M steps)。
  • 对比 PlaNet: Dreamer 完胜。因为 PlaNet 的规划视野有限，且 CEM 优化不如梯度下降精准。
  • 长程任务 (Long-Horizon): 如 Acrobot Swingup，Dreamer 表现优异，证明了 Value Network 成功捕捉了 $H$ 步之外的收益。

5.2 为什么要学图像重构？(Ablation Study)

论文对比了三种表示学习方法：

  1. 重构像素 (Pixel Reconstruction) - Dreamer 默认。
  2. 对比学习 (Contrastive Learning, NCE)。
  3. 仅预测奖励 (Reward Prediction Only)。

结果显示：重构像素通常表现最好。

  • 教授点评：虽然像素重构很难，但它提供了最丰富的监督信号，迫使隐状态 $s_t$ 理解物体的物理位置、接触关系等。仅预测奖励往往信号太稀疏，导致表示学习不够鲁棒。

5.3 局限性与未来

尽管 Dreamer 很强，但它也有局限：

  1. 视觉复杂性：重构整个图像在背景复杂或视角剧烈变化时非常困难（后续 DreamerV3 试图解决此问题）。
  2. 模型误差利用：Actor 可能会利用动力学模型的漏洞（Adversarial Example），导致在想象中表现好，在现实中表现差（虽然 KL 正则化缓解了这个问题）。

课后思考题

  1. 在 Dreamer 中，如果我们将 Imagination Horizon $H$ 设为 1，它会退化成什么？（提示：类似于带模型的 Actor-Critic，但失去多步规划优势）。
  2. 为什么说 RSSM 中的随机节点（Stochastic State）对于防止过拟合和处理环境随机性至关重要？如果去掉 $z_t$ 只保留确定性 $h_t$ 会怎样？
  3. Dreamer 能否应用于离散动作空间（如 Atari）？（提示：DreamerV2 解决了这个问题，使用了 Categorical Latent State）。
