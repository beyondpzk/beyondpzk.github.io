ROPE

我的理解就是把一个向量的一半看成是复数的实部,一半呢,看成是虚部.

对，你这个理解非常到位：RoPE 本质上就是把向量按偶/奇维两两配对，当成复数的实部/虚部，然后乘上一个与位置相关的单位复数，相当于“在复平面旋转”。

更严格地说（以每个 head 维度 D 为例，D 必须为偶数）：

  • 把第 2i 维当作实部 (a_i)，第 2i+1 维当作虚部 (b_i)
  • 组成复数：

$$ z_i = a_i + j b_i

$$

  • 对位置 (p)，定义旋转因子（单位复数）： \(e^{j\theta_i(p)} = \cos\theta_i(p) + j\sin\theta_i(p),\quad \theta_i(p)=p\cdot \omega_i\)
  • RoPE 就是做复数乘法： \(z'_i = z_i \cdot e^{j\theta_i(p)}\)

把复数乘法展开，就回到你在代码里看到的二维旋转公式：

\[\begin{aligned} a'_i &= a_i\cos\theta_i - b_i\sin\theta_i \\ b'_i &= a_i\sin\theta_i + b_i\cos\theta_i \end{aligned}\]

为什么这样能编码“相对位置”也可以用复数很直观地解释：

  • 位置 (p) 的 query：$z_q e^{j\theta(p)}$
  • 位置 (t) 的 key：$z_k e^{j\theta(t)}$
  • 点积对应到复数形式后会出现类似 $e^{j(\theta(t)-\theta(p))}$ 的差值项
    => 只跟 (t-p) 有关，因此天然是相对位置。

补一句工程实现：模型里不真的用 complex dtype，而是用实数张量把每对维度做一次“旋转”，但数学上等价于复数乘法。

RoPE（Rotary Position Embedding，旋转位置编码）详解（含具体例子）

RoPE 是一种把位置信息注入到 Transformer 注意力里的方法。它的核心特点是：不是把“位置向量”加到 token embedding 上（像绝对位置编码那样），而是对每个位置的 Q/K 向量做旋转变换，使得注意力分数天然依赖相对位置（token 间距离）。

RoPE 被 LLaMA、GLM、Qwen 等大量模型使用，尤其适合长上下文扩展（配合 NTK/YaRN 等变体）。

1. 先说结论：RoPE 在做什么？

对某个位置 p 的 query/key 向量：

  • 先把向量按二维一组分块（pairs）
  • 每一组应用一个与位置 p 相关的旋转角度
  • 旋转后的 Q、K 再去做点积注意力

形式上：

[ q’_p = R(p)\,q_p,\quad k’_p = R(p)\,k_p ]

注意力分数变为：

[ q’_p \cdot k’_t = q_p^T R(p)^T R(t) k_t = q_p^T R(t-p)k_t ]

关键：只依赖 相对位移 (t - p)，因此 RoPE 本质上编码了相对位置。

2. RoPE 的数学定义（二维旋转）

假设每个 head 的维度 d 是偶数。把向量的第 2i、2i+1 维视为一个二维坐标：

[ (x_i, y_i) ]

对位置 p，第 i 组的旋转角为：

[ \theta_{i}(p) = p \cdot \omega_i ]

其中频率：

[ \omega_i = 10000^{-2i/d} ]

旋转矩阵：

[ R(\theta)= \begin{bmatrix} \cos\theta & -\sin\theta
\sin\theta & \cos\theta \end{bmatrix} ]

于是：

[ \begin{aligned} x’_i &= x_i\cos\theta_i - y_i\sin\theta_i
y’_i &= x_i\sin\theta_i + y_i\cos\theta_i \end{aligned} ]

对 Q 和 K 都做同样操作，然后再算注意力。

3. 直观解释：为什么“旋转”能表达相对位置？

如果只看二维平面：

  • 同一个向量在不同位置会被旋转不同角度
  • 两个位置的旋转差值就是相对位移

点积里出现的 R(p)^T R(t) = R(t-p)，意味着：

  • 注意力分数只与两个 token 的距离 (t-p) 有关
  • 不依赖它们的绝对位置

这就是 RoPE “天然相对位置编码”的根源。

4. 具体数值例子（一步步算）

为了可算，我们用一个极简 head：

  • head_dim d = 4（即两组二维对）
  • 取简化频率：
    • 第 0 组：(\omega_0 = 1)
    • 第 1 组：(\omega_1 = 0.1)
      （真实 RoPE 用 10000^{-2i/d}，这里为了好算）

假设某层某个 head 上：

  • 位置 p=2 的 query 向量（未 RoPE）： [ q = [1, 0,\; 0, 1] ] 也就是两组二维向量：(1,0) 和 (0,1)

  • 位置 t=5 的 key 向量（未 RoPE）： [ k = [1, 0,\; 1, 0] ] 两组：(1,0) 和 (1,0)

4.1 先算旋转角

对位置 p=2：

  • 第 0 组角度：(\theta_0(p)=2\cdot 1 = 2)
  • 第 1 组角度：(\theta_1(p)=2\cdot 0.1 = 0.2)

对位置 t=5：

  • (\theta_0(t)=5)
  • (\theta__1(t)=0.5)

4.2 把 q 在位置 p 旋转

第 0 组 (1,0) 旋转角 2：

  • (x’ = 1\cdot\cos2 - 0\cdot\sin2 = \cos2)
  • (y’ = 1\cdot\sin2 + 0\cdot\cos2 = \sin2)

所以第 0 组变成：(cos2, sin2)

第 1 组 (0,1) 旋转角 0.2：

  • (x’ = 0\cdot\cos0.2 - 1\cdot\sin0.2 = -\sin0.2)
  • (y’ = 0\cdot\sin0.2 + 1\cdot\cos0.2 = \cos0.2)

所以：

[ q’_p = [\cos2,\;\sin2,\; -\sin0.2,\;\cos0.2] ]

4.3 把 k 在位置 t 旋转

k 的第 0 组 (1,0) 旋转角 5：

变为 (cos5, sin5)

第 1 组 (1,0) 旋转角 0.5：

变为 (cos0.5, sin0.5)

所以：

[ k’_t = [\cos5,\;\sin5,\; \cos0.5,\;\sin0.5] ]

4.4 算 RoPE 后的点积注意力分数

[ s = q’_p \cdot k’_t ] 分组相加：

第 0 组点积： [ \cos2\cos5 + \sin2\sin5 = \cos(2-5)=\cos(-3)=\cos3 ]

第 1 组点积： [ (-\sin0.2)\cos0.5 + (\cos0.2)\sin0.5 = \sin(0.5-0.2)=\sin0.3 ] （这是一个三角恒等式：(\sin(a-b)=\sin a\cos b - \cos a\sin b)，这里符号相当于做了对应变换）

所以总分数：

[ s = \cos3 + \sin0.3 ]

你会看到它只跟 相对位移有关：

  • 第 0 组体现了 cos(p-t) 形式（这里是 cos(2-5)）
  • 第 1 组体现了类似 sin((t-p)*0.1) 形式的相对项（因为角度与位置成正比）

这就是“相对位置被注入到注意力分数里”的具体体现。

5. RoPE 在实现上怎么做？（工程视角）

真实模型不会显式构造巨大旋转矩阵，而是：

  1. 预先算好每个位置、每个维度对的 cos / sin 表：
    • cos[p, i] = cos(p * ω_i)
    • sin[p, i] = sin(p * ω_i)
  2. 对 Q/K 做“偶奇维交换 + 乘 cos/sin”的向量化计算

典型伪代码（单个 head）：

# q,k: [seq, head_dim]  head_dim is even
# cos,sin: [seq, head_dim/2]

q1 = q[..., 0::2]     # even dims
q2 = q[..., 1::2]     # odd dims
q_rot = stack([-q2, q1], dim=-1).reshape_as(q)  # (x,y)->(-y,x)

q_out = q * cos_repeat + q_rot * sin_repeat
# k 同理

cos_repeat/sin_repeat 会把 [seq, head_dim/2] broadcast 成 [seq, head_dim]（偶奇维共享同一 cos/sin）。

6. RoPE 的优势与局限

优势

  1. 自然的相对位置：注意力分数依赖 (t-p)，对泛化友好
  2. 不增加可训练参数（纯函数式位置编码）
  3. 推理友好：可增量计算，适合 KV-cache
  4. 长上下文扩展空间大：很多扩展方法（NTK scaling、YaRN、PI 等）都是围绕 RoPE 做的

局限 / 需要注意

  1. 外推到超长上下文：直接用训练时的 RoPE 频率，长度超过训练上限时可能退化，需要 scaling/插值策略
  2. 位置索引的定义：从 0 开始还是 1 开始、是否有 prefix/特殊 token，都要一致
  3. 不同实现细节：base（10000）、theta 计算、维度排列方式不同会影响兼容性（如加载权重时）

7. RoPE 和“绝对位置编码”“相对位置偏置”的区别

  • 绝对位置编码（Sinusoidal / learned)：加到输入 embedding 上，注意力里不直接出现相对位移结构
  • 相对位置偏置（ALiBi、T5 bias）：直接在 attention logits 上加偏置项
  • RoPE：通过对 Q/K 的几何旋转，让点积自然呈现相对位移结构（不需要额外 bias）

小插曲:为什么位置编码是加在Q和K上面.

在自注意力里，注意力权重是由 (Q) 和 (K) 的相似度决定的（打分），而输出是用这个权重去对 (V) 做加权求和得到的。所以把“位置信息”注入到 打分这一步就够了——这也是为什么很多位置编码（尤其 RoPE）主要作用在 (Q/K) 上。

下面分几层说明。

1. 注意力里“位置”影响的是谁？

标准注意力（省略缩放）：

[ \text{Attn}(Q,K,V)=\text{softmax}(QK^\top)V ]

  • (QK^\top) 决定了“该看谁”（注意力分布）
  • 乘 (V) 决定了“从被看的 token 里取什么内容”（内容聚合）

位置编码的主要目标通常是：让模型知道“哪个 token 在哪个位置、谁在谁前面、相距多远”，这最直接体现在“该看谁”的分布上，因此把位置放进 (Q/K) 就能改变 logits (QK^\top)，从而改变注意力权重。

2. 为什么 RoPE 特别强调只对 (Q/K) 做旋转？

RoPE 的核心性质来自：

[ q’_p = R(p)q_p,\quad k’_t = R(t)k_t ]

于是打分：

[ q’_p \cdot k’_t = q_p^\top R(p)^\top R(t)k_t = q_p^\top R(t-p)k_t ]

这里得到的 (R(t-p)) 正是“只依赖相对位置”的关键。
这个漂亮的“相对位移只出现在打分里”的结构，本质上就是围绕 (Q/K) 的点积建立的。

如果你把同样的旋转也施加到 (V) 上，它不会影响注意力权重（softmax 的输入没变），只会改变“被取出来的内容向量长什么样”，并不直接解决“如何根据位置决定关注谁”。

3. 直观类比：先决定看谁，再决定抄什么

  • (Q/K)：像“检索条件”和“索引键”，决定匹配关系
  • (V)：像“被检索出来的正文内容”

位置编码更像是在索引阶段加入“顺序/距离”的规则：比如“更倾向看前一个词”“不能看未来”等，这些规则应当作用在匹配/检索阶段（(Q/K)）。

4. 那为什么有些方法会对输入 embedding 加位置（等价也影响到 V）？

绝对位置编码（learned/sinusoidal）常见做法是：

[ x_p = \text{token_emb}_p + \text{pos_emb}_p ]

然后 (Q,K,V) 都由同一个 (x_p) 线性投影得到，所以位置信息也“间接”进入了 (V)。

这类方法可以工作，但它把位置混进了值向量，等于“正文内容本身带了位置成分”，并不总是最干净/高效的方式。RoPE/ALiBi 这类方法更倾向于把位置主要放在打分/权重里。

5. 能不能也对 V 做位置编码？会怎样？

可以，但通常：

  1. 不会改变注意力分布（除非你同时改变 logits 或者引入额外项）
  2. 会让输出内容带上“旋转/偏置”的位置痕迹，模型可以适配，但未必带来收益
  3. 对 RoPE 而言，还可能破坏一些你希望保留的性质（比如让输出的语义空间更稳定）

因此在实践中，RoPE 这类“基于打分结构”的位置方案，默认只用于 (Q/K)。

6. 一个小补充：KV-Cache 也是原因之一（工程视角）

自回归推理时缓存的是 (K/V)。如果位置编码主要体现在 (K) 上，那么缓存的 (K) 已经是“带位置信息的键”，后续新 token 的 (Q) 只要用对应位置的旋转即可对上历史相对关系；(V) 保持“内容值”也更自然。

如果你愿意，我可以再进一步用一个对比实验（很小的张量例子）展示：

  • 只对 (Q/K) 加 RoPE 时，注意力 logits 如何随相对距离变化；
  • 再把 (V) 也旋转时，logits 不变但输出向量如何变化。
