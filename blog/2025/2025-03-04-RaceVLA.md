# RaceVLA：面向竞速无人机的视觉-语言-动作端到端导航

> **论文**：*RaceVLA: VLA-based Racing Drone Navigation with Human-like Behaviour*
> **作者**：Valerii Serpiva\*, Artem Lykov\*, Artyom Myshlyaev, Muhammad Haris Khan, Ali Alridha Abdulkarim, Oleg Sautenkov, Dzmitry Tsetserukou（\* 为共同第一作者）
> **单位**：Intelligent Space Robotics Laboratory, Skolkovo Institute of Science and Technology (Skoltech), Moscow, Russia
> **发布时间**：2025-03-04（arXiv），目前尚未发表于正式会议/期刊
> **arXiv**：[arXiv:2503.02572](https://arxiv.org/abs/2503.02572)
> **项目主页**：[https://racevla.github.io/](https://racevla.github.io/)
> **代码仓库**：[https://github.com/SerValera/RaceVLA](https://github.com/SerValera/RaceVLA)

## 摘要

RaceVLA 是首个面向**竞速无人机（Racing Drone）**的视觉-语言-动作（Vision-Language-Action, VLA）端到端导航系统。它接收机载第一人称视角（FPV）视频流与自然语言指令，直接输出四自由度飞行控制信号——三轴线速度 $(V_x, V_y, V_z)$ 与偏航角速度 $\omega$，使无人机能够在复杂、动态的赛道环境中完成穿门、循迹、避障等任务。该方法以 Stanford AI Lab 的 **OpenVLA-7B** 为基座，通过 LoRA 在作者采集的约 200 段、20,000 张图像的真实飞行数据上微调得到。实验表明，RaceVLA 在真实赛道上平均速度可达 **1.04 m/s**，最高速度达 **2.02 m/s**；在 OpenVLA 提出的四类泛化轴（视觉、运动、物理、语义）评测中，RaceVLA 在运动泛化（75.0 vs. 60.0）和语义泛化（45.5 vs. 36.3）上超过 OpenVLA，并在全部四个轴上优于 RT-2。

## 一、研究背景与动机

### 1.1 认知机器人与 VLA 范式

近年来，视觉-语言-动作（VLA）模型在机器人领域展现出强大的端到端控制能力。与过去“感知 → 规划 → 控制”的多模块级联方案不同，VLA 模型将视觉观察与自然语言指令直接映射为底层控制信号：

$$
\mathbf{a}_t = f_\theta\left(\mathbf{I}_t, \mathbf{l}\right)
$$

其中 $\mathbf{I}_t$ 为当前视觉观测，$\mathbf{l}$ 为自然语言任务描述，$\mathbf{a}_t$ 为动作输出。这种端到端架构不仅简化了系统，还使机器人能够借助大规模预训练获得跨任务、跨环境的泛化能力。

在 PaLM-E、RT-1、RT-2、RT-X、OpenVLA 等工作的推动下，VLA 已经在机械臂操作、人形机器人、四足机器人等平台上得到验证。然而，无人机——尤其是需要毫秒级决策的高速竞速无人机——在 VLA 方向仍然探索较少。

### 1.2 竞速无人机导航的独特挑战

竞速无人机面临以下关键难点：

- **高动态性**：飞行速度可达数米每秒，要求控制频率足够高、动作连续平滑；
- **视角剧烈变化**：FPV 相机随无人机高速运动，背景、目标尺寸与形变持续变化；
- **强实时性**：传统轨迹生成或 LLM 规划方法存在模块延迟，难以满足竞速场景；
- **语言-动作对齐**：飞行员通过自然语言下达“穿过拱门”“从右侧门通过”等指令，系统必须理解语义并实时响应。

现有无人机系统多依赖 Transformer 做高层规划、轨迹生成或技能选择，缺乏像 VLA 这样从像素到推力的直接映射。RaceVLA 正是为了填补这一空白而提出。

## 二、核心贡献

RaceVLA 的主要贡献可概括为：

1. **首个面向竞速无人机的 VLA 系统**：将 VLA 从机械臂操作领域迁移到高速飞行机器人，验证了其可行性。
2. **动作空间适配**：将 OpenVLA 的 7D 机械臂动作向量改造为 4D 无人机控制向量：
   $$
   \mathbf{a} = \left[V_x,\; V_y,\; V_z,\; \omega\right]^\top
   $$
3. **真实世界数据集开源**：采集 200 段、约 20,000 张图像的真实飞行数据，覆盖拱门、方形门、环形赛道等任务。
4. **完整的软硬件闭环系统**：包括 FPV 相机、机载 Intel NUC、OpenVINS 视觉惯性里程计、Flask API 通信、以及量化到 int8 的 VLA 模型，在 NVIDIA RTX 4090 上达到 4 Hz 推理频率。
5. **系统的泛化评测**：在视觉、运动、物理、语义四个维度与 OpenVLA、RT-2 进行对比，RaceVLA 在运动与语义泛化上优于 OpenVLA，并在四轴上均优于 RT-2。

## 三、方法详解

### 3.1 系统架构概览

RaceVLA 的整体架构如图 2 所示，包含三个主要部分：

1. **机端**：FPV 相机（RealSense T265）采集图像，Intel NUC 运行 OpenVINS 定位与底层控制，Speedy Bee F405 飞控执行 MAVROS 指令；
2. **通信层**：通过 Flask API 将图像上传至服务端，并接收控制指令；
3. **服务端**：运行量化后的 OpenVLA-7B 模型，输入图像与语言指令，输出 4D 控制向量。

整个流程是**迭代式**的：每收到一帧图像，模型立即预测下一时刻动作并下发，而不是等待无人机到达某个航点。这种“逐帧控制”对于竞速无人机的连续飞行至关重要。

### 3.2 VLA 模型：从 OpenVLA-7B 到 4D 动作空间

RaceVLA 以 **OpenVLA-7B** 为基座。OpenVLA 原本用于机械臂操作，其动作空间为 7 维：

$$
\mathbf{a}^{\text{arm}} = \left[\Delta x, \Delta y, \Delta z, \Delta \phi, \Delta \theta, \Delta \psi, \Delta g\right]^\top
$$

其中前三项为平移增量，接下来三项为旋转增量，最后一项为夹爪开闭。RaceVLA 将其替换为适合无人机飞行的 4D 向量：

$$
\mathbf{a}^{\text{drone}} = \left[V_x,\; V_y,\; V_z,\; \omega\right]^\top
$$

模型目标可形式化为最大化条件动作似然：

$$
\theta^* = \arg\max_\theta \sum_{(\mathbf{I}, \mathbf{l}, \mathbf{a}) \in \mathcal{D}} \log p_\theta\left(\mathbf{a} \mid \mathbf{I}, \mathbf{l}\right)
$$

由于无人机任务不需要复杂物体交互，这种 4D 动作设计显著降低了动作空间复杂度，有利于运动与语义泛化。

### 3.3 自定义竞速无人机平台

作者搭建了一台 8 英寸竞速无人机，关键硬件配置如下：

| 组件 | 型号/说明 |
|------|-----------|
| 飞行控制器 | Speedy Bee F405，运行 ArduPilot 固件 |
| FPV/定位相机 | RealSense T265 |
| 机载计算机 | Intel NUC |
| 视觉惯性里程计 | OpenVINS |
| 通信协议 | MAVROS / ROS |
| 服务端 GPU | NVIDIA RTX 4090 |
| 推理精度 | int8 量化 |
| 推理延迟 | 0.25 s / 帧 |
| 控制频率 | 4 Hz |

4 Hz 的控制频率低于传统 PID/规划器，但对于 VLA 模型而言已是当时权衡实时性与模型容量的结果。未来提升速度的关键在于进一步降低推理延迟。

### 3.4 数据集采集与训练流程

#### 3.4.1 数据集构成

- **规模**：200 段飞行片段，约 20,000 张图像；
- **任务**：
  - “Fly through the arch gate”（穿过拱门）
  - “Fly through the square gate”（穿过方形门）
  - “Fly through multiple gates on circular track”（环形赛道多门）
  - “Fly through gates on circular track”（环形赛道门）
- **采集设备**：Vicon 动作捕捉系统以 60 Hz 记录速度，T265 相机以 30 Hz 采集图像；
- **数据对齐**：位置、速度分量 $(V_x, V_y, V_z)$、偏航角变化与图像帧同步；
- **格式**：使用 RLDS（Reinforcement Learning Dataset Specification）格式，兼容 OpenVLA。

#### 3.4.2 训练超参数

| 超参数 | 设置 |
|--------|------|
| 基座模型 | OpenVLA-7B |
| 微调方式 | LoRA（rank = 32） |
| 批次大小 | 16 |
| 学习率 | $5 \times 10^{-4}$ |
| 梯度步数 | 7000 |
| 图像增强 | 关闭（保持数据真实性） |
| 训练 GPU | 单张 NVIDIA A100 |
| 训练监控 | Weights & Biases（W&B） |

训练过程中监控训练损失、动作预测准确率与 L1 损失，并使用平滑指标保证评估稳定性。

## 四、实验

### 4.1 真实竞速任务评估

作者在真实赛道环境中进行了两类测试：

#### 测试一：单门/多门任务

任务包括穿过单门、根据语言指令选择左侧或右侧门、以及连续穿过多个门。飞行参数如下：

| 指标 | 数值 |
|------|------|
| 平均速度 | 0.51 m/s |
| 最大速度 | 0.80 m/s |
| 速度标准差 | 0.17 m/s |
| 平均偏航角速度 | 0.243 rad/s |
| 平均任务时长 | 26.92 s |

#### 测试二：环形赛道连续飞行

在“沿环形赛道穿过多个门”任务中，记录了三圈轨迹。结果如下：

| 指标 | 数值 |
|------|------|
| 平均速度 | 1.04 m/s |
| 最大速度 | 2.02 m/s |
| 速度标准差 | 0.27 m/s |
| 平均偏航角速度 | 0.40 rad/s |
| 偏航角速度标准差 | 0.12 rad/s |

这些结果表明 RaceVLA 能够在真实环境中保持较高的速度连续性，并完成复杂赛道导航。

### 4.2 VLA 泛化能力评估

为了量化泛化性能，作者参考 OpenVLA 的评测框架，设计了四个泛化轴，并进行了 200 组实验：

- **视觉泛化（Visual）**：新背景、新干扰物、目标外观变化；
- **运动泛化（Motion）**：未见的物体位置与朝向、新的赛道布局；
- **物理泛化（Physical）**：门的形状、尺寸变化；
- **语义泛化（Semantic）**：未见的无人机任务指令与目标对象。

与机械臂不同，竞速无人机的相机是运动的，因此单条训练样本中就包含丰富的视角变化，这是 RaceVLA 在运动泛化上表现突出的重要原因。

### 4.3 结果分析

下表汇总了 RaceVLA、OpenVLA 与 RT-2 在四个泛化轴上的得分：

| 泛化轴 | RaceVLA | OpenVLA | RT-2 |
|--------|---------|---------|------|
| 视觉（Visual） | 79.6 | 87.0 | 52.0 |
| 运动（Motion） | 75.0 | 60.0 | 55.0 |
| 物理（Physical） | 50.0 | 76.7 | 26.7 |
| 语义（Semantic） | 45.5 | 36.3 | 38.8 |

**关键观察**：

- RaceVLA 在**运动泛化**和**语义泛化**上超过 OpenVLA，说明动态相机视角和简化的动作空间有助于模型适应新布局和新指令；
- RaceVLA 在**视觉泛化**和**物理泛化**上低于 OpenVLA。原因可能包括：
  - 高速运动中目标尺寸变化大、形变严重；
  - 无人机必须通过机动飞行绕过干扰物，而不是像机械臂那样从固定视角观察；
  - 不同形状/大小的门在 FPV 图像中呈现显著的空间尺度差异，影响空间理解；
- RaceVLA 在**全部四个轴**上均优于 RT-2，体现了其在动态环境中实时调整的鲁棒性。

## 五、优势与局限

### 5.1 优势

1. **端到端架构**：省去传统无人机系统中目标检测、轨迹规划、底层控制等多个独立模块，降低系统复杂度；
2. **语言指令驱动**：用户可通过自然语言指定任务，无需编程航点；
3. **真实世界验证**：在真实无人机与真实赛道上完成实验，而不是仅在仿真中验证；
4. **开源生态**：数据集、预训练权重、代码均已在项目主页与 GitHub 开源，便于复现与扩展；
5. **运动/语义泛化突出**：动态相机为模型提供了丰富的视角变化，使其在未见布局与指令上表现较好。

### 5.2 局限

1. **控制频率有限**：4 Hz 的推理频率对于高速竞速仍有瓶颈。若要进一步提升速度，必须显著降低模型推理延迟，例如采用更小的模型、更高效的量化或边缘部署；
2. **视觉与物理泛化较弱**：在高速、尺度变化大的场景中，模型对未见目标外观和门尺寸变化的适应能力不足；
3. **数据集规模相对较小**：200 段、约 20,000 张图像的数据集虽然足以验证概念，但距离覆盖大规模、多样化环境仍有差距；
4. **动作空间简化**：仅输出 4D 速度/偏航控制，未涉及翻滚、俯仰等更精细的姿态控制，限制了极限机动能力；
5. **对动态障碍物的应对未充分验证**：实验主要围绕静态门和赛道布局，未展示对移动障碍物的实时规避能力。

## 六、历史意义与后续影响

RaceVLA 是 VLA 模型在**空中机器人**领域的早期探索之一。它的意义在于：

- **拓展了 VLA 的适用范围**：从地面机械臂、移动机器人延伸到高速飞行的无人机；
- **提出了 VLA 动作空间适配的思路**：将通用 VLA 模型通过动作空间改造与 LoRA 微调快速迁移到新平台；
- **为无人机竞速与自主飞行提供新范式**：展示了“语言 + FPV 视频 → 直接控制”的可行性；
- **推动数据集与评测标准建设**：其四类泛化评测为后续空中 VLA 研究提供了参考框架。

后续研究可能沿以下方向发展：

- 构建更大规模、更多场景的真实/仿真飞行数据集；
- 开发更轻量、更高频率的 VLA 模型，实现 10–30 Hz 甚至更高的控制频率；
- 引入世界模型、模型预测控制（MPC）与 VLA 结合，提升高速场景下的安全性；
- 将 VLA 扩展到多机协同、动态避障、灾难搜救等更复杂的空中任务。

---

## 七、总结

RaceVLA 将视觉-语言-动作（VLA）模型首次应用于竞速无人机导航。通过在 OpenVLA-7B 基础上进行动作空间改造与 LoRA 微调，作者实现了从 FPV 图像和自然语言指令到 4D 飞行控制信号的端到端映射。在真实赛道上，RaceVLA 达到了 1.04 m/s 的平均速度和 2.02 m/s 的峰值速度；在泛化评测中，它在运动与语义维度超越 OpenVLA，并在四轴上全面优于 RT-2。

尽管当前推理频率与视觉/物理泛化仍有提升空间，但 RaceVLA 证明了 VLA 在高度动态空中平台上的潜力。随着模型效率提升、数据集扩大以及多模态感知技术的发展，基于 VLA 的无人机有望在未来的竞速、物流巡检、搜救等场景中发挥更大作用。
