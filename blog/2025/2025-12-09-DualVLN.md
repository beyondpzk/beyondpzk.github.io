# DualVLN：慢思考、快执行——迈向通用视觉-语言导航的双系统基础模型

> **摘要**：尽管近来的大视觉-语言模型（VLM）提升了视觉-语言导航（VLN）的泛化能力，现有方法大多采用端到端流水线，将视觉-语言输入直接映射到短视距离散动作。这种设计往往产生碎片化的运动、高延迟，并且在动态避障等真实世界挑战中表现不佳。本文提出 DualVLN，首个双系统 VLN 基础模型，将高层推理与低层动作执行协同整合。System 2（基于 VLM 的全局规划器）通过图像接地推理"慢思考"，预测中期路径点目标；System 1（轻量级多模态条件 Diffusion Transformer 策略）利用来自 System 2 的显式像素目标和潜在特征"快执行"，生成平滑准确的轨迹。双系统设计实现了在复杂动态环境中的鲁棒实时控制和自适应局部决策。通过解耦训练，VLM 保留了泛化能力，而 System 1 实现了可解释且有效的局部导航。DualVLN 在所有 VLN 基准上超越此前方法，真实世界实验证明了其在动态环境中的鲁棒长程规划和实时适应能力。

## 1. 引言

视觉-语言导航（VLN）是机器人学中的关键任务——智能体接收自然语言指令和视觉观测，规划并执行到达目标位置的路径。该领域经历了三个阶段：离散动作 VLN（基于预定义节点图）、连续动作 VLN-CE（Habitat 仿真器中的连续控制）、以及物理真实 VLN-PE（引入真实运动控制器）。

近年来，大视觉-语言模型（VLM）的进步为 VLN 带来了新范式。VLN 系统可以将 VLM 的强先验知识通过后训练迁移到导航任务中，在多样化指令和环境间实现前所未有的泛化。然而，现有 Vision-Language-Action（VLA）模型大多采用紧耦合的端到端范式，直接将视觉-语言输入映射到短视距离散动作（如"前进 0.25m"）。这带来了三个关键问题：

- **动作碎片化与不自然**：每一步都是短视距决策，导致轨迹不连贯、机器人运动抖动
- **高延迟**：每个动作都需要调用大 VLM，难以满足实时控制需求
- **缺乏层次化协调**：视觉-语言推理、全局规划和局部控制纠缠在同一个网络中，难以处理敏捷控制和动态避障

DualVLN 的核心理念是解决以上问题，通过将 VLN 解耦为"慢思考"和"快执行"两个互补系统，使各自可以充分专业化。

## 2. 动机：为什么需要双系统？

### 2.1 认知科学的启发

受到认知科学中**双系统理论**的启发（System 1 快速直觉 vs. System 2 缓慢推理），DualVLN 将 VLN 解耦为：

- **System 2（慢系统）**：基于 7B 参数预训练 VLM（Qwen-VL-2.5），负责低频（2Hz）、鲁棒的高层推理，输出中期像素级导航目标
- **System 1（快系统）**：轻量级多模态条件 Diffusion Transformer（DiT）策略，负责高频（30Hz）、实时的低层轨迹生成

### 2.2 为什么解耦训练如此重要？

解耦带来三方面好处：
1. **VLM 保持泛化能力**：不被低层控制任务破坏其经过大规模预训练获得的强大先验知识
2. **异步运行**：System 2 每 500ms 做一次规划，System 1 每 33ms 生成一次轨迹，兼顾长程规划与实时反应
3. **可解释性**：System 2 输出的像素目标是人类可理解的中间表示

## 3. DualVLN 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         DualVLN                                 │
├─────────────────────────────┬───────────────────────────────────┤
│        System 2             │          System 1                 │
│   "Ground Slow"             │     "Move Fast"                   │
│   7B VLM (Qwen-VL-2.5)      │   Diffusion Transformer (DiT)     │
│   2 Hz                      │   30 Hz                           │
├─────────────────────────────┼───────────────────────────────────┤
│ • 自导向视角调整            │ • 显式像素目标条件                │
│ • 最远像素目标定位          │ • System 2 潜在特征条件           │
│ • STOP 决策                 │ • RGB + 历史观测条件              │
│ • 输出像素坐标 + 潜在向量   │ • 输出 32 个稠密平滑轨迹航点      │
└─────────────────────────────┴───────────────────────────────────┘
```

**运行流程**：
1. System 2 接收当前 RGB 图像、历史观测和语言指令
2. System 2 判断是否需要调整视角（左转/右转/抬头/低头各 15°）
3. System 2 在当前图像中预测最远的可见导航航点，表示为像素坐标 (u, v)
4. System 2 同时输出 4 个可学习的 latent query，提取该像素目标的紧凑潜在表示
5. System 1 以像素目标和潜在特征为条件，结合当前和历史观测，生成 32 个平滑轨迹航点
6. 低层控制器跟踪该轨迹，完成实际运动

## 4. System 2：基于 VLM 的全局规划器

System 2 基于 **Qwen-VL-2.5 (7B)** 进行全参数微调，核心能力包括三个方面。

### 4.1 最远像素目标定位

将智能体在 3D 空间中的未来轨迹投影到当前 egocentric RGB 图像上，利用深度图和相机-点距离判断每个轨迹点是否可见。在可见点中选择最远的一个作为正样本，得到像素坐标监督 (u*, v*)。

这种方式使 VLM 学会了在复杂室内环境中识别"沿指令方向能走多远"的关键位置，而不是仅做短视距动作决策。

### 4.2 自导向视角调整

直接将 3D 轨迹投影到 2D 图像存在两个问题：
- 若视角太高，地面点可能被遮挡
- 若当前朝向偏离指令方向，航点可能不在视野内

受人类导航时"环顾四周、低头看路"的启发，System 2 可以主动决定何时调整相机角度。可选动作包括：
- `Turn Left 15°`
- `Turn Right 15°`
- `Look Up 15°`
- `Look Down 15°`

这些动作最多连续执行 4 次，以寻找更有利的视角来预测像素目标。

### 4.3 System 2 的输出形式

System 2 根据当前情况输出三类结果之一：
1. **视角调整序列**：如 `→ → → →`（连续右转 60°）
2. **像素目标坐标**：如 `234 447`
3. **STOP**：任务完成

### 4.4 注意力分析

论文可视化了 System 2 在不同 Transformer 层的注意力图，揭示了有趣的现象：
- 浅层主要关注**广义上下文和空间线索**（物体、场景布局、方向提示）
- 深层注意力逐渐集中在**特定目标像素区域**
- 最深层的注意力对 **STOP token 赋予显著权重**，表明模型整合视觉和语言输入来判定任务完成

这种从宽泛场景理解到精确定位的渐进式注意力精炼，是 VLM 能够完成像素目标接地的关键原因。

## 5. System 1：多模态条件 Diffusion Transformer 策略

System 1 是一个轻量级的 Diffusion Transformer（DiT）策略，负责将 System 2 的规划转化为平滑、可执行、避障的低层轨迹。

### 5.1 为什么用扩散模型？

扩散模型在生成高维多模态分布方面表现出色：
- 生成**平滑、连续**的轨迹，而非离散短视距动作
- 天然支持**多模态**：同一指令下可能存在多条合理路径
- 通过条件机制融合像素目标、语言意图、历史观测等多源信息

### 5.2 两种条件：显式像素目标 + 隐式潜在特征

DualVLN 的一个关键设计是**同时使用显式和隐式两种目标表示**：

- **显式像素目标**：System 2 输出的像素坐标 (u, v)，编码为图像中的位置嵌入，提供明确的方向指引
- **隐式潜在特征**：在 System 2 的输出文本中插入 4 个特殊 token `<TRAJ>`，经过 VLM 前向传播后提取隐藏状态，将 3584 维特征线性投影到 768 维，作为 cross-attention 条件输入 DiT

仅依赖显式像素目标无法充分利用 VLM 丰富的隐藏特征，导致双系统退化为模块化流水线。隐式潜在特征提供了更丰富、更自适应的引导，使 System 1 能自动从 VLM 隐藏状态中提取任务相关表示。

### 5.3 Flow Matching 轨迹生成

给定真实轨迹航点 X₀ 和两个条件信号（轨迹潜在特征 Z' 和融合 RGB token F），在每个训练步随机采样扩散时间步 u ~ U(0,1) 和噪声 ϵ ~ N(0, I)。加噪轨迹定义为：

```
X_u = α_u X₀ + σ_u ϵ
```

DiT 被训练为在条件 Z' 和 F 下预测轨迹在时间步 u 的速度：

```
\dot{X}_u_pred = f_θ(X_u, u, Z' ⊕ F)
```

训练目标最小化预测速度与真实速度之间的均方误差：

```
L_flow = E[ ||\dot{X}_pred_u - \dot{X}_u||² ]
```

推理时，从随机噪声轨迹 X₁ ~ N(0, I) 出发，通过多次去噪步骤逐步恢复出清晰轨迹。

### 5.4 高频 RGB 与 Q-Former 压缩

由于双系统异步运行，System 2 在时间 t 产生的潜在目标在时间 t+k 时可能已过时。System 1 需要同时编码 System 2 最后一帧（时间 t）和当前观测（时间 t+k）的 RGB 特征。

两者先用 ViT 提取高维视觉特征，通过 self-attention 融合，再用 **Q-Former** 压缩为 32 个紧凑 token，作为 DiT 的高频视觉条件。这种设计使 System 1 能跟踪已行进的距离并适应环境变化。

### 5.5 网络设计

- 隐藏维度：384
- Transformer 层数：12
- 注意力头数：6
- RGB 编码器：DepthAnythingV2-Small 的 ViT
- 潜在特征投影：3584 → 768

轻量设计使 System 1 能在边缘设备上以 30Hz 实时运行。

## 6. 两阶段解耦训练

### 6.1 Stage 1：训练 System 2

- 基于 Qwen-VL-2.5 (7B) 全参数微调（视觉编码器和 LLM 骨干均解冻）
- 训练数据：StreamVLN 的数据配方
- 训练任务：自导向视角调整 + 最远像素目标定位 + STOP 动作预测
- 优化器：AdamW，初始学习率 2×10⁻⁵
- 批量大小：128 对话样本
- 训练步数：14,000 步（1 epoch）

### 6.2 Stage 2：训练 System 1

- **冻结 Qwen-VL**，保留泛化能力
- 可训练参数：4 个可学习 latent queries + DiT 扩散策略
- 训练数据：仅使用 System 2 的像素目标定位样本
- 优化器：AdamW，初始学习率 10⁻⁴
- 批量大小：128 轨迹样本
- 训练步数：15,000 步

**解耦训练的关键**在于 System 2 输出一个稳定的、可解释的中间表示（像素目标 + 潜在特征），System 1 只需学会如何从这个表示生成平滑轨迹。

## 7. Social-VLN：动态避障基准

### 7.1 动机

现有 VLN 基准（R2R-CE、RxR-CE、VLN-PE）大多在静态环境中评估，忽略了真实世界中动态障碍物（尤其是行人）的影响。DualVLN 论文提出了首个 **Social-VLN** 基准，系统评估机器人在执行导航任务时的社交意识和避障能力。

### 7.2 典型场景

- **正面接近**：行人与机器人相向而行
- **走廊交会**：在狭窄走廊中与行人错身
- **多行人同时出现**
- **路口等待**：在交叉路口等待行人通过
- **等待间隙通过**：在行人流中寻找间隙

### 7.3 评估指标

在标准 VLN 指标（NE、SR、SPL、OS、nDTW）基础上，新增 **人类碰撞率（HCR）** 指标，显式量化动态行人环境中的不安全交互。

### 7.4 训练数据收集

在每个训练片段中，使用人类检测传感器持续监控 egocentric 视图。当人类掩码像素比例超过预设阈值时，触发修改后的 A* 算法重新规划无碰撞轨迹。该过程在 60 个 MP3D 场景中生成了 **763K 社交导航片段**。

## 8. 实验结果

### 8.1 VLN-CE 基准（Habitat 仿真器）

| 方法 | R2R Val-Unseen NE↓ | SR↑ | SPL↑ | RxR Val-Unseen NE↓ | SR↑ | SPL↑ | nDTW↑ |
|------|-------------------|-----|------|-------------------|-----|------|-------|
| StreamVLN | 4.98 | 64.2 | 56.9 | 6.22 | 52.9 | 46.0 | 61.9 |
| NaVILA | 5.22 | 62.5 | 54.0 | 6.77 | 49.3 | 44.0 | 58.8 |
| **DualVLN** | **4.05** | **70.7** | **64.3** | **4.58** | **61.4** | **51.8** | **70.0** |

DualVLN 在 R2R 上 SR 提升约 **6.5%**，NE 降低近 1 米；在更具挑战性的 RxR 上提升更为显著。相比于此前方法，DualVLN 还引入了**全景观测**——利用上下两个 180° 鱼眼相机的 360° 全向输入，极大改善了空间感知和方向推理能力。

### 8.2 VLN-PE 基准（物理运动控制器）

VLN-PE 要求模型输出被真实运动控制器执行。DualVLN 在**零样本迁移**设定下（仅在 VLN-CE 训练，未在 VLN-PE 微调）超越了所有在 VLN-PE 上训练或微调的基线：

| 方法 | R2R Seen NE↓ | SR↑ | SPL↑ | R2R Unseen NE↓ | SR↑ | SPL↑ |
|------|-------------|-----|------|---------------|-----|------|
| NaVid | 6.20 | 21.58 | 17.45 | 5.94 | 22.42 | 18.58 |
| **DualVLN** | **4.13** | **58.97** | **47.78** | **4.66** | **51.60** | **42.49** |

DualVLN 的 SR 超过 NaVid 两倍，证明其轨迹输出可以直接迁移到真实物理控制器。

### 8.3 Social-VLN 动态环境实验

在动态环境中，DualVLN 和 StreamVLN 的成功率都有所下降，但 DualVLN 仍保持领先：
- DualVLN 成功率下降约 27%
- StreamVLN 下降约 26%

说明动态避障仍是挑战性任务，但 DualVLN 的双系统设计使其在避障后能更好地恢复原始任务轨迹。

### 8.4 真实世界实验

在走廊（易）、卧室（中）、办公室（难）三种真实场景中，DualVLN 与 NaVid、NaVILA、StreamVLN 对比：
- **走廊**：NE 约 0.3m，SR 接近 100%
- **卧室**：NE 约 0.6m，SR 显著高于基线
- **办公室（房间到房间）**：NE 约 2.2m，SR 大幅领先

定性分析显示：NaVid 在复杂任务中容易失败，NaVILA 能处理长程任务但常错过最终目标，StreamVLN 能部分避障但牺牲了任务完成度，而 DualVLN 在静态和动态场景中均保持高 SR 和低 NE。

## 9. 消融实验与关键洞察

### 9.1 不同目标表示的作用

| 变体 | NE | SR | SPL | OS |
|------|-----|-----|------|------|
| **DualVLN 完整版** | **4.05** | **64.3** | **58.5** | **70.7** |
| w/o Sys.2 Train（端到端联合训练）| +0.17 | -9.1 | -7.0 | -2.7 |
| w/o Pixel Goal（仅潜在特征）| +0.93 | -2.1 | -2.7 | -9.8 |
| w/o Latent Goal（仅像素目标）| +0.21 | -3.4 | -3.4 | -3.0 |

**三个关键洞察**：
1. **解耦训练显著优于端到端联合训练**（w/o Sys.2 Train 导致 SR 下降 9.1%），说明保护 VLM 泛化能力极其重要
2. **显式像素目标**对 System 1 至关重要（缺失导致 NE 增加近 1 米），提供了可解释的方向指引
3. **隐式潜在特征**提供补充性的语义信息，两者结合效果最佳

### 9.2 System 1 的鲁棒性与数据缩放

- System 1 对"方向正确但位置略有偏差"的像素目标具有鲁棒性
- 但如果像素目标语义上完全错误（如指向墙），System 1 也会失败
- 使用 System 2 训练数据的 **1%** 就已能取得有竞争力性能，**10%** 接近饱和——说明 System 2 是当前系统的性能瓶颈

### 9.3 像素目标与轨迹的一致性

将 System 1 预测的轨迹点投影回图像平面，计算其与像素目标的距离和角度偏差。结果显示大多数点集中在目标区域，验证了生成的轨迹确实朝向像素目标并到达其附近。

## 10. 技术贡献总结

1. **首个双系统 VLN 基础模型**：显式分离高层规划（System 2）与低层执行（System 1），实现异步慢-快控制
2. **最远像素目标定位**：将 VLN 规划转化为图像空间中的像素级目标预测，兼具可解释性和泛化性
3. **自导向视角调整**：让 VLM 主动调整视角，解决投影和视野受限问题
4. **显式+隐式双目标表示**：同时利用像素目标和 VLM 潜在特征，实现 System 1 的丰富条件化
5. **多模态条件 DiT 策略**：融合显式像素目标、VLM 潜在特征和 RGB 观测，生成平滑避障轨迹
6. **注意力机制分析**：揭示了 VLM 从全局上下文到精确定位的渐进式注意力精炼过程
7. **Social-VLN 基准**：首次系统评估 VLN 模型在动态行人环境中的社交避障能力（含 763K 社交导航数据）
8. **SOTA 性能**：在 VLN-CE、VLN-PE 上全面超越此前方法，真实世界验证稳定

## 11. 局限与未来方向

### 当前局限

1. **System 2 是瓶颈**：System 1 的数据缩放实验表明，系统性能上限受限于像素目标质量
2. **依赖深度图构建训练数据**：真实世界中深度图可能噪声较大
3. **未显式建模长期记忆**：当前历史观测处理方式相对简单，超长指令需要更强的记忆机制
4. **动态场景仍有性能下降**：虽优于基线，动态环境中仍有约 27% 的 SR 下降

### 未来方向

- **更强的 System 2**：引入 3D 场景图、拓扑记忆、长期上下文推理
- **端到端可微分双系统**：保持解耦优势的同时探索更紧密的协同训练
- **真实世界自监督数据收集**：利用真实机器人自主采集数据持续学习
- **多模态动作输出**：除轨迹外，输出头部姿态、手臂动作等全身控制指令
- **跨 embodiment 泛化**：从轮式/腿式平台扩展到无人机、机械臂等

## 12. 总结与启示

DualVLN 是 VLN 领域的一个里程碑式工作。它将认知科学中的双系统理论引入机器人导航，成功将大 VLM 的强推理能力与扩散模型的平滑轨迹生成能力结合起来。"慢思考、快执行"的设计理念不仅解决了端到端 VLA 模型动作碎片化、高延迟、缺乏层次化协调的问题，还通过像素目标和潜在特征构建了一个清晰可解释的中间表示。

从更宏观的视角看，DualVLN 代表了**层次化 VLA 架构**的一种通用设计范式：用大模型做低频、稳健的高层规划，用小模型做高频、敏捷的低层控制。这种解耦设计在提升系统可解释性的同时，也保留了 VLM 经过大规模预训练获得的泛化能力。
