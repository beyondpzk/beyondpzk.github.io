VLA-OS：视觉 - 语言 - 动作模型中的规划表示与范式结构化研究

第一部分：研究背景与动机

1.1 具身智能与机器人操作任务的挑战

构建能够感知、推理并与物理环境交互的智能机器人始终是机器人学社区的核心挑战。传统方法依赖于针对特定任务的策略学习，但这种方法在泛化能力和任务理解方面存在局限。

近年来，研究重点转向开发用于机器人操作任务的基础模型，通过在大规模数据集上训练视觉 - 语言 - 动作模型（Vision-Language-Action Models, VLAs）来实现。与计算机视觉和自然语言处理中的端到端基础模型不同， VLA 研究出现了一个重要转变：从仅生成动作的端到端范式，转向包含任务规划的策略学习流程。

1.2 范式转变的动因

这一转变源于机器人操作任务的固有层次结构特性：

  • 高层任务规划：理解任务意图、分解子目标、推理场景
  • 低层物理交互：精确的运动控制、力反馈处理、实时响应

相比仅生成动作的端到端 VLA，引入任务规划的方法展现出：

  • 在复杂长视距任务上具有更强的任务推理和理解能力
  • 更高的成功率
  • 更好的样本效率

1.3 当前研究的核心问题

然而，现有方法在多个维度上存在显著差异：

  • 网络架构设计
  • 规划范式选择
  • 表示形式
  • 训练数据来源

这导致研究界难以确定性能提升的确切来源，阻碍了领域的系统性进步。本论文聚焦于五个核心研究问题：

  1. 表示问题：任务规划和策略学习应采用何种表示？多表示组合是否更优？
  2. 范式问题：应采用单体模型联合执行规划与学习，还是分层范式？
  3. 瓶颈问题：任务规划与策略学习，哪个是当前操作任务的更大挑战？
  4. 可扩展性问题：引入任务规划的 VLA 是否保留基础模型的可扩展性优势？
  5. 性能问题：采用任务规划的 VLA 是否具有更好的泛化和持续学习能力？

第二部分：VLA 范式分类学（约 45 分钟）

2.1 四种 VLA 范式

论文将现有 VLA 方法按任务规划的整合方式分为四类：

（1）PlanningOnly-VLA（仅规划型）

  • 利用预训练 LLM/VLM 进行推理和任务规划，不生成低层动作
  • 将任务分解为可由预训练子技能执行的子任务，或输出预定义运动的参数
  • 局限性：VLM 和低层技能通常无法用新数据进一步训练，性能常低于其他范式
  • 本研究未纳入此范式

（2）ActionOnly-VLA（仅动作型）

  • 端到端方式，直接将视觉和语言输入映射到机器人动作
  • 代表性工作：RT-2、OpenVLA 等
  • 优点：架构简洁，推理速度快
  • 缺点：缺乏显式推理能力，长视距任务表现受限

（3）Integrated-VLA（整合型）

  • 使用单一模型同时执行任务规划和策略学习
  • 可进一步分为：
    • 显式规划：行动生成条件于规划嵌入或结果（如 Embodied CoT、CoT-VLA）
    • 隐式规划：规划的表示作为辅助损失，推理时不需要规划头（如 MDT、PIDM）
  • 关键特征：规划与动作生成共享VLM 主干，存在梯度连接

（4）Hierarchical-VLA（分层型）

  • 使用两个独立模型分别处理任务规划和策略学习
  • 规划模型与动作模型之间无梯度连接
  • 代表性工作：RT-H 及后续研究
  • 优点：模块化设计，规划错误限制在表示层，不影响低层策略

2.2 三种规划表示形式

论文研究并标注了三类任务推理表示：

（1）语言推理表示（Language Reasoning）

包含 8 个关键字段：

  • Task（任务描述）
  • Plan（高层步骤序列）
  • Subtask（当前子任务）
  • Subtask Reason（子任务理由）
  • Move（具体运动命令）
  • Move Reason（运动理由）
  • Gripper Position（夹爪位置轨迹）
  • Object Bounding Boxes（物体边界框）

（2）视觉推理表示（Visual Reasoning）

  • 使用离散位置令牌 <loc_i> 表示图像中的位置
  • 包含三类信息：
    • 物体边界框（top-left 和 bottom-right bin）
    • 末端执行器流（序列化的位置令牌）
    • 目标物体可供性（affordance 热图区域）
  • 相比语言表示，具有更强的空间语义信息，更贴近输入图像

（3）图像预想表示（Image Foresight）

  • 使用第三人称视角的未来第 K 步图像作为短期目标
  • 通过自回归方式生成，采用 VAR 模型的粗到细生成范式
  • 是最通用的规划表示形式

第三部分：VLA-OS 模型家族设计

3.1 整体架构设计理念

VLA-OS 是一个统一且可组合的 VLA 模型家族，支持不同的任务规划范式和表示形式。设计原则包括：

  1. 模块化解耦：将任务规划头与策略头设计为可插拔组件
  2. 统一的主干网络：所有变体共享相同的 VLM 架构，可从 HuggingFace 直接下载
  3. 最小化架构偏差：通过控制参数量和训练数据，隔离范式与表示的影响

VLA-OS 的三个主要变体：

  • VLA-OS-A：对应 ActionOnly-VLA 范式
  • VLA-OS-I：对应 Integrated-VLA 范式
  • VLA-OS-H：对应 Hierarchical-VLA 范式

3.2 VLA-OS-A：仅动作型基础模型

VLA-OS-A 直接生成动作，无任务规划阶段，同时作为其他范式的基础模型。

架构细节：

视觉编码器：

  • 使用 DINOV2 + SigLIP 的集成，优于单一视觉编码器 (好像OpenVLA就是这么搞的吧,两个vit,一个负责语义,一个负责理解)
  • 将输入图像块编码并投影到语言嵌入空间

LLM 主干：

  • 选择 Qwen2.5 系列（0.5B、1.5B、3B、7B）
  • 模型命名：-S（Small）、-B（Base）、-M（Middle）、-L（Large）
  • 使用 LLaVa v1.5 数据混合进行 VLM 微调

动作头（Action Head）：

  • 独立的 Transformer 解码器，层数与 LLM 相同
  • 采用分块因果注意力机制（Block-wise Causal Attention）
  • 本体感知令牌（proprioception tokens）的查询可关注：
    • LLM 的键值
    • 本体感知键值
  • 动作令牌的查询可关注：
    • LLM 的键值
    • 本体感知令牌
    • 自身令牌

动作空间设计：

  • 归一化连续增量末端执行器姿态（δp）
  • 夹爪开/合动作（σ）
  • 动作分块生成：a_t = ([δp, σ]_t, ..., [δp, σ]_{t+L-1})
  • 对于灵巧手：使用增量关节值作为动作空间

3D 动作头扩展：

  • 接收多视角深度图像输入
  • 使用相机内外参将 RGBD 融合为 3D 点云
  • 注入 CLIP 特征到点云（参考 3D Diffuser Actor）
  • 使用最远点采样（FPS）下采样点云
  • 每个点视为一个令牌输入动作头

3.3 VLA-OS-I：整合型模型

VLA-OS-I 在 VLA-OS-A 基础上添加可插拔的规划头，支持多种规划表示。

三种规划头设计：

（1）语言规划头：

  • Transformer 架构，层数与 LLM 主干相同
  • 从 LLM 每一层提取键值作为条件
  • 使用 LLM 的词表进行解码
  • 训练损失：交叉熵损失

（2）视觉规划头：

  • 架构与语言规划头相同
  • 扩展词表以预测位置令牌（共 P×P 个，P=32，即 1024 个位置令牌）
  • 使用位置令牌表示边界框、末端执行器流、可供性区域
  • 训练损失：交叉熵损失

（3）图像预想规划头：

  • 自回归图像生成模型
  • 采用 VAR 的粗到细生成范式
  • 迭代量化残差图像
  • 应用 Bitwise Self-Correction（BSC）技术减少训练 - 推理差距
  • 训练损失：遵循原论文方法

两种使用方式：

隐式规划（Implicit Planning）：

  • 动作头独立于规划头
  • 规划头仅作为辅助损失
  • 推理时不执行规划头
  • 优点：避免规划累积误差，提高推理速度

显式规划（Explicit Planning）：

  • 动作头关注规划头每一层的键值
  • 推理时必须先执行任务规划，再生成动作
  • 优点：可以链式思考方式解决复杂任务
  • 缺点：规划累积误差，推理延迟增加

3.4 VLA-OS-H：分层型模型

VLA-OS-H 使用两个独立网络分别处理任务规划和策略学习，无梯度连接。

高层模型（规划器）：

  • VLM + 规划头
  • 生成规划表示（语言、视觉或图像）
  • 从 Integrated-VLA 微调获得

低层模型（策略）：

  • 编码器 - 解码器 Transformer 架构
  • 层数为其他 VLA-OS 变体的一半（保持参数量相近）
  • 输入包括：
    • 原始视觉观测
    • 语言指令
    • 规划表示
    • 来自 AM-Radio 的冻结图像特征
    • 来自 Qwen2.5 的语言特征
  • 梯度不回传到高层 VLM 主干

3.5 参数量控制

为确保公平比较，论文对所有变体的规划头参数量进行限制：

  • 规划头参数限制为 VLM 的约 5%
  • 分层模型的动作头层数减半，补偿额外的规划模型参数

语言推理标注流程：

  1. 使用各类 VLM 进行初始推理生成
  2. 进行一致性和合理性过滤
  3. 提取任务、计划、子任务、运动命令等关键字段
  4. 标注物体边界框和夹爪位置

视觉推理标注：

  • 从仿真引擎获取实例语义图
  • 使用形态学操作减少噪声
  • 检测物体轮廓并计算边界框
  • 使用 SAM2 模型推断目标物体掩码

  • 转换为位置令牌表示

  • 从轨迹中选择未来第 K 帧作为目标图像
  • 无需额外标注工作

4.3 训练细节

VLM 预训练：

  • 基座模型：Qwen2.5 系列
  • 数据：LLaVa v1.5 混合数据（558K 图像描述 + 665K 多模态指令）
  • 优化：PyTorch FSDP + BF16 混合精度
  • 训练轮数：2 epochs

动作头训练：

  • 多模态演示数据集：流匹配损失（Flow Matching Loss）
  • 简单单模态数据集：L1 行为克隆损失

规划头训练：

  • 语言和视觉规划头：标准下一令牌预测损失（交叉熵）
  • 图像预想规划头：VAR 特定损失

持续学习实验：

  • 算法：顺序微调（SEQL）
  • 任务序列：LIBERO-LONG 的 10 个任务
  • 指标：前向迁移（FWT）、负后向迁移（NBT）、成功率曲线下面积（AUC）

第五部分：核心研究发现

5.1 模型规模与架构效率

发现 1：对于下游任务，在大数据集上训练的大型 VLA 模型并不一定优于从头训练的小型模型。模型架构和算法设计在当前仍然至关重要。

实验结果： | 模型 | 训练方式 | LIBERO 平均成功率 | |——|———|—————-| | Diffusion Policy | 从头 | 72.4% | | OpenVLA | 微调 | 76.5% | | CoT-VLA | 微调 | 81.1% | | DiT Policy | 微调 | 82.4% | | VLA-OS-A-S（0.5B） | 从头 | 85.6% | | π₀-FAST | 微调 | 85.5% | | π₀（SOTA） | 微调 | 94.2% |

VLA-OS-A-S（0.5B 参数，从头训练）优于所有微调基线，表明架构设计的重要性。

5.2 规划表示形式比较

发现 2：对于整合型 VLA 范式，隐式规划可产生正向性能增益，而显式规划在从头训练时会导致显著性能下降。

分析：

  • 隐式规划：辅助损失增强表示学习，推理时等同 ActionOnly-VLA，避免规划累积误差
  • 显式规划：规划令牌长度远超动作令牌（约 2000 vs 8），加剧累积误差；动作头仅接收 VLM 和规划头的嵌入，缺乏原始输入，无误差校正能力

发现 3：视觉接地（Visually Grounded）的规划表示（视觉规划和图像预想）在性能、推理速度和训练成本上均优于语言规划。

实验结果（LIBERO-LONG）： | 表示组合 | VLA-OS-I-I | VLA-OS-I-E | VLA-OS-H | |———|———–|———–|———| | L | 68.0% (↑2.0) | 60.5% (↓5.5) | 63.5% (↓2.5) | | V | 71.0% (↑5.0) | 52.5% (↓13.5) | 69.0% (↑3.0) | | IF | 72.5% (↑6.5) | 67.5% (↑1.5) | 71.7% (↑5.7) | | L+V+IF | 71.7% (↑5.7) | 50.7% (↓15.3) | 74.2% (↑8.2) |

发现 4：当同时使用多种规划表示时，分层型 VLA 的表现优于整合型 VLA。

5.5 规划头预训练收益

规划头预训练实验：

  • 方法：在 PlanningOnly 配置下先预训练规划头，再集成到 VLA 中
  • 结果：
    • VLA-OS-I：79.1%（+5.8%）
    • VLA-OS-H：79.8%（+5.6%）

表明任务规划预训练对两种范式都有显著增益。

5.6 可扩展性分析

数据可扩展性：

  • 在 LIBERO-LONG 上测试从 100 到 5000 条演示的影响
  • 所有 VLA 范式均展现出随数据量增加的性能提升
  • 数据边际收益递减点约在 2000 条演示附近

模型可扩展性：

  • 在 LIBERO-90（90 个任务）上测试 0.5B 到 7B 参数规模
  • 从头训练时，0.5B 模型性能最佳
  • 模型规模超过 1B 后出现性能下降

发现 12：对于从头训练且约 5000 条演示的任务，LLM 主干应限制在 0.5B 参数，或保持总模型规模小于 1B。

5.7 持续学习能力

范式 FWT（↑） NBT（↓） AUC（↑）
VLA-OS-A 0.71 0.32 0.25
VLA-OS-I 0.75 0.43 0.29
VLA-OS-H 0.80 0.45 0.32

发现 13：相比无规划范式，任务规划范式实现更高的前向迁移但遗忘更快。

发现 14：视觉接地表示在正向迁移和遗忘抑制上均优于语言表示。

持续学习指标定义：

  • FWT：对新任务的初始性能
  • NBT：学习新任务后对旧任务性能的负影响
  • AUC：整个学习过程中的平均性能

第六部分：深入讨论与开放问题

6.1 为何视觉接地表示更优？

可能原因：

  1. 空间对齐：视觉表示与机器人观测直接对应，减少模态转换误差
  2. 信息损失：语言表示需将连续空间信息离散化为文本，引入信息损失
  3. 泛化能力：视觉表示对物体位置、视角变化更鲁棒
  4. 策略跟随：低层策略更容易从视觉表示中解码空间信息

6.2 为何分层型在泛化上更优？

  1. 误差隔离：规划错误被限制在表示层，不传播到动作头的嵌入表示
  2. 独立优化：规划与策略可各自优化，不存在梯度冲突
  3. 多源输入：动作头可接收规划输出和原始观测，具有误差校正能力
  4. 迁移能力：规划模型可单独更换或升级

6.3 为何分层型推理成本更高？

  1. 串行推理：必须先完成规划，再生成动作
  2. 规划令牌长度：约 2000 个规划令牌 vs 8 个动作令牌
  3. 多头推理：需分别运行规划头和策略头

6.4 规划累积误差问题

论文通过 Figure 5 展示：当 VLA-OS-H 使用与 VLA-OS-I-E 相同但有误差的规划头时，仍能纠正行为，而 VLA-OS-I-E 不能。

原因：

  • VLA-OS-I-E 的动作头完全依赖规划头的键值，无原始观测输入
  • VLA-OS-H 的动作头可直接访问原始观测，可进行误差校正

这一发现建议：若采用显式整合型规划，必须保留动作头对原始信息流的访问通道。

6.5 为何小模型在从头训练时表现更优？

可能解释：

  1. 过拟合风险：大模型在小数据集上易过拟合
  2. 计算预算：固定训练步数下，小模型可更有效收敛
  3. 任务复杂度：当前基准可能未充分发挥大模型容量

这提示我们：盲目增加模型规模未必带来收益，需结合数据规模和任务需求选择参数量。

本次研讨课系统探讨了 VLA-OS 研究工作的核心内容：从 VLA 范式分类学、模型架构、训练细节到实验发现。关键收获包括：

这篇paper有什么结论

这篇论文系统地对比了三种VLA范式和三种规划表示,并得出了以下核心结论:

一、对比的VLA范式

范式 架构特点 代表模型
ActionOnly-VLA 端到端直接生成动作，无任务规划 OpenVLA, RT-2
Integrated-VLA 单一模型同时执行规划和动作生成 CoT-VLA, Embodied CoT
Hierarchical-VLA 两个独立模型分别处理规划和动作 RT-H

注：PlanningOnly-VLA因无法用给定数据集训练且性能较差，未被纳入研究。

二、核心研究发现

发现1：视觉接地表示优于语言表示

在任务性能、泛化能力、训练速度和推理速度上，视觉规划(Visual)和图像预想(Image Foresight)均优于语言规划(Language)。

视觉 (V) ↑5.0% ↑3.0%
图像预想 (IF) ↑6.5% ↑5.7%

发现2：分层型VLA整体表现最优

Hierarchical-VLA在以下维度达到最优或相当水平:

  • 任务性能
  • 泛化能力
  • 规划头预训练收益
  • 持续学习能力

代价: 训练成本更高，推理速度更慢

基准 ActionOnly Integrated Hierarchical
LIBERO-LONG 66.0% 73.3% 74.2%
泛化能力 6.1% 6.2% 7.4%

发现3：隐式规划优于显式规划

对于Integrated-VLA:

  • 隐式规划(规划仅作辅助损失)：性能提升 +5.0%~7.3%
  • 显式规划(动作依赖规划输出):性能下降 -2.5%~-15.3%

原因：显式规划存在严重的规划累积误差，且动作头无法访问原始观测进行误差校正。

发现4：策略学习比任务规划更难

在LIBERO基准上，无论使用何种规划表示，**策略学习 **(Policy Learning)。这表明当前VLA系统的主要瓶颈在于低层动作执行而非高层规划。

发现5：规划类VLA的持续学习特性

  • 带任务的VLA (Integrated/Hierarchical) 相比 ActionOnly:
    • ✅ 更高的前向迁移能力 (Forward Transfer)
    • ❌ 更快的遗忘速度 (Negative Backward Transfer)
  • 视觉表示相比语言表示:
    • ✅ 更好的前向迁移
    • ✅ 更慢的遗忘

发现6：模型规模并非越大越好

对于从头训练且约5000条演示的任务:

  • LLM骨干应限制在0.5B参数
  • 总模型规模应小于1B

超过此规模后性能反而下降，表明当前任务和数据规模下，架构设计比参数量更重要。

三、实践建议

论文建议研究方向:

  1. 优先使用视觉接地表示 (边界框、可供性热图、图像预想)
  2. 推荐采用Hierarchical-VLA架构,但需优化其训练和推理效率
  3. 采用隐式规划避免误差累积
  4. 小规模模型从头训练在当前数据规模下可能优于大规模模型微调
  5. 规划头预训练可带来约5-6%的性能提升

四、总结

论文的核心结论是：“视觉接地表示 + 分层型VLA + 隐式规划”是目前性能最优的组合,但Hierarchical-VLA的训练和推理效率问题仍有待解决。

这些发现为VLA社区提供了清晰的架构选择依据和未来的改进方向。

我的理解(智驾VLA vs 具身VLA)

关于自动驾驶 VLA 与具身 VLA 时序难度的讨论

你的观察很有洞察力的，但这个问题其实比表面看起来更复杂，值得深入讨论。

一、时序特性的本质区别

维度 具身智能 (机器人操作) 自动驾驶
典型任务时长 数秒到数分钟 数分钟到数小时
规划粒度 动作级 (毫秒 - 秒) 轨迹级 (秒 - 分钟)
错误累积敏感性 极高 (操作不可逆) 中高 (可通过修正补偿)
环境可预测性 低 (物体状态高度可变) 中高 (物理规律稳定)
反馈延迟 即时 可能有延迟

二、为什么具身VLA的”长时序”确实更难？

1. 任务层次结构的深度

具身任务（如”整理餐桌”）：

任务：整理餐桌
  └─ 子任务1：拿起脏盘子
  │    └─ 动作：移动机械臂 → 对齐夹爪 → 闭合 → 抬起
  └─ 子任务2：放入洗碗机
  │    └─ 动作：移动到洗碗机 → 打开门 → 放入 → 关门
  └─ 子任务3：擦拭桌面
       └─ 动作：拿起抹布 → 来回擦拭 → 放回

自动驾驶（如”从A点到B点”）：

任务：到达目的地
  └─ 子任务1：当前路口直行
  │    └─ 动作：保持车道 → 避让 → 减速
  └─ 子任务2：下一个路口左转
  │    └─ 动作：变道 → 等待 → 转弯
  └─ 子任务3：停车场泊车
       └─ 动作：找车位 → 倒车 → 调整

关键差异：

  • 机器人操作中，子任务之间往往存在强依赖关系，前一步失败会导致后续全部失败
  • 驾驶任务中，子任务相对可解耦，一个路口走错可以重新规划路线

2. 状态空间的可预测性

具身环境：

  • 物体可能是可变形的（如衣服、绳子）
  • 物体可能被遮挡或移动
  • 接触动力学高度非线性

驾驶环境：

  • 道路结构相对稳定
  • 交通规则提供强约束
  • 其他车辆行为有一定可预测性

3. 感知 - 动作闭环的紧密程度

特性 具身VLA 自动驾驶VLA
采样频率 50-200Hz 10-30Hz
动作延迟容忍度 毫秒级 百毫秒级
感知 - 动作耦合 紧耦合 相对松耦合

三、但自动驾驶也有其独特的”长时序”挑战

虽然你提到自动驾驶”大多数场景不需要长时序”，这个说法在某些场景下需要细化：

1. 需要长时序的驾驶场景

  • 多路口导航（可能需要5-10分钟的规划窗口）
  • 高速公路汇入/汇出（需要提前数百米准备）
  • 复杂路口的博弈（需要预测其他车辆5-10秒后的行为）
  • 停车入库（多步骤调整）

2. 为什么这些场景”感觉”不难？

  • 结构化环境：道路标识、交通规则提供强先验
  • 有限交互：主要是避让，无需主动操作物体
  • 错误可恢复性：开错路口可以掉头，不像机器人操作可能直接导致任务失败

四、论文中与你的观察对应的发现

VLA-OS 论文中的一些发现其实支撑了你的观点：

“Compared to end-to-end VLAs that only generate actions, these methods demonstrate stronger capabilities in task reasoning and comprehension for long-horizon tasks”

论文发现引入任务规划的VLA在长视距任务上优势更明显。这暗示：

  1. 短时序任务（如简单的抓取、单步操作）可能不需要复杂的规划
  2. 时序长度是决定是否需要任务规划的关键因素之一

如果将VLA-OS的框架迁移到自动驾驶：

  • 高速巡航等短时序场景：ActionOnly-VLA可能足够
  • 复杂城市导航：可能需要Hierarchical-VLA

  2. 具身VLA的”规划累积误差”问题在自动驾驶中是否存在？如何权衡？

你对这个领域有深入思考的话，这个比较视角其实可以作为未来研究的一个出发点。

自动驾驶VLA与具身VLA

一、共同点（自动驾驶 VLA 与具身机器人 VLA 都成立的部分）

1) 都是在做“视觉-语言-动作”的端到端融合

VLA 的核心是把视觉、语言与动作三模态统一到一个多模态“中枢/大脑”，让系统形成“观察-思考-行动”的闭环，从而实现“看懂、听懂、动手/行动”。[1][2]

2) 都强调用语言（或语言化的中间推理）增强决策的通用性

检索结果将 VLA 描述为“以语言为中心的认知模拟”，用统一多模态表示来完成决策与行动映射，目标是更通用、更可迁移的智能决策范式。[2]

3) 都可视为从 VLM/LLM 走向“可执行动作”的路径

检索结果指出预训练常见产物是 LLM 或 VLM，而 VLA 在此基础上进一步面向动作输出与控制落地（尤其在自动驾驶侧常体现为后训练）。[3]

二、关键差异点（自动驾驶 VLA vs 具身机器人 VLA）

差异1：控制与动作空间复杂度（自由度/精度）差别巨大

  • 自动驾驶：控制相对“简单”，车辆通常被描述为只需约 6 自由度；常见工程路径是由 VLM/VLA 解码出 waypoints（轨迹点），再交给 MPC（模型预测控制）将轨迹点分解为横向/纵向控制，由底盘与转向执行。[4]
  • 具身机器人：动作“更像小脑”，控制更复杂，最高可达 35 自由度，并且动作精确度与连续控制难度更高。[4]

结论：同为 VLA，但自动驾驶更容易把“动作”抽象为轨迹/航点再由传统控制器执行；具身机器人更难把动作抽象得这么“低维”，需要处理更高维、更精细的执行问题。[4]

差异2：训练数据侧重点不同（具身更依赖“遥操作轨迹视频数据”）

检索结果强调：

  • 具身智能相比自动驾驶，需要加入大量“机器人真人遥控轨迹视频数据”。[3]
  • 自动驾驶则被描述为“基本不怎么干预预训练”，其 VLA 工作重点更多在后训练（SFT+RL）。[3]

结论：具身机器人更强调“示教/遥操作”的行动轨迹数据闭环；自动驾驶侧更突出在已有模型基础上进行监督微调与强化学习式后训练（至少在该来源的观察框架中）。[3]

注：检索结果未给出两类数据的规模对比、采集成本、标注形式等量化细节，因此无法进一步做“数据工程复杂度”的定量结论（信息不足）。

差异3：系统分工方式不同（自动驾驶更常见“VLA→轨迹→MPC”的层次化落地）

自动驾驶可由多模态模型输出 waypoints，再由 MPC 负责可控性与执行稳定性；这意味着 VLA 在自动驾驶中常被放在“高层决策/规划”位置，而不是直接驱动底层执行器。[4]

具身机器人侧，来源强调“小脑部分较为复杂”，暗示端到端或更紧耦合的控制难题更突出。[4]

差异4：增强手段侧重点（自动驾驶讨论中更常把 RL/世界模型与 VLA 结合）

检索结果提到 IRL-VLA 架构：用强化学习与世界模型来增强 VLA。[3]
同时另一来源专门对比了 VLA 与 WA（世界模型范式），说明在智能驾驶语境下，经常把“世界模型/物理建模”作为与 VLA 并列或互补的路线来讨论。[2]

注：检索结果并未明确说明“具身机器人侧是否同样普遍采用 IRL-VLA 或世界模型增强”，因此不能断言其普及程度（信息不足）。

三、自动驾驶中的 VLA：难点与挑战（基于检索结果可支撑的要点）

1) “VLA 输出”与“可控执行”之间的工程衔接

自动驾驶常见做法是：VLM/VLA 给出 waypoints，再由 MPC 做横纵向控制分解与执行。挑战在于：
