VPT:Video PreTraining (VPT): Learning to Act by Watching Unlabeled Online Videos

paper link

利用少量标注数据训练逆动力学模型（IDM）为海量无标注互联网视频生成伪标签，进而通过大规模行为克隆预训练出具备强大零样本能力与微调潜力的通用行为基础模型。

视频预训练（VPT）——通过观看无标注在线视频学习行动

第一部分：引言与研究背景

1.1 大模型时代的“行为”缺失

在自然语言处理（NLP）和计算机视觉（CV）领域，我们已经见证了“基础模型（Foundation Models）”的巨大成功。像GPT系列、BERT、CLIP等模型，通过在海量互联网数据（文本、图像）上进行预训练，展现出了惊人的泛化能力和少样本学习能力。

然而，在 序列决策（Sequential Decision Making） 领域，如机器人控制、游戏AI、计算机操作代理等，我们还没有看到类似规模的成功。为什么？

  • 核心矛盾：互联网上充斥着大量的视频数据（比如YouTube上的Minecraft游戏视频），这些数据包含了丰富的行为信息。但是，这些视频是无标注的（Unlabeled）。我们只有像素（Observations），没有动作（Actions）。
  • 传统方法的局限：
    • 强化学习（RL）：从零开始训练（Tabula Rasa）在长视界（Long-horizon）、稀疏奖励的任务中效率极低。探索（Exploration）极其困难。
    • 模仿学习（IL/BC）：需要专家演示数据（State-Action Pairs）。获取大规模的带动作标注的数据极其昂贵且困难（例如，需要通过特殊的记录软件让大量人类玩家玩游戏）。

1.2 本文的核心贡献 (VPT)

这篇OpenAI的论文（NeurIPS 2022），提出了一种范式转移：视频预训练（Video PreTraining, VPT）。

核心思想：利用少量的标注数据训练一个“逆动力学模型（Inverse Dynamics Model, IDM）”，用IDM去给海量的无标注互联网视频打上“伪标签（Pseudo-labels）”，最后用这些伪标签数据训练一个大规模的行为克隆（Behavioral Cloning, BC）模型。

里程碑结果：

  • 在Minecraft（我的世界）中，使用原生键鼠操作（Native Human Interface）。
  • 实现了“零样本（Zero-shot）”造工作台。
  • 通过微调，成为首个能够从零开始制作钻石镐（Diamond Pickaxe）的AI代理。这是一个人类熟手平均需要20分钟（约24,000个操作步）才能完成的任务。

第二部分：VPT 方法论详解

2.1 整体流程 (Pipeline)

VPT的半监督模仿学习流程，它分为四个关键步骤：

  1. 收集少量标注数据：雇佣承包商（Contractors）玩游戏，记录视频 + 按键/鼠标动作。
  2. 训练逆动力学模型 (IDM)：学习映射 $f: (Observation_Sequence) \rightarrow Action$。
  3. 大规模伪标注：下载海量YouTube视频，用IDM预测每一帧的动作。
  4. 训练VPT基础模型：在这些伪标注数据上进行大规模行为克隆。

2.2 核心组件一：逆动力学模型 (IDM)

这是VPT最精妙的地方。为什么不直接用承包商数据训练BC模型？

  • 这是一个关于信息论与因果性的问题。
    • 行为克隆 (BC) 是因果的（Causal）：$P(a_t o_1, …, o_t)$。这很难，因为需要推断意图及未来分布。
    • 逆动力学 (IDM) 可以是非因果的（Non-causal）：$P(a_t o_{t-k}, …, o_t, …, o_{t+k})$。既然我们是在离线处理视频，我们完全可以利用未来帧的信息来推断当前帧发生了什么动作。
    • 结论：IDM比BC更容易训练，数据效率（Sample Efficiency）高出几个数量级（论文图3证明了这一点）。

2.3 核心组件二：数据清洗与处理

不要小看工程细节。互联网数据是极其嘈杂的（Noisy）。

  • Web_Clean 数据集：从27万小时视频中筛选出7万小时。
  • 分类器：训练了一个基于CLIP特征的SVM分类器，用于剔除带有人脸摄像头、UI遮挡、非生存模式的视频片段。

第三部分：模型架构深度解析

这部分我们需要深入模型内部。请大家注意，VPT涉及两个主要模型：IDM和VPT Foundation Model。

3.1 逆动力学模型 (IDM) 架构

IDM的目标是输入一段视频序列，输出每一帧的动作。

  • 输入：128帧连续图像（128 x 128 x 3）。
  • 第一层：3D 卷积 (Temporal Convolution)
    • 这是一个非常关键的设计。
    • Kernel Size: $(5, 1, 1)$。这意味着它在时间维度上进行卷积。
    • 非因果性：当前时间步 $t$ 的嵌入实际上融合了 $t-2$ 到 $t+2$ 的像素信息。
  • 主干网络 (Backbone)：ResNet 架构。
    • 分为3个阶段（Stages），通道数分别为 64, 128, 128。
    • 每个阶段包含卷积、最大池化和两个残差块（ResNet Blocks）。
    • 输出：将空间维度展平，得到 $128 \times 131072$ 的序列向量。
  • Transformer 头 (Attention Mechanism)：
    • 使用非掩码（Unmasked）的残差Transformer块（Bi-directional Attention）。
    • 这意味着预测 $t$ 时刻动作时，Attention机制可以看到整个128帧序列（过去和未来）。
    • 层数：4层。
  • 输出头 (Action Head)：
    • 预测键盘按键（二分类）和鼠标移动（多分类，离散化）。

3.2 VPT 基础模型 (Foundation Model) 架构

VPT基础模型本质上是一个通过行为克隆训练的策略网络（Policy Network）。

  • 因果性约束 (Causality)：与IDM不同，VPT必须是因果的，因为代理在玩游戏时看不到未来。
  • 差异点：
    • 移除3D卷积：第一层不能使用时间维度的卷积混合未来信息。
    • 掩码 Transformer (Masked Transformer)：使用类似于GPT的Causal Mask，确保 $t$ 时刻只能注意到 $t$ 及之前的时刻。
    • 记忆机制：采用了Transformer-XL风格的记忆（Memory），允许跨Batch关注历史信息。
  • 参数量：论文主要实验了 0.5 Billion (5亿) 参数的模型。虽然比起GPT-3很小，但在RL领域已属巨型。

3.3 动作空间设计 (Action Space)

这是具身智能的一个难点。Minecraft原生接口极其复杂。

  • 混合动作空间：
    • 键盘：二值化（Binary），如W, A, S, D, Space, Inventory等。
    • 鼠标：这是难点。鼠标移动是连续信号 $(\Delta x, \Delta y)$。
  • 离散化处理 (Discretization)：
    • 模型将鼠标移动离散化为11个Bin（桶）。
    • Foveated Binning (中央凹式分桶)：这种设计非常符合人体工程学。对于细微移动（如瞄准），Bin的粒度很细；对于大幅度移动（如转身），Bin的粒度较粗。
    • 公式化描述：输出是一个分类分布（Categorical Distribution）。

第四部分：微调与强化学习

仅靠预训练（Pre-training）是不够的。预训练模型学会了“像人一样行动”，包括漫无目的的游荡。要完成特定任务（如挖钻石），需要微调。

4.1 行为克隆微调 (BC Fine-tuning)

  • 数据集：contractor_house（承包商在10分钟内造房子）。
  • 结果：预训练模型在这个小数据集上微调后，能力大幅提升。
    • 不仅能造房子，还涌现出了制作石质工具的能力（这是预训练模型做不到，但包含在潜空间里的能力）。
  • 结论：VPT作为一个以行为为先验（Behavioral Prior）的基础模型，极大地提升了下游任务的数据效率。

4.2 强化学习微调 (RL Fine-tuning) - 挑战“钻石挖掘”

这是本论文的高光时刻。

  • 任务难度：从空手开始 -> 挖木头 -> 造板 -> 造台 -> 造木镐 -> 挖石 -> 造石镐 -> 挖铁 -> 烧铁 -> 造铁镐 -> 挖钻石 -> 造钻石镐。
    • 任何一步出错都可能导致失败。
    • 这是一个极度稀疏奖励（Sparse Reward）的问题。
  • 算法：Phasic Policy Gradient (PPG)。
    • PPG是PPO的一个变体，通过分离Policy和Value的优化阶段来提高样本效率。
  • 关键技巧：KL 散度约束 (KL Constraint)
    • 公式：$L_{kl} = \rho \text{KL}(\pi_{pt}, \pi_{\theta})$
    • 直觉：在RL微调时，如果让模型自由探索，它会迅速发生灾难性遗忘 (Catastrophic Forgetting)，忘记怎么砍树、怎么走路。
    • 解决方案：增加一个Loss项，惩罚RL策略 $\pi_{\theta}$ 偏离预训练策略 $\pi_{pt}$ 太远。这保证了模型保留了基本的“人类行为先验”，只在需要优化的任务上进行调整。
  • 奖励塑形 (Reward Shaping)：
    • 不仅仅奖励最终的钻石镐，还对中间产物（原木、木板、工作台等）给予奖励。

第五部分：结果分析与总结

5.1 实验结果分析

  • 数据规模定律 (Scaling Laws)：随着数据量从1小时增加到7万小时，VPT的性能呈对数线性增长。这验证了在行为学习领域，Scale is all you need 可能依然成立。
  • IDM vs BC：论文有力地证明了，用少量数据训练IDM去标注海量数据，比直接用少量数据训练BC效果好得多。

5.2 局限性与未来方向 (Discussion)

  1. 动作空间的依赖：VPT强依赖于键鼠操作。如果换到机器人关节控制，IDM还能这么准吗？（机器人状态估计通常比游戏更难）。
  2. 视觉差异：IDM训练数据（无UI）和网络视频（有UI、水印）的Domain Gap是通过分类器硬剔除解决的，有没有更软性的方法？
  3. 幻觉与安全性：基于互联网视频训练，模型是否会学到有害行为（如在游戏中破坏、攻击行为）？

5.3 总结

VPT 证明了 “Semi-Supervised Imitation Learning at Scale” 是可行的。它为解决强化学习中的探索难题提供了一条新路：不要从随机初始化开始探索，从包含了人类智慧的先验（Prior）开始探索。

这对于具身智能（Embodied AI）的启示是巨大的：也许未来的通用机器人，也是通过观看数百万小时的人类操作视频长大的。

思考

  1. 为什么IDM中使用非因果卷积（Non-causal Convolution）对于动作预测至关重要？
  2. 在RL微调阶段，如果没有KL散度约束，你认为Agent的行为会发生什么变化？为什么？
  3. VPT的方法能否直接应用到真实世界的机械臂抓取任务中？如果不能，主要的障碍是什么？
