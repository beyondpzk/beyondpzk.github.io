# GigaWorldPolicy

[paper link](https://arxiv.org/abs/2603.17240)

**《具身智能前沿：世界模型与动作生成体系的融合——GigaWorld-Policy 深度解析》**

---

### 引言与概述

在当前的具身智能（Embodied AI）研究中，如何赋予机器人理解物理世界并做出连续、合理决策的能力，是核心挑战之一。传统的视觉-语言-动作模型（VLA，如 RT-2, $\pi_0$）虽然在跨模态对齐上取得了显著进展，但它们往往面临一个本质上的非对称性问题：**监督信号的稀疏性（Supervision Sparsity）**。机器人的视觉观测和语言指令是高维且语义丰富的，但与其对应的动作标签（如关节角度、末端执行器位姿）却是低维且低发散度的。这种非对称性极易导致模型学习到“捷径（Shortcuts）”，即模型并未真正理解物理规律，而是简单地将复杂的场景映射到几个固定的动作模板上。

为了解决这一问题，近年来学界开始引入**世界模型（World Models）**，特别是基于大规模视频生成的模型。其核心思想是：利用未来视频帧的预测，为动作学习提供极其稠密的时空监督信号。然而，现有的“世界-动作模型”（如基于逆动力学的 Visual CoT 方法，或联合预测方法）在实际部署时遇到了灾难性的**推理延迟**问题和**误差级联**问题。

---

### 第一部分：研究背景与现有范式的根本缺陷

在深入具体模型之前，我们必须厘清当前领域内的两种主流范式及其内在局限。

#### 1.1 视觉-语言-动作（VLA）模型的局限
当前的 VLA 模型（如基于 VLM 改造的模型）主要被优化用于**判别性推理（Discriminative Reasoning）**，而非高保真度的生成。当研究人员试图在这些框架中加入“未来状态监督”（例如预测下一帧图像）时，VLM 的判别性架构很难强制保证预测动作的连续性和物理一致性。

#### 1.2 现有视频生成驱动的世界-动作模型（WAM）的痛点
另一条路线是直接改造 Sora、Wan 等视频生成大模型，将其应用于机器人控制。主要有两种做法：
1.  **双向联合预测（Joint Action-Video Prediction）**：使用双向注意力机制，让动作和视频相互预测（如 Motus）。缺点：在推理时，必须通过极其耗时的扩散采样迭代来生成未来视频，导致极高的推理延迟（数秒级），无法满足机器人闭环控制的需求。
2.  **两阶段逆动力学（Two-Stage IDM / Visual CoT）**：先生成未来视频片段，再由逆动力学模型将视频转化为动作（如 Mimic-Video）。缺点：视频预测的任何微小误差（即幻觉）都会直接向下游传递，导致动作解码彻底失效（误差级联）。

**GigaWorld-Policy 的破局点**：打破“必须先预测视频才能输出动作”的束缚。将未来视觉动态（Future Visual Dynamics）仅仅作为训练阶段的**推理信号和稠密监督源**，而在推理阶段将其完全解耦，实现纯动作的毫秒级生成。

### 第二部分：问题建模与数学表述

我们将机器人操作形式化为一个序列决策问题。
*   在时间步 $t$，机器人接收来自固定摄像机视角集合 $S = \{left, front, right\}$ 的多视角 RGB 观测 $o_t = \{o_t^v\}_{v \in S}$。
*   接收自然语言指令 $l$ 以及本体感觉状态（Proprioceptive state）$s_t$。
*   模型的任务是预测一个长度为 $p$ 的动作块（Action Chunk）：$a_{t:t+p-1} = (a_t, a_{t+1}, \dots, a_{t+p-1})$。

GigaWorld-Policy 采用了一个统一的生成模型 $g_\Theta$，同时参数化两个互补的条件分布：

1.  **动作建模（Action Modeling）**：基于演示数据，模型学习在给定观测、状态和语言指令的条件下，采样未来的动作块：
    $$ (a_{t:t+p-1}, c_t) \sim g_\Theta(\cdot | o_t, s_t, l) $$
    其中 $c_t$ 是动作的隐层条件信号，用于指导视觉预测。
2.  **视觉前向动力学建模（Visual Feedforward Dynamics Modeling）**：在相同的上下文和预测的动作条件信号下，模型学习采样未来的观测序列，捕捉视觉特征的演变：
    $$ (o_{t+\Delta}, o_{t+2\Delta}, \dots, o_{t+K\Delta}) \sim g_\Theta(\cdot | o_t, s_t, l, c_t) $$
    其中 $\Delta$ 是预测观测的时间步长（Stride），$K = \lfloor p/\Delta \rfloor$ 是动作域内的预测帧数。

### 第三部分：GigaWorld-Policy 核心架构深度解析

GigaWorld-Policy 的骨干网络基于一个拥有 50 亿参数的扩散 Transformer (DiT，具体为 Wan 2.2 5B)。本节我们将详细拆解其网络结构。

#### 3.1 输入表征与 Token 化 (Tokenization)

要让同一个 DiT 骨干处理多模态数据，必须进行精心的 Token 化设计。

*   **视觉观测的复合与稀疏采样**：为了在不修改底层视频模型结构的前提下实现多视角一致性，模型将三个视角的图像拼接成一张与标准输入分辨率一致的“复合图像”（Composite Image）：
    $$ o_t^{comp} = Compose(o_t^{left}, o_t^{front}, o_t^{right}) $$
    为减少计算冗余，模型采用稀疏采样策略，仅预测以 $\Delta$ 为间隔的未来帧集合 $\{o_{t+k\Delta}^{comp}\}_{k=1}^K$。
*   **统一的 VAE 编码**：当前复合图像 $o_t^{comp}$ 与未来预测图像均通过预训练的变分自编码器（VAE）进行压缩，随后被打平为时空视觉 Token $T_o$（当前观测）和 $T_f$（未来观测）。
*   **状态与动作的线性映射**：本体状态 $s_t$ 和动作块 $a_{t:t+p-1}$ 直接通过线性投影（Linear Projections）映射到 DiT 的隐藏层维度，得到状态 Token $T_s$ 和动作 Token $T_a$。
*   **语言指令**：通过预训练的文本编码器提取特征 $T_l$，但注意，$T_l$ **不参与**后续序列的拼接，而是作为全局条件注入。

#### 3.2 共享注意力计算与异构位置编码

不同于采用混合专家网络（MoE）来分别处理动作和图像的模型，GigaWorld-Policy 采用了一组**完全共享的 Transformer Blocks**。
*   所有类型的 Token 在每一层共享 Query、Key、Value 的投影矩阵。这种紧密耦合强制模型在同一个表征空间内对齐动作规律与视觉物理约束。
*   **位置编码（Positional Encoding）**的解耦：视觉 Token ($T_o, T_f$) 采用 2D 图像网格位置编码；状态和动作 Token ($T_s, T_a$) 采用 1D 时间轴位置编码，以此保留各模态底层的几何与拓扑结构。

#### 3.3 核心架构创新：基于因果掩码的自注意力机制 (Causal Self-Attention Mask)

这是支撑 GigaWorld-Policy 高效推理的灵魂设计。在每个扩散步，所有模态 Token 被拼接为一个统一序列：$T_t = [T_o ; T_s ; T_a ; T_f]$。语言指令 $l$ 则通过交叉注意力（Cross-Attention）注入。

模型设计了一个**分块因果掩码（Blockwise Causal Mask）**来严格控制信息流，其规则如下（极其关键）：
1.  **$T_s$ 与 $T_o$（历史与当前信息）**：可以双向相互关注，建立对当前环境的完整认知。但不可关注 $T_a$ 和 $T_f$。
2.  **$T_a$（动作）**：可以关注 $\{T_s, T_o\}$，但**绝对不可关注** $T_f$。这一设计确保了动作生成完全不依赖对未来视觉的预测。
3.  **$T_f$（未来视觉）**：可以全面关注 $\{T_s, T_o, T_a\}$。这实现了**前向动力学（Forward Dynamics）**的建模——未来的画面是由当前状态和将要执行的动作共同决定的。

**深度思考**：为什么要这么设计？
如果允许 $T_a$ 看到 $T_f$（即全连接自注意力），模型在训练时就会产生“信息泄露（Information Leakage）”，动作解码会严重依赖未来视频。这会导致在推理阶段，若不生成视频就无法生成准确动作。通过这种非对称的因果截断，模型在训练时利用 $T_f$ 的梯度回传约束 $T_a$ 的表征空间，而在推理时可以安全地将 $T_f$ 模块直接“砍掉”。

### 第四部分：连续流匹配与多阶段训练范式

GigaWorld-Policy 放弃了传统的 DDPM 目标，转而采用最优传输理论中的**流匹配（Flow Matching）**。流匹配在训练高分辨率视频和连续动作时展现出了更好的收敛速度和稳定性。

#### 4.1 目标函数设计
对于任意模态变量 $x$（动作 Token 表达 $a$ 或未来视频 VAE 隐向量 $z_f$），模型采样时间 $s \sim \mathcal{U}(0, 1)$ 和噪声 $\epsilon \sim \mathcal{N}(0, I)$，构建插值噪声变量：
$$ x^{(s)} = (1 - s)\epsilon + sx $$
其对应的目标速度场（Velocity Field）为：$\dot{x}^{(s)} = x - \epsilon$。

模型分别对未来视频和动作块进行流匹配优化：
*   **视频预测损失**：要求模型根据历史信息和**执行的动作**预测未来隐向量的演变：
    $$ \mathcal{L}_{video} = \mathbb{E}_{s, \epsilon} \left[ \left\| g_\Theta(z_f^{(s)}, s | T_s, T_o, T_a, T_l) - \dot{z}_f^{(s)} \right\|_2^2 \right] $$
*   **动作预测损失**：要求模型仅根据历史信息预测动作：
    $$ \mathcal{L}_{action} = \mathbb{E}_{s, \epsilon} \left[ \left\| g_\Theta(a^{(s)}, s | T_s, T_o, T_l) - \dot{a}^{(s)} \right\|_2^2 \right] $$
总体优化目标为两者的加权和：$\mathcal{L}_{all} = \lambda_{video}\mathcal{L}_{video} + \lambda_{action}\mathcal{L}_{action}$。（论文中设置 $\lambda_{action}=5, \lambda_{video}=1$）。

#### 4.2 三阶段课程学习 (Curriculum Pre-training)
由于真实的机器-环境交互数据极度稀缺，GigaWorld-Policy 设计了严密的知识注入路径：
1.  **互联网视频初始化**：基于 Wan 2.2 5B，获取对物理世界常识（如重力、遮挡、物体材质）的基础理解。
2.  **具身数据预训练 (Embodied Data Pre-training)**：注入约 10,000 小时的第一人称人类视频（Ego4D, EgoDex）及多种机器人的操作视频（Open X-Embodiment 等）。此阶段**仅优化 $\mathcal{L}_{video}$**。这使得模型学会理解末端执行器（人手或机械爪）与物体交互时的视觉动态模式，建立起稳健的视角和表征。
3.  **目标机器人后训练 (Post-training)**：在目标机器人的专有轨迹数据上，联合优化 $\mathcal{L}_{video}$ 和 $\mathcal{L}_{action}$，完成控制接口和状态分布的最终适配。

### 第五部分：高效部署与纯动作推理机制

理论的优雅最终必须服务于工程的落地。这一部分我们分析模型在真实机器人上的部署优势。

在推理时（Inference），控制的低延迟是生死攸关的。由于在架构设计时采用了严格的因果掩码，动作 Token $T_a$ 的生成仅依赖上下文 $w_t = (T_l, T_s, T_o)$。

**推理执行流程：**
1. 获取当前观测和指令构建 $w_t$。
2. 初始化动作高斯噪声 $a^{(0)} \sim \mathcal{N}(0, I)$。
3. 利用欧拉法等 ODE 求解器，沿学习到的速度场从 $s=0$ 积分到 $s=1$：
   $$ \frac{da^{(s)}}{ds} = g_\Theta(a^{(s)}, s | w_t) $$
4. 获得降噪后的 $a^{(1)}$，解码为物理控制命令 $\hat{a}_{t:t+p-1}$。
5. **极其重要：整个过程中不实例化任何未来视频 Token ($T_f$)**。

这一设计直接砍掉了处理超长序列视频 Token 的算力开销。论文数据表明，这种 Action-only decoding 模式在 NVIDIA A100 上单次推理仅需 **360 毫秒**，相较于 Motus 的 3231 毫秒，实现了 **9倍加速**，完美跨越了世界模型在机器人高频闭环控制中的门槛。

### 第六部分：实证分析与实验发现

在此部分，我们审视论文的关键实验，以验证上述理论设计的有效性。

#### 6.1 效率与成功率的平衡 (Efficiency-Performance Trade-off)
在 RoboTwin 2.0 仿真平台（50+ 任务，涵盖域随机化）及现实世界（AgileX PiPER 双臂）测试中：
*   面对 VLA 基线（如 $\pi_0$）：GigaWorld-Policy 牺牲了极小的推理时间（360ms vs 225ms），换取了仿真环境下近乎翻倍的成功率（0.86 vs 0.48），并在真机测试中表现出显著更强的鲁棒性。
*   面对世界模型基线（如 Motus）：性能相当甚至略优（真机 0.83 vs 0.76），但速度快了整整一个数量级。高频的控制循环进一步降低了误差积累，这是其真机性能超越 Motus 的重要物理原因。

#### 6.2 数据效率 (Data Efficiency)
实验表明，得益于未来视频的稠密监督和预训练物理先验，GigaWorld-Policy 展现出了惊人的 Sample Efficiency。它仅需 VLA 模型 **10% 的演示数据**，即可达到相同的测试成功率。这对于数据极其昂贵的机器人领域具有巨大的经济价值。

#### 6.3 烧蚀实验揭示的深度见解 (Ablation Insights)
*   **预训练的作用**：如果不做具身数据预训练，直接从通用视频基座微调，真机成功率暴跌至 0.57（相比完整版的 0.83）。这证明了 Ego-video 和多元 Robot-video 构成了通往目标机器人的关键“知识桥梁”。
*   **预测步长 $\Delta$ 的奥秘**：预测多少未来才够用？实验发现 $\Delta=12$ 时成功率最高（0.83）。如果预测太密集（如 $\Delta=4$），反而会加重模型的生成负担，稀释了动作学习的容量；如果不预测未来（$\Delta=0$，退化为纯 VLA），成功率跌至 0.60。这说明**适度的未来推演（Anticipation）为当前决策提供了最优的正则化约束**。
*   **因果掩码 vs 全自注意力**：若使用无掩码的全自注意力，训练出的视频质量甚至不如因果掩码。物理直觉解释：全自注意力导致动作和未来相互“作弊”，而因果掩码强制模型进行**严格的前向因果推理**（动作必须导致结果，而非结果泄露给动作），这反而提升了模型对物体状态变化的捕捉精度（见论文图9的 PSNR/SSIM 提升）。

### 总结与前沿思考

GigaWorld-Policy 重新定义了**世界模型服务于动作策略**的正确姿势。它的核心贡献并非提出了一种全新的生成数学，而是通过**系统级的架构工程（Causal Mask 隔离）与目标函数解耦**，深刻回答了“如何既要世界模型的稠密监督，又不要它的推理延迟”这一痛点。

**思考题：**
1. GigaWorld-Policy 目前的前向动力学是在像素空间（Pixel-level）进行的。在面对更长跨度的规划（Long-horizon planning）时，像素级预测是否会遭遇容量瓶颈？是否有必要引入隐空间的世界模型（Latent World Models, 如 Dreamer 架构）与 DiT 进行结合？
2. 在流匹配的多任务平衡中（$\lambda_{action} = 5, \lambda_{video} = 1$），这种固定的标量权重是否最优？如何设计一种自适应的损失加权机制，使得在训练初期偏重物理规律学习，后期偏重精准操作微调？

## Training samples示例

根据论文的描述，GigaWorld-Policy 的训练样本是**多模态的序列数据**。由于这是一个“世界-动作模型”（World-Action Model），它的训练样本不仅包含当前的状态和要执行的动作，还包含**未来的视觉预测目标**，以便模型学习物理世界的动态变化。

具体来说，在模型进行机器人策略训练（Post-training）时，一个完整的训练样本由以下五个核心部分组成：

*   **当前多视角图像 (Current Observations, $o_t$)**：来自固定摄像机视角（通常为左侧、前方、右侧）的 RGB 图像。为了保持跨视角的一致性，这三个视角的图像会被拼接成一张与标准输入分辨率相同的“复合图像”（Composite Image）。
*   **未来视觉观测 (Future Observations)**：为了提供密集的环境动态监督信号，样本中包含了未来时间步的真实视频帧。为了减少冗余，模型不会逐帧预测，而是以固定的时间步长（$\Delta$）稀疏地抽取未来的复合图像（例如 $o_{t+\Delta}, o_{t+2\Delta}...$）作为学习目标。
*   **本体状态 (Proprioceptive State, $s_t$)**：机器人自身的物理状态数据，例如机械臂当前的关节角度和位置等。
*   **语言指令 (Language Instruction, $l$)**：描述当前任务的自然语言文本，比如“Clean the desk”（清理桌面）或“Scan a QR Code”（扫描二维码）。
*   **动作块 (Action Chunk, $a_{t:t+p-1}$)**：这是专家演示数据中，机器人在接下来 $p$ 个时间步内实际执行的动作序列（控制指令）。

### 数据是如何输入给模型的？
在模型内部，这些样本数据会被统一转化为 Token 序列：
1. **视觉信息**（当前图像和未来图像）会通过一个预训练的 VAE 编码器被转化为时空视觉 Token ($T_o$ 和 $T_f$)。
2. **状态和动作**则通过线性层映射为状态 Token ($T_s$) 和动作 Token ($T_a$)。
3. 这四类 Token 会被直接拼接成一个长序列：`[当前图像; 当前状态; 动作块; 未来图像]`，然后送入具有因果掩码（Causal Mask）的 Transformer 中进行联合训练。语言指令则通过交叉注意力（Cross-Attention）作为外部条件注入。

### 预训练阶段的样本差异
值得注意的是，这篇论文采用了渐进式的训练策略。在**预训练阶段 (Pre-training)**，模型使用了约 10,000 小时的海量视频数据（见论文 Table 1）。此时的训练样本主要是：
* 大规模的互联网人类第一人称视角视频（如 Ego4D, EgoDex）。
* 多种不同形态机器人的真实操作视频（如 Open X-Embodiment, Agibot）。

在预训练阶段，样本的主要形态是长视频片段以及部分附带的动作标签，核心目的是让模型先学会“理解物理规律”和“预测视觉动态”，随后再在目标机器人的专有数据上组装成上述的完整多模态样本进行微调。

但是上面的长序列,并不是[1,2,3,4,1,2,3,4,...] 这种,而就是一个training sample就是一个[1,2,3,4]. 也就是模型reasoning的能力是固定不变的,就是历史的那么多.
不像Dino-World那样其实可以reasoning多桢.

## 为什么不把未来的预测放在action的前面,类似Visual CoT的思路

`[当前图像 -> 当前状态 -> 未来图像 -> 动作块]` 这种“先预测未来画面，再提取具体动作”的思路，正是所谓的**“视觉思维链 (Visual CoT)”**。

在机器人控制领域，这种思路被称为**逆动力学模型 (Inverse Dynamics Model, IDM)**。论文中的对比方法（如图 2(c) 所示的基于视频生成的两阶段策略，如 UniPi 等）就是完全按照你说的这种 Visual CoT 逻辑工作的。

但 GigaWorld-Policy 刻意**没有**采用这种顺序，而是坚持了 `[当前图像 -> 当前状态 -> 动作块 -> 未来图像]` 的顺序。你的理解完全正确：它把未来图像放在最后，本质上就是为了**增加监督密度**和**起到正则化/物理解释的作用**。

论文之所以放弃 Visual CoT 顺序，主要是为了解决实际机器人部署中的两个致命痛点：

### 1. 推理延迟极高（核心痛点）
这是本文最核心的出发点。生成视频 Token 的计算代价极大，且非常耗时。
* **如果是 Visual CoT 顺序**：机器人在每走一步之前，都**必须**先把未来的视频画面完整生成出来，然后才能根据画面解码出动作。这会导致极高的控制延迟（可能长达数秒），在现实世界中机器人根本无法做到闭环的实时反应。
* **GigaWorld-Policy 的顺序**：因为动作 Token 排在未来图像 Token 前面（在因果掩码 Causal Mask 下，动作不依赖未来图像，见图 4），**在实际推理（Inference）时，模型可以直接“砍掉”未来图像的预测分支**。模型只生成动作 Token 就直接输出给机器人执行。这让它的推理速度提升了 9 倍（单次推理只需 0.36 秒）。

### 2. 视频生成幻觉导致的误差级联 (Error Propagation)
由于当前的视频生成模型并不完美，经常会产生违背物理规律的“幻觉”或细节模糊。
* **如果是 Visual CoT 顺序**：视频预测的任何微小误差（比如把杯子的把手生成歪了），都会向下游传递，直接导致动作解码出错，这种误差在长序列操作中会迅速累积。
* **GigaWorld-Policy 的顺序**：模型以动作作为条件去预测未来（这被称为**前向动力学 Forward Dynamics**），即使未来图像预测得有些瑕疵，也不会反向污染当前正在生成的动作。

### 总结
你对模型现状的理解非常精准。正如论文在第一节（Introduction）中所说，GigaWorld-Policy 是一个 **Action-centered（以动作为中心）** 的模型。
它把未来视频放在后面，仅仅是在**训练阶段**充当一个“陪练”。它强迫模型在学动作的同时，必须在隐空间里推演“这个动作会带来什么视觉后果”。这提供了一个极其丰富的**稠密监督信号（Dense Supervision）**，强迫模型学到符合物理直觉的动作分布；而一旦训练完成上了赛场（推理阶段），这个繁重的“视频预测”包袱就可以被直接卸下。
