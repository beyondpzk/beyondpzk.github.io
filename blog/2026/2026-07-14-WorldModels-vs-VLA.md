# 世界模型与VLA对比

过去一年，机器人学习与自动驾驶领域最火热的两条路线——**世界模型（World Models）** 与 **视觉-语言-动作模型（VLA, Vision-Language-Action）**——正在从“各说各话”走向“正面交锋”。二者都试图从原始感知输入直接产生动作，但哲学、架构与工程取舍截然不同。本文从目标、结构、训练、推理与落地五个维度系统对比，试图回答一个从业者常问的问题：**什么时候该用世界模型，什么时候该用 VLA？**

---

## 一、核心目标：预测未来 vs. 直接映射

### 1.1 世界模型：先学会物理，再做事

世界模型的核心假设是：**如果智能体能够准确预测环境在动作作用下的演化，就可以在该预测模型内部进行规划、推理与反事实想象**。典型代表包括 Dreamer、JEPA-WM、DriveWorld、GAIA-1 等。

其目标函数通常写作：

$$\hat{s}_{t+1} = f(s_t, a_t; \theta)$$

或者更现代的潜空间版本：

$$\hat{z}_{t+1} = P_\theta(E_\phi(o_t), a_t)$$

世界模型本身**不直接输出动作**，它输出的是“下一个状态”。动作由规划器（MPC、CEM、梯度优化）在线求解得到。因此，世界模型更像一个“内心的物理模拟器”。

### 1.2 VLA：把感知、语言与动作统一为 next-token

VLA 的核心假设是：**大规模预训练获得的视觉-语言对齐能力，可以直接迁移到动作生成**。典型代表包括 RT-2、OpenVLA、π0、AutoVLA、NaVILA 等。

其目标函数是自回归的：

$$a^* = \arg\max_a P(a \mid o, I; \theta)$$

其中 $I$ 可以是语言指令（如“拿起红色杯子”）或高层次目标。VLA 模型**直接输出动作**（或离散化的动作 token），不依赖显式的环境动力学模型。

### 1.3 一句话差异

| 维度 | 世界模型 | VLA |
|------|---------|-----|
| 核心能力 | **预测未来状态** | **直接生成动作** |
| 中间产物 | 状态/潜变量 $s_{t+1}$ 或 $z_{t+1}$ | 动作 $a_t$ 或动作 token |
| 规划方式 | 在模型内部做 MPC/CEM/GD | 模型一次性自回归输出 |

## 二、架构对比：显式动力学 vs. 隐式压缩

### 2.1 世界模型的典型架构

```
观测 o_t ──► 编码器 E ──► 潜状态 z_t ──┐
                                      ▼
动作 a_t ───────────────────────► 预测器 P ──► z_{t+1}
                                      │
                                      ▼
                                  规划器/奖励函数
                                      │
                                      ▼
                                    动作序列
```

关键模块：
- **编码器**：从像素或点云压缩到潜空间（常用 DINOv2、V-JEPA、VQ-VAE）。
- **预测器/动力学模型**：学习 $P(z_{t+1} \mid z_t, a_t)$。
- **规划器**：利用学到的模型进行动作优化，常用 CEM、MPPI、梯度下降。
- **奖励/目标函数**：可以是目标图像、语言目标或学习的奖励模型。

### 2.2 VLA 的典型架构

```
图像/视频 o ──► 视觉编码器 ──┐
                            ▼
语言指令 I ──► 文本编码器 ──► 多模态 LLM ──► 动作头/动作 token
                            ▲
本体感觉/历史动作 s ────────┘
```

关键模块：
- **视觉-语言 backbone**：Qwen2.5-VL、LLaVA、PaLI-X 等。
- **动作表示**：连续 MLP 回归、离散动作 token、扩散动作头（π0）。
- **训练目标**：行为克隆（BC）、DPO、GRPO、AWR 等。

### 2.3 架构哲学差异

世界模型把“环境动力学”显式地分离出来，因此：
- 可以**离线想象**：在潜空间里 roll out 未来，用于数据增强与风险评估。
- 可以**更换任务目标**：同一个动力学模型，配合不同奖励函数做不同任务。
- 但多了**误差累积**风险：长程 rollout 会漂移。

VLA 把动力学隐式压缩在 LLM 的参数里，因此：
- **端到端简洁**：一个模型吃一切、出一切。
- 可以**利用互联网预训练知识**：语言、常识、视觉概念直接迁移。
- 但**可解释性差**，面对分布外场景容易“硬猜”。

## 三、训练方式：自监督预测 vs. 模仿/对齐学习

### 3.1 世界模型的训练

世界模型通常依赖**自监督或弱监督**学习目标：

1. **像素级重构**：如 World Models、PlaNet 使用 VAE 重构未来帧。
2. **潜空间预测**：如 JEPA 系列避免像素重构，直接在表征空间做预测。
3. **视频生成**：如 Sora、GAIA-1、DriveWorld 用扩散/Transformer 生成未来视频。

损失函数示例：

$$\mathcal{L}_{WM} = \mathbb{E}\left[ \| z_{t+1} - P_\theta(E_\phi(o_t), a_t) \|^2 \right]$$

优点：不需要大量动作标注，视频数据即可训练。
缺点：预测准确 ≠ 任务有用；容易学到与任务无关的细节。

### 3.2 VLA 的训练

VLA 训练通常是**监督微调或强化微调**：

1. **行为克隆（BC）**：从人类/专家演示数据学习 $P(a \mid o, I)$。
2. **离线 RL**：如 AWR、IQL、Diffusion Policy 改善分布外泛化。
3. **在线 RL/GRPO**：如 AutoVLA、π0 使用 GRPO 优化驾驶或机器人指标。
4. **语言-动作对齐**：通过 VLM 预训练 + 机器人数据微调。

损失函数示例：

$$\mathcal{L}_{VLA} = -\mathbb{E}\left[ \log P(a^* \mid o, I; \theta) \right]$$

优点：目标直接对准任务性能。
缺点：对高质量动作数据依赖大；泛化受限于演示分布。

### 3.3 数据效率对比

| 指标 | 世界模型 | VLA |
|------|---------|-----|
| 所需动作标注 | 少（视频即可） | 多（需要成对 $(o, I, a)$） |
| 所需数据量 | 中等偏大 | 大（尤其是泛化） |
| 数据来源 | 视频、仿真 rollout | 专家演示、人类遥操作 |
| 预训练优势 | 自监督视频预测 | 互联网 VLM 预训练 |

## 四、推理方式：在线规划 vs. 单次生成

### 4.1 世界模型推理： planning in the head

世界模型在推理时通常执行 MPC 循环：

```python
for t in range(T):
    # 1. 观测并编码当前状态
    z_t = encoder(o_t)
    # 2. 采样/优化未来动作序列
    a_seq = planner.optimize(z_t, goal, predictor)
    # 3. 只执行第一个动作
    execute(a_seq[0])
    # 4. 下一时刻重新观测并规划
```

优势：
- 可以**重规划**，对模型误差有一定容错。
- 可以**评估多步风险**，适合安全关键任务（自动驾驶、机械臂避障）。
- 可以**零样本迁移**到新目标，只要奖励函数改变。

劣势：
- **推理耗时**：CEM 需要多次前向传播。
- **长程漂移**：模型 rollout 越长，误差越大。
- **奖励设计依赖**：没有现成奖励函数时难以使用。

### 4.2 VLA 推理： one-shot generation

VLA 推理通常是单次或少量前向传播：

```python
# 一次前向得到动作
action_tokens = vla.generate(image, instruction)
actions = action_tokenizer.decode(action_tokens)
execute(actions)
```

优势：
- **速度快**，尤其是 Fast-Thinking 模式（如 AutoVLA 的直觉模式）。
- **工程简单**，部署友好。
- 可以**自然语言交互**，指令灵活。

劣势：
- 缺乏**显式未来推演**，长程安全难以保证。
- 动作一旦生成，无法像 MPC 那样重新优化。
- 需要精心设计思维链（CoT）才能做复杂推理。

### 4.3 时延与可靠性权衡

| 场景 | 推荐路线 | 理由 |
|------|---------|------|
| 高速自动驾驶 | 世界模型 + MPC | 需要多步安全推演 |
| 家庭服务机器人 | VLA | 指令灵活，延迟要求相对宽松 |
| 工业抓取（固定任务） | VLA / Diffusion Policy | 数据充分，单次生成即可 |
| 开放环境探索 | 世界模型 | 可想象未来，评估风险 |
| 需要人类语言指令 | VLA | 原生支持自然语言 |

## 五、当前融合趋势：取长补短

### 5.1 世界模型 + VLA 的几种融合形态

**形态一：VLA 作为动作生成器，世界模型作为验证器**
- VLA 快速生成候选动作。
- 世界模型 rollout 未来，筛选安全动作。
- 代表：部分自动驾驶系统中的“双系统”架构。

**形态二：世界模型作为数据增强器**
- 用世界模型生成大量合成 rollout。
- 用这些 rollout 训练或微调 VLA。
- 代表：UniWorld、DriveWorldVLA 中的数据引擎。

**形态三：统一潜空间 + 语言条件**
- 训练一个共享潜空间，既支持预测又支持语言条件生成。
- 代表：DriveWorldVLA、RAENWM、JEPA 与 VLA 的结合尝试。

**形态四：VLA 做 System 2，世界模型做 System 1**
- 简单场景直接 VLA 快思考。
- 复杂/危险场景触发世界模型慢思考规划。
- 代表：AutoVLA 的双思考模式可与此类比。

### 5.2 未来可能走向

个人认为，两条路线会长期共存，并逐渐收敛到一种**“可规划的 VLA”**或**“可语言交互的世界模型”**：

- VLA 需要补上**未来推演能力**，否则难以进入高安全领域。
- 世界模型需要补上**语言理解与互联网知识**，否则难以处理开放语义任务。

## 六、总结

| 维度 | 世界模型 | VLA |
|------|---------|-----|
| 核心思想 | 学习环境动力学，内部规划 | 学习感知-语言-动作直接映射 |
| 是否显式建模未来 | 是 | 否（隐式） |
| 动作来源 | 规划器优化 | 模型直接生成 |
| 数据需求 | 视频/状态转移数据 | 专家动作数据 |
| 推理延迟 | 通常较高 | 可做到很低 |
| 可解释性 | 较好（可 rollout 未来） | 较差（黑盒生成） |
| 语言交互 | 需额外引入 | 原生支持 |
| 安全关键场景 | 更适合 | 需要额外机制 |
| 开放语义任务 | 较弱 | 更强 |

**选择建议**：

- 如果你的任务**安全关键、需要多步推演、奖励/目标明确**，优先世界模型 + MPC。
- 如果你的任务**需要自然语言交互、数据充分、追求部署简单**，优先 VLA。
- 如果目标是**通用机器人或 L4+ 自动驾驶**，二者融合几乎不可避免。
