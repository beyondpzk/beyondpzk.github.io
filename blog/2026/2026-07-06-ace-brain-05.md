## 摘要

ACE-Brain-0.5 是大晓机器人推出的新一代**统一具身基模型（Unified Embodied Foundation Model）**，目标是将具身智能从「单点动作生成」推向 **Physical Agentic AI**——即在真实物理世界中围绕目标自主感知、规划、执行、评估并持续进化的智能体。

与当前主流的 VLA（Vision-Language-Action）模型或多模型 Agent 编排方案不同，ACE-Brain-0.5 在**单一 8B 参数主干**下同时整合了：
1. **空间感知（Spatial Perception）**：3D 场景理解、视觉 grounding、可供性识别；
2. **决策规划（Decision Making）**：长程任务分解、子目标规划、空间推理；
3. **具身交互（Embodied Interaction）**：导航与操作动作生成；
4. **自我评估（Self Monitoring）**：任务进度估计与执行质量判断；
5. **自我进化（Self Improvement）**：基于 rollout 反馈持续优化行为。

其核心训练创新是 **SSR+**（Scaffold-Specialize-Reconcile-Reactivate）：在空间智能 scaffold 的基础上，分别训练 QA、Grounding、导航、操作、进度评估等专家能力，通过任务向量合并调和进同一参数空间，再用轻量混合 SFT 重新激活任务接口，从而避免直接混合训练导致的接口冲突和灾难性遗忘。

在 18 个空间感知基准、VLN-CE 导航基准、LIBERO / SimplerEnv-Bridge 操作基准以及 Robometer 进度评估基准上，ACE-Brain-0.5 均取得了领先表现，部分指标超过 GPT-5.4、Gemini-2.5-Pro、GR00T N1.6、π₀/π₀.₅、OpenVLA、NaVid、Uni-NaVid 等主流模型。

## 一、背景：为什么需要统一具身基模型？

### 1.1 具身智能的三阶段演化

回顾具身智能的发展，大致经历了三个阶段：

- **第一阶段：模块化流水线**  
  感知、规划、执行由独立模块负责，可解释性强，但模块间接口冗余、误差累积，难以适应开放环境。

- **第二阶段：端到端 VLA**  
  直接从视觉-语言输入生成动作（如 RT-2、OpenVLA、π₀），显著提升了操作能力，但往往偏重「动作执行」，在空间推理、长程规划和自我纠错方面存在短板。

- **第三阶段：多模型 Agent 编排**  
  用大语言模型调度多个专家模型或工具链（VLM、VLA、planner、critic），能完成复杂流程，但本质仍是多模型协作，缺乏统一的认知表征，导致系统迟滞、接口割裂。

ACE-Brain-0.5 试图进入**第四阶段**：把感知、规划、执行、评估、进化统一到一个基模型中，形成真正的闭环认知架构。

### 1.2 Physical Agentic AI 的核心诉求

真实世界任务很少是「一步动作」就能完成的。以「机器人去酒店洗衣房洗衣服」为例，一个完整的认知闭环包括：

1. **定位**：找到洗衣房、洗衣机、舱门、按钮、洗涤剂；
2. **理解**：明白「放入衣物 → 加洗涤剂 → 选择程序」的执行顺序；
3. **执行**：开门、投放、按键等连续操作；
4. **评估**：判断舱门是否关好、衣物是否卡住、程序是否启动；
5. **恢复**：如果失败，诊断问题并重新规划。

传统系统需要多个专用模块拼接完成这条链路，链路越长，不稳定性越突出。ACE-Brain-0.5 的目标是让这五个环节都在同一个大脑里完成。

### 1.3 从 ACE-Brain-0 到 ACE-Brain-0.5

ACE-Brain-0（2026-03，arXiv:2603.03198）已经证明「空间智能」可以作为跨本体（自动驾驶、无人机、机器人）的共享 scaffold。它提出了 **SSR（Scaffold-Specialize-Reconcile）** 训练范式：
- 先训练一个 **Spatial Expert** 作为空间认知 scaffold；
- 再分别训练自动驾驶（AD）、无人机（UAV）、机器人（Embodied）领域专家；
- 最后通过**数据无关的模型合并**（data-free model merging）把专家能力调和到同一模型中。

ACE-Brain-0.5 在此基础上进一步提出 **SSR+**，新增 **Reactivate** 阶段，把能力从「理解」扩展到「规划、行动、评估、进化」，并在单一模型中同时支持 grounding、导航、操作、进度估计等异质接口。

## 二、模型架构：慢脑 + 快脑的双时间尺度设计

ACE-Brain-0.5 采用一个共享的 8B 参数多模态大语言模型作为「认知主干」，并围绕它构建了两条时间尺度不同的通路：

```text
用户指令 + 历史信息
        ↓
┌─────────────────────────────────────┐
│         慢脑（Slow Brain）           │
│  视觉编码器 + 8B MLLM               │
│  负责：空间感知、任务分解、状态理解、   │
│        高层规划、文本/框/点/轨迹输出   │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│         快脑（Fast Brain）           │
│  快速视觉编码器 + 动作专家             │
│  负责：实时感知、低延迟动作生成、       │
│        导航/操作连续控制               │
└─────────────────────────────────────┘
                ↓
           机器人执行
```

### 2.1 慢脑：统一场景-任务表征

慢脑是 ACE-Brain-0.5 的认知核心，主要组件包括：

- **多模态输入**：单张图像、多视角图像、自我中心视频（egocentric video）、文本指令、历史观测与动作；
- **视觉编码器 + MLP Projector**：将视觉输入编码为视觉 token，对齐到语言模型输入空间；
- **8B 参数 MLLM Decoder**：基于 Qwen3-VL 等开源多模态骨干，统一处理视觉-语言-动作上下文，生成高层推理和结构化输出。

慢脑的输出不是单一动作，而是多种**异质接口**：

| 输出类型 | 形式 | 作用 |
|---|---|---|
| 文本推理 | 自然语言 | 任务理解、规划说明、失败诊断 |
| Grounding | 边界框 / 点 | 目标定位、可操作区域识别 |
| Affordance | 区域/掩码 | 判断物体哪里可以抓、哪里可以按 |
| 子目标规划 | 文本 + 结构化标记 | 将长程任务分解为可执行步骤 |
| 轨迹/航点 | 连续坐标序列 | 导航路径或操作轨迹 |
| 进度评估 | 标量/文本 | 判断当前执行到哪个阶段、是否成功 |

### 2.2 快脑：实时动作生成

机器人控制不能每动一下都重新「深思熟虑」。快脑提供低延迟的视觉-动作回路：

- **快速视觉编码器**：专门处理最新多视角观测，提取实时视觉特征；
- **动作专家（Action Expert）**：接收视觉特征、当前动作噪声/ proprioception、慢脑输出的高层计划，直接生成导航或操作动作；
- **连续输出**：支持机器人 base 的速度/角速度、机械臂的关节位置/末端位姿、夹爪开合等连续控制信号。

快慢脑的协同使得 ACE-Brain-0.5 既能做复杂的空间推理和任务规划，又能满足机器人控制的实时性要求。

### 2.3 统一表征 vs. 专用头

ACE-Brain-0.5 的关键设计选择是**共享主干 + 专用解码接口**：
- 所有任务共享同一个场景-任务隐状态；
- 不同任务通过各自的输出格式（文本、坐标、框、动作 token）进行解码；
- 这种设计让模型在不同能力之间形成统一认知表征，而不是像多模型 Agent 那样在模块之间传递信息。

## 三、训练策略：SSR+

把空间问答、grounding、导航、操作、进度评估这五种能力塞进同一个 8B 模型，最大的挑战是**接口冲突**：
- 空间问答需要文本生成；
- grounding 需要输出点/框；
- 导航需要序列动作预测；
- 操作需要连续动作片段；
- 进度评估需要对轨迹做时间维度判断。

如果直接混合训练，很容易出现「保留了任务知识，却输出了错误格式」或者「强化了 A 任务却干扰了 B 任务」的问题。

ACE-Brain-0.5 的解决方案是 **SSR+**：

```
Scaffold → Specialize → Reconcile → Reactivate
  脚手架      专家化       调和        再激活
```

### 3.1 Stage 1：Scaffold（空间脚手架）

继承 ACE-Brain-0 的思路，首先在大量空间认知数据上训练一个 **Spatial Expert**。这一步的目标是建立：
- 3D 场景理解能力；
- 跨视角空间关系推理；
- 视觉-语言对齐的 grounding 基础；
- 对自动驾驶、无人机、机器人共享的几何/语义先验。

Scaffold 阶段使用的是通用空间智能数据，不针对具体机器人体态，因此学到的表征具有跨本体迁移性。

### 3.2 Stage 2：Specialize（专家化）

在 scaffold 的基础上，分别训练多个面向不同能力的专门化 checkpoint：

| 专家 | 能力 | 输出接口 |
|---|---|---|
| QA Expert | 空间问答、任务规划、推理 | 文本 |
| Grounding Expert | 目标定位、可供性识别 | 点/边界框/掩码 |
| Navigation Expert | 视觉语言导航 | 航点/动作序列 |
| Manipulation Expert | 机器人操作 | 连续动作片段 |
| Progress Expert | 任务进度评估 | 标量/相对排序/文本 |

每个专家在自己的数据分布上充分优化，因此能够获得较高的单任务能力。

### 3.3 Stage 3：Reconcile（调和）

这是 SSR 的核心：把多个专家 checkpoint **合并到同一参数空间**。具体做法是：

- **任务向量（Task Vector）**：对每个专家，计算其参数与 scaffold 参数之间的差值，得到该任务对应的「任务向量」；
- **加权合并**：按照任务重要性和参数冲突程度，对多个任务向量进行加权求和，再叠加回 scaffold；
- **数据无关（Data-Free）**：合并过程不需要重新访问训练数据，大大降低了计算成本；
- **冲突消解**：通过任务向量选择/裁剪（如 TIES-Merging、WUDI 等模型合并技术）减少不同任务之间的参数冲突。

合并后的模型已经同时具备多种能力，但各任务的输出接口可能还不一致。

### 3.4 Stage 4：Reactivate（再激活）

SSR+ 在原有 SSR 基础上新增的第四阶段，目标是：
- **重新激活任务能力**：合并后的模型可能因为参数平均而损失了部分单任务性能，需要用轻量 SFT 恢复；
- **校准输出接口**：让模型学会在不同任务 prompt 下输出正确的格式（文本、框、动作、进度）；
- **减少任务间冲突**：通过混合 SFT 让模型在统一参数空间内稳定切换不同任务；
- **恢复任务切换能力**：合并模型需要明确知道当前要解决什么任务，并调用对应的输出接口。

Reactivate 阶段的训练数据量远小于 Specialize 阶段，起到「精修」和「接口对齐」的作用。

### 3.5 训练数据与 RL 后训练

ACE-Brain-0.5 的训练数据覆盖：
- **空间认知数据**：3D 场景问答、视觉 grounding、可供性、空间推理；
- **导航数据**：VLN-CE（R2R/RxR）等视觉语言导航轨迹；
- **操作数据**：LIBERO、SimplerEnv、RoboVQA 等机器人操作演示；
- **进度评估数据**：机器人轨迹与人工标注的进度/偏好标签；
- **通用 VLM 数据**：保持基础视觉-语言能力，防止灾难性遗忘。

在 SSR+ 之后，模型还可以进一步通过 **GRPO（Group Relative Policy Optimization）** 等强化学习方法进行后训练，利用 rollout 反馈提升长程任务成功率和自我纠错能力。

## 五、与相关工作的对比

| 路线 | 代表模型 | 优点 | 局限 |
|---|---|---|---|
| 端到端 VLA | OpenVLA、π₀、GR00T | 动作生成能力强 | 缺乏长程规划、自我评估 |
| 多模型 Agent 编排 | LLM + VLM + VLA + Critic | 能完成复杂流程 | 接口割裂、系统迟滞、误差累积 |
| 空间智能基座 | ACE-Brain-0 | 跨本体通用表征 | 偏重理解，缺少闭环执行 |
| **统一具身基模型** | **ACE-Brain-0.5** | 感知-规划-执行-评估统一 | 仍需在真实世界大规模验证 |

ACE-Brain-0.5 的定位不是替代 VLA，而是把 VLA 纳入一个更大的闭环认知框架：VLA 负责「快脑」中的动作生成，而统一基模型负责整体任务理解和监控。

## 六、关键洞察

1. **空间智能是 Physical Agentic AI 的通用 scaffold**  
   自动驾驶、无人机、机器人虽然形态不同，但都需要对 3D 物理空间进行建模。ACE-Brain-0.5 继承 ACE-Brain-0 的 insight，把空间智能作为统一底座。

2. **统一模型需要专门的能力调和机制**  
   直接混合训练异质接口容易导致冲突。SSR+ 通过「先专家化、再合并、再激活」的方式，让多种能力在同一参数空间中稳定共存。

3. **慢脑 + 快脑是机器人落地的必要架构**  
   高层推理需要语义能力和大上下文，底层控制需要低延迟。ACE-Brain-0.5 的双时间尺度设计兼顾了两者的需求。

4. **自我评估是闭环智能的关键**  
   机器人不仅要能执行动作，还要能判断自己是否接近目标、是否失败、是否需要重规划。进度评估能力让 ACE-Brain-0.5 具备了基本的「自我检查」能力。

5. **开源生态将加速 Physical Agentic AI 演进**  
   ACE-Brain-0.5 同步开源了技术报告、代码、模型权重和项目主页，为全球开发者提供了一个可验证、可扩展的统一具身基模型起点。

## 七、局限与未来方向

### 7.1 局限

1. **真实世界验证仍有限**：目前主要结果集中在仿真和公开基准上，真实机器人长程任务的量化数据还比较有限。
2. **模型规模与延迟的 trade-off**：8B 模型在边缘设备上部署仍有挑战，未来需要蒸馏、量化、投机解码等优化。
3. **动作输出的精细度**：虽然 LIBERO 和 SimplerEnv 表现优异，但面对高动态、接触丰富的真实操作任务，连续动作精度仍需提升。
4. **Reconcile 阶段的泛化性**：任务向量合并的权重和策略可能需要针对新任务重新调优，自动化程度有待提高。
5. **自我进化的闭环尚未充分展开**：技术报告强调了 Self Improvement 的愿景，但基于 rollout 反馈的持续学习实验细节披露较少。

### 7.2 未来方向

1. **Scaling**：验证更大参数规模（如 30B+）的统一具身基模型是否能带来进一步的认知能力跃升。
2. **真实世界数据闭环**：建立从真实机器人部署中收集反馈、自动筛选数据、增量训练的飞轮。
3. **多本体迁移**：把 ACE-Brain-0.5 的能力迁移到更多人形机器人、轮式机器人、无人机、自动驾驶车辆上。
4. **世界模型结合**：将 ACE-Brain-0.5 与世界模型（World Model）结合，实现更长程的规划、想象与反事实推理。
5. **安全与对齐**：在 Physical Agentic AI 中引入人类反馈强化学习（RLHF）和安全约束，确保机器人在真实世界中的行为可控、可解释。

---

## 八、总结

ACE-Brain-0.5 是大晓机器人在统一具身智能方向上的重要一步。它不再把机器人基础模型仅仅看作「动作策略」或「视觉理解器」，而是将其重新定义为**面向 Physical Agentic AI 的闭环认知大脑**：在一个 8B 参数主干中同时承载空间感知、决策规划、具身交互、自我评估和自我进化五大能力。

其技术核心——**SSR+ 训练范式**和**慢脑/快脑双时间尺度架构**——为如何解决「多种异质机器人能力在统一模型中冲突」的问题提供了一条可行路径。而其在空间推理、导航、操作、进度评估等多维度基准上的 SOTA 表现，也初步验证了这条路径的有效性。

当然，从仿真 SOTA 到真实世界的大规模自主执行，还有很长的路要走。但 ACE-Brain-0.5 的开源，无疑为整个行业提供了一个值得参考的统一建模框架，推动具身智能从「单点能力竞争」走向「系统级智能」。
