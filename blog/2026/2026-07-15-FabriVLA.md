# FabriVLA：面向精密多任务操作的轻量级 VLA 模型

> **论文**：*FabriVLA: A Lightweight Vision-Language-Action Model for Precise Multi-Task Manipulation*  
> **作者**：Shiyuan Yang, Borong Zhang, Jizheng Zhang, Zhijia Tao, Junfei Guo, Donglai Ran, Xu Bian, Qingbiao Li  
> **发布时间**：2026-07-09 (v1), 2026-07-10 (v2)  
> **arXiv**：https://arxiv.org/abs/2607.08575

---

## 摘要

FabriVLA 是一个面向精密多任务机器人操作的轻量级视觉-语言-动作（VLA）模型。它基于 **InternVL3.5-1B** 视觉-语言骨干，配合带有**门控自注意力（Gated Self-Attention）**的 **Flow Matching** 动作头，并引入**浅层 VLM 特征融合（Shallow VLM Layer Fusion）**来增强空间上下文。模型通过单阶段联合优化从预训练 VLM 和随机初始化的动作头端到端训练。

在 **Meta-World MT50** 基准（50 个多样化操作任务）上，FabriVLA 取得 **tier-average 成功率 90.0%**，**overall episode-level 成功率 92.0%**，证明了一个基于 1B 规模 VLM 的紧凑模型可以在不依赖数十亿参数 VLA backbone 的情况下达到强劲性能。

## 一、核心指标与贡献

### 1.1 SOTA 亮点

| 指标 | FabriVLA | 对比最佳基线 |
|---|---|---|
| 参数量 | **0.89B** | LA4VLA（更大） |
| Meta-World MT50 Tier Avg. | **90.0%** | LA4VLA 87.5% |
| Meta-World MT50 Overall Episode | **92.0%** | LA4VLA 约 89% |
| Easy Tier | **95.0%** | 最佳 |
| Medium Tier | **88.2%** | LA4VLA 更高 |
| Hard Tier | **86.7%** | LA4VLA 更高 |
| Very Hard Tier | **90.0%** | LA4VLA 更高 |
| 机器人数据预训练 | **无** | 部分基线有 |

### 1.2 三大核心贡献

1. **轻量级 VLA 架构**：基于 1B 规模 InternVL3.5-1B，总参数量仅 0.89B，远低于主流大型 VLA；
2. **门控自注意力动作头**：在 Flow Matching 动作头中引入可学习门控的自注意力，让动作 token 之间建立时序依赖，同时保持平滑的优化路径；
3. **浅层 VLM 特征融合**：将 VLM 深层语义特征与浅层空间细节特征融合，提升精密操作所需的物体定位与接触式操作能力。

## 二、研究背景与问题定义

### 2.1 为什么需要轻量级 VLA？

现有大规模 VLA（如 RT-2、π0、GR00T N2 等）通常依赖数十亿参数的视觉-语言 backbone，虽然性能强劲，但存在两个现实瓶颈：

- **推理延迟高**：真实机器人控制需要实时闭环，大模型难以满足；
- **部署成本高**：需要高端 GPU 或服务器，限制了在边缘设备和真实机器人上的应用。

这催生了轻量级 VLA 的研究方向：在保持操作精度的同时，大幅降低模型规模和计算开销。

### 2.2 关键洞察

FabriVLA 的核心论点是：

> **精密操作需要的不只是高层语义理解，还需要低层空间细节。** 因此，动作头不仅要消费 VLM 的最终语义输出，还要直接访问 VLM 的浅层特征，以获得物体边界、相对位置等空间信息。

同时，操作轨迹具有强时序结构，因此动作头中的动作 token 需要能够相互关注（self-attention），而不是彼此独立。

## 三、模型架构详解

### 3.1 整体结构

```
RGB 观测 (448×448) + 任务指令
            ↓
    InternVL3.5-1B VLM Backbone (14 layers)
            ├─ Deep features (Layer 14)
            └─ Shallow features (Layer 6)
                    ↓
            [Shallow Layer Fusion]
                    ↓
            机器人状态编码 (state token)
                    ↓
    Flow Matching Action Head (8 层 Gated Transformer Blocks)
                    ↓
        50 步 × 24 维动作序列
```

### 3.2 Vision-Language Backbone

- **骨干**：InternVL3.5-1B，保留 **14 层 Transformer**；
- **输入**：图像 resize 到 448×448，任务指令文本化；
- **输出**：视觉 token + 语言 token 的联合上下文表示。

### 3.3 状态与动作编码

- **机器人状态**：`s ∈ R^24`，包含末端执行器位姿、夹爪状态、物体位置等，通过两层 MLP（ReLU）编码为 `z_s ∈ R^1024`；
- **动作序列**：50 步预测 horizon，每步 24 维动作，通过共享 MLP（24→1024→1024→1024）+ horizon 位置编码，生成动作 token `Z_a ∈ R^{50×1024}`。

### 3.4 门控自注意力 Transformer Block

动作头的核心是 8 层 **FabriVLATransformerBlock**。每个 block 包含三个步骤：

#### 1. Gated Self-Attention

```
A' = A + g · SelfAttn(LayerNorm(A))
```

- `g` 是可学习门控参数，初始化为 0；
- 训练初期：g=0，自注意力被抑制，block 退化为纯 cross-attention；
- 训练后期：g 逐渐打开，动作 token 之间建立时序依赖。

这种设计让模型可以平滑地从"独立预测每一步动作"过渡到"联合规划整段轨迹"。

#### 2. Cross-Attention to Context

```
A'' = A' + CrossAttn(LayerNorm(A'), C, C)
```

动作 token 与 VLM 上下文 token 做 cross-attention，把视觉-语言信息注入动作预测。

#### 3. Feed-Forward with Time Conditioning

```
A''' = A'' + FFN(LayerNorm(A'') + TimeEmb(t))
```

- FFN：1024 → 4096 → 1024，GELU 激活；
- TimeEmb：flow matching 时间步的正弦位置编码。

### 3.5 浅层 VLM 特征融合（Shallow Layer Fusion）

这是 FabriVLA 的关键设计之一。InternVL3.5 的深层（Layer 14）提供语义信息，浅层（Layer 6）保留更多空间细节。

融合方式：

```
fused = concat(deep, shallow) · W,  W ∈ R^{2048×1024}
W 初始化为 [I | 0]
```

- 初始化时 fused 等于 deep-only，保证向后兼容；
- 训练后模型学会利用浅层特征提升精密操作；
- 仅增加约 2.1M 参数。

**为什么选 Layer 6？** 作者发现 Layer 6 在"保留足够视觉细节"和"经过足够 Transformer 处理"之间取得平衡。

### 3.6 Flow Matching 训练

FabriVLA 采用 Flow Matching 生成动作序列：

#### 插值

```
x_t = (1 - t) · ε + t · a
ε ~ U([-1, 1]^{50×24})
t ~ Beta(2, 2)
```

Beta(2,2) 分布在中间区域密度最高，让训练样本集中在"既不是纯噪声也不是真值"的 informative 区域。

#### 速度场预测

模型学习条件速度场：

```
v_θ(x_t, t | s, image, prompt)
```

训练目标：

```
L = E[||v_θ(x_t, t) - (a - ε)||^2]
```

#### 推理

从均匀噪声出发，用 50 步 Euler 积分：

```
x_{t+Δt} ≈ x_t + v_θ(x_t, t) · Δt,  Δt = 1/50
```

最终得到 50 步动作 chunk，去归一化后闭环执行。

## 四、训练策略

### 4.1 数据

- **来源**：Evo-1 Meta-World demonstration dataset；
- **规模**：50 个任务 × 50 条轨迹 = **2,500 条轨迹**；
- **特点**：每任务样本量有限，考验模型在少样本下的泛化能力。

### 4.2 训练配置

| 配置项 | 数值 |
|---|---|
| 训练步数 | 100,000 steps（选用 93,000-step checkpoint） |
| Batch size | 40 |
| GPU | 5 × NVIDIA RTX PRO 6000 |
| 学习率 | 2.0e-5，1,000 warmup，cosine decay |
| 优化器 | AdamW，weight decay 1e-4 |
| 精度 | BF16 mixed precision |
| 并行策略 | DeepSpeed ZeRO-2 + FP32 master weights |
| 数据增强 | 随机裁剪、旋转、颜色抖动 |
| 训练框架 | HuggingFace Accelerate |

### 4.3 关键技术细节：DeepSpeed ZeRO-2 + FP32 Master Weights

论文特别指出：**如果仅用 BF16 而不使用 FP32 master weights，VLM backbone 的梯度更新会被 BF16 量化噪声淹没，导致 backbone 几乎不更新。** 使用 DeepSpeed ZeRO-2 + FP32 master weights 是保证单阶段联合训练有效性的关键工程技巧。

## 五、实验评估

### 5.1 评估协议

- **基准**：Meta-World MT50，50 个操作任务；
- **分组**：easy / medium / hard / very hard 四个难度层级；
- **评估**：每个任务 10 个 episode，随机初始状态，seed=4042；
- **成功标准**：400 步 horizon 内完成任务目标；
- **指标**：
  - **Tier-average**：四个 tier 得分的平均；
  - **Overall episode-level**：500 个 episode 中成功比例；
  - **Bucket score**：任务组内 per-task 成功率平均。

### 5.2 与近期 VLA 的对比

| 方法 | 参数量 | 机器人数据预训练 | Easy | Medium | Hard | Very Hard | Tier Avg. |
|---|---|---|---|---|---|---|---|
| FabriVLA | **0.89B** | 无 | **95.0** | 88.2 | 86.7 | 90.0 | **90.0** |
| LA4VLA | 更大 | 有/无 | - | **更高** | **更高** | **更高** | 87.5 |
| Evo-Depth | - | 有 | - | - | - | - | 84.4 |

**关键结论**：
- FabriVLA 用最小参数量取得最高 tier-average 成功率；
- 在 easy tier 上最强，在 medium/hard/very hard 上仅次于 LA4VLA；
- 无机器人数据预训练，依然超过有预训练的深度增强基线。

### 5.3 任务需求分组分析

论文还按主要操作需求对 50 个任务做了互斥分组：

| 任务分组 | FabriVLA 表现 |
|---|---|
| 平面精密滑动 | **强** |
| 铰接接触驱动 | **强** |
| 插装/抽取 | **强** |
| 工具介导操作 | **弱** |
| 粗粒度到达与搬运 | **弱** |
| 抓取与放置 | **弱** |

**结论**：FabriVLA 擅长局部空间对齐和精密接触操作，但在需要长程搬运和复杂工具使用的任务上仍有提升空间。

### 5.4 训练动态

- 性能从 20k 到 93k steps 持续提升；
- 93k checkpoint 达到最佳 tier-average 和 overall episode-level；
- 100k checkpoint 出现轻微过拟合/性能下降。

### 5.5 消融研究：浅层特征融合

| 配置 | Tier Avg. | Overall Episode |
|---|---|---|
| Deep-only (Layer 14) | 82.9% | 86.8% |
| + Shallow Layer 6 Fusion | **90.0%** | **92.0%** |

- **Tier-average 提升 7.1%**；
- **Overall episode-level 提升 5.2%**；
- 在所有四个难度 tier 上均有提升。

这说明浅层特征对精密操作至关重要，仅依赖 VLM 深层语义特征会损失必要的空间细节。

## 六、关键洞察与局限

### 6.1 关键洞察

1. **1B 规模 VLM 足以支撑多任务精密操作**：不需要数十亿参数，关键是让动作头同时看到语义和空间特征；
2. **浅层特征融合比模型规模更重要**：2.1M 参数的浅层融合带来 7.1% 的 tier-average 提升，边际收益远高于增加 backbone 参数量；
3. **门控自注意力提供平滑优化路径**：从"独立动作预测"到"联合轨迹规划"的过渡是渐进的，避免了训练不稳定；
4. **单阶段联合训练可行**：只要工程细节到位（如 FP32 master weights），从头联合训练 VLM + 动作头不需要多阶段课程；
5. **小数据也能 work**：每任务仅 50 条轨迹，说明合理的架构和训练策略可以缓解数据稀缺问题。

### 6.2 局限

1. **仅在仿真环境评估**：Meta-World 是仿真基准，真实机器人部署效果未验证；
2. **无长程任务**：50 步动作 horizon 适合短时操作，但难以处理需要多阶段规划的长程任务；
3. **语言指令简单**：任务 prompt 是短文本指令（如"Dunk the basketball into the basket"），未涉及复杂多轮语言理解；
4. **工具使用和长程搬运较弱**：论文自己指出这两类任务是主要短板；
5. **未与更大规模 VLA 直接对比**：缺少与 π0、RT-2 等十亿级 VLA 在相同设置下的对比。

### 6.3 未来方向

1. **真实机器人验证**：将 FabriVLA 部署到真实机械臂，验证仿真到真实的迁移；
2. **长程任务规划**：引入上层 Planner，把长程任务拆成短程操作序列；
3. **更复杂的语言指令**：支持多轮对话、模糊意图、纠错指令；
4. **多模态观测**：加入深度、力觉、触觉等传感器，提升接触式操作能力；
5. **模型扩展**：探索 2B/4B 规模的 VLM backbone 是否能进一步提升性能而不显著增加延迟。

## 七、与 ABot-N0 / Qwen-RobotNav 的对比

| 维度 | FabriVLA | ABot-N0 | Qwen-RobotNav |
|---|---|---|---|
| **任务类型** | 机械臂操作 | 移动导航 | 移动导航 |
| **VLM Backbone** | InternVL3.5-1B | Qwen3-4B | Qwen3-VL |
| **参数量** | 0.89B | ~4.1B | 2B/4B/8B |
| **动作头** | Flow Matching | Flow Matching | MLP |
| **关键创新** | 浅层特征融合 + 门控自注意力 | Brain-Action 解耦 + SAFE-GRPO | 参数化观察接口 |
| **训练数据** | 2,500 条演示轨迹 | 16.9M 轨迹 + 5.0M 推理样本 | 15.6M 样本 |
| **部署目标** | 精密操作（未部署） | Unitree Go2 边缘部署 | Unitree Go2 边缘部署 |
| **主要评估** | Meta-World MT50 | 7 个导航基准 | 多导航基准 + 自动驾驶 |

**核心差异**：
- FabriVLA 聚焦**操作（manipulation）**，ABot-N0/Qwen-RobotNav 聚焦**导航（navigation）**；
- FabriVLA 强调**轻量级**（0.89B），后两者更大；
- FabriVLA 的**浅层特征融合**是操作任务特有的需求，导航任务更依赖长程上下文和语义记忆。

## 八、结论

FabriVLA 证明了**轻量级 VLA 模型可以在多任务精密操作上达到强劲性能**。它通过三个关键设计——InternVL3.5-1B backbone、门控自注意力 Flow Matching 动作头、浅层 VLM 特征融合——在仅 0.89B 参数的条件下，取得了 Meta-World MT50 tier-average 90.0% 的成功率。

对于希望构建可部署机器人操作系统的研究者和工程师，FabriVLA 提供了一个重要参考：**模型规模不是唯一决定因素，让动作头同时访问语义和空间特征、并保持时序一致性，往往比单纯增加参数更有效。**
