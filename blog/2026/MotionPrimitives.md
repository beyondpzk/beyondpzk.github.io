# 运动原语：机器人动作世界的"字母表"

在机器人学习里，我们常说 "end-to-end"、"behavior cloning"、"diffusion policy"，却很少停下来问一个更基础的问题：**机器人到底在学什么？** 它学的是每一个关节的力矩吗？是末端执行器的三维坐标吗？还是一段完整的"擦桌子"或"开门"动作？

运动原语（Motion Primitives）就是这个问题的答案之一。它不是某种特定算法，而是一种对机器人动作进行"分层表达"的思想：**把复杂任务拆成有限、可复用、语义清晰的基本动作单元**。就像英语用 26 个字母拼出所有单词，机器人也可以用一组运动原语拼出绝大多数操作。

---

## 1. 为什么需要运动原语？

想象你要教一个机械臂"把杯子从桌上拿到水槽"。如果直接在整个状态-动作空间上优化，问题会非常难：

- 状态空间巨大：关节角、末端位姿、图像、力觉……
- 动作序列极长：一次抓取可能涉及上千个时间步；
- 复用性差：学会了"拿杯子"，对"拿瓶子"几乎毫无帮助；
- 数据效率低：端到端模仿学习需要大量完整任务轨迹。

运动原语的核心思想是：**不要在原始动作空间里硬学，而是先定义一组"动作字母"，再学习怎么把它们组合成"单词"和"句子"**。

这样带来几个好处：

1. **降维**：把高维连续动作压缩到低离散原语空间；
2. **泛化**：同一个 "reach" 原语可以用在抓取、放置、推动等多种任务；
3. **可解释**：每个原语都有物理意义，调试和安全性分析更容易；
4. **数据效率**：只需要学习少量原语参数，以及原语之间的组合策略。

## 2. 运动原语的几种经典定义

### 2.1 从轨迹模板到动态系统

最早的一类运动原语是 **DMP（Dynamic Movement Primitives，动态运动原语）**，由 Schaal 等人在 2000 年代初提出。

DMP 把一条轨迹看作一个非线性动态系统：

$$
\tau \dot{z} = \alpha_z (\beta_z (g - y) - z) + f(s)
$$

其中 $y$ 是位置，$z$ 是速度，$g$ 是目标点，$f(s)$ 是一个由基函数加权组合而成的非线性项，$s$ 是一个从 1 衰减到 0 的相位变量。

**关键性质**：

- **目标吸引性**：无论起点在哪，系统最终都会收敛到目标 $g$；
- **时间可缩放**：通过调整 $\tau$ 可以加快或减慢执行；
- **形状可调制**：通过改变 $f(s)$ 的权重，可以生成不同形状的轨迹。

DMP 的一个经典应用是机器人打乒乓球、写字、搅拌等需要光滑轨迹的任务。

### 2.2 基于样条/关键帧的原语

在工业机械臂中，更常见的做法是 **关键帧 + 插值**。比如一个装配任务被拆成：

```
approach -> grasp -> lift -> transport -> place -> release -> retreat
```

每个阶段用一个或几个关键帧定义，中间用样条、直线或圆弧插值。这种原语更工程化，优点是可预测、可验证，缺点是泛化能力有限。

### 2.3 基于运动学的原语

近年来在操作（manipulation）领域流行的一类方法是把动作定义成 **参数化技能（parameterized skills）**，例如：

- `Pick(object)`
- `Place(object, target)`
- `Push(object, direction, distance)`
- `Open(drawer, handle)`

每个原语内部封装了完整的控制器或策略网络，外部只暴露几个高层参数。这种方法在 **TAMP（Task and Motion Planning）** 和 **VLA（Vision-Language-Action）** 模型中都很常见。

## 3. 运动原语 vs. 端到端策略

很多人会问：既然扩散策略（Diffusion Policy）、RT-1/RT-2、π₀ 这些端到端模型这么强，为什么还要运动原语？

答案是：**两者不是对立关系，而是不同抽象层**。

| 维度 | 端到端策略 | 运动原语 |
|------|-----------|---------|
| 输入 | 图像/语言指令 | 原语 ID + 参数 |
| 输出 | 原始动作（关节位置/速度） | 一段参数化轨迹或技能 |
| 数据需求 | 大 | 中 |
| 可解释性 | 低 | 高 |
| 长程任务 | 容易遗忘/漂移 | 通过组合保持结构 |
| 安全性 | 难验证 | 每个原语可单独验证 |

一个越来越流行的范式是 **"高层用 LLM/VLM 规划原语序列，低层用策略网络执行原语"**。例如：

1. 视觉语言模型看到"把红色积木放到蓝色盒子里"；
2. 它输出原语序列：`pick(red_block) -> place(blue_box)`；
3. 每个原语由一个专门的扩散策略或阻抗控制器执行。

这种两层结构兼具了端到端学习的灵活性和运动原语的结构化优势。

## 4. 运动原语从哪里来？

### 4.1 人工设计

早期机器人系统几乎全靠专家手工设计。比如 Boston Dynamics 的 Atlas 跑酷动作、工业机器人的码垛程序，都是工程师把任务拆成一个个预定义原语。优点是稳定可靠，缺点是对每个新任务都要重新设计。

### 4.2 从示教数据中学习

**Learning from Demonstration (LfD)** 是另一条路。让人远程操作机器人完成多次任务，然后从数据中提取重复出现的轨迹片段作为原语。

常见方法包括：

- **GMM/GMR**：用高斯混合模型拟合多演示轨迹，生成平均轨迹；
- **PCA**：把高维轨迹投影到低维潜空间，每一维对应一种运动模式；
- **聚类**：把演示轨迹分段聚类，每一类就是一个原语；
- **DMP 加权**：用 Locally Weighted Regression（LWR）学习 DMP 的权重。

### 4.3 从交互中自动发现

更前沿的方向是让机器人通过与环境交互**自动发现**有用的原语。例如：

- **Options / Skill Discovery**：在强化学习框架下，把"有用的动作序列"定义为 option，通过内部奖励自动发现；
- **世界模型 + 规划**：像 DreamerV3、TD-MPC 这类方法，先在潜空间里学习世界模型，然后在抽象动作空间做规划，这些抽象动作本身就是一种原语。

## 5. 运动原语在当代机器人学习中的位置

### 5.1 与 VLA 模型的结合

VLA 模型（如 RT-2、OpenVLA、π₀）直接把视觉-语言映射到动作 token。但长程任务中，纯端到端方法容易出现**组合性错误**和**时间尺度问题**。

一个自然的改进是：让 VLA 输出**原语 token** 而不是原始动作。比如：

```
"pick the mug" -> [PRIMITIVE: pick, target: mug]
"pour water"   -> [PRIMITIVE: tilt, angle: 45deg, duration: 2s]
```

这样语言模型负责"说什么"，底层策略负责"怎么做"，两者解耦。

### 5.2 与扩散策略的结合

扩散策略（Diffusion Policy）擅长生成光滑、多模态的轨迹。把扩散策略限制在某个原语的参数空间里，可以显著提升采样效率和动作质量。例如：

- 原语 `push` 的参数是 `(contact_point, direction, speed)`；
- 扩散模型只在这个低维参数空间上生成分布；
- 底层控制器把参数转换成具体轨迹。

### 5.3 与人形机器人

人形机器人的动作空间尤其高维（全身 20+ DOF）。运动原语在这里几乎是必需的。常见的原语包括：

- 平衡原语（standing, stepping, crouching）
- 步态原语（walk, turn, side-step, back-step）
- 操作原语（reach, grasp, manipulate）
- 接触原语（push, lean, support）

把这些原语组合起来，才能实现真正有用的全身行为。

## 6. 一个简单例子：推开障碍物

假设机器人要推开面前的积木。用运动原语描述，可以只定义三个参数：

```python
primitive = Push(
    target="red_block",
    direction=[1.0, 0.0, 0.0],  # 沿 x 轴
    distance=0.15,              # 推 15 cm
    speed=0.05                  # 速度 5 cm/s
)
```

底层控制器根据这个原语：

1. 规划末端接近路径；
2. 建立接触；
3. 沿指定方向施加力；
4. 移动指定距离后释放。

如果换成端到端学习，模型需要直接输出每一个时间步的关节角度；而用了原语，模型只需要输出 4 个参数。

---

## 7. 总结

运动原语不是某一篇论文的专属概念，而是机器人领域几十年来的核心思想之一：

> **用有限、结构化、可复用的动作单元，来表达无限、复杂、多变的任务行为。**

它既可以很经典（DMP、关键帧），也可以很现代（VLA 中的技能 token、扩散策略的参数空间）。无论机器人学习怎么发展，**在"做什么"和"怎么做"之间建立一个清晰的中间层**，始终是一个有价值的设计选择。

对于正在关注 VLA、世界模型和具身智能的同学来说，理解运动原语能帮助你更好地判断：一个系统到底是在原始动作空间里 brute-force 地学习，还是真正学会了**组合与泛化**。
