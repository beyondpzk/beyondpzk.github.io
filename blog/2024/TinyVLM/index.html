TinyVLM

本文主要讨论的是如何用较大的VLM模型蒸馏出一个小的VLM模型.这个小的VLM模型可能因为要在端上跑,所以算子会有单独设计,模型完全是train from scratch.

下面是一个可行的技术方案. 比如大的VLM模型是72B,小的VLM模型是2B.

这种方案虽然工程量巨大，但能保证你的 2B 模型在每一个组件（ViT, LLM, Projector）和每一个阶段（Pre-train, SFT）都得到“老师”的指点，从而打破 Scaling Law 的限制，实现“小钢炮”级别的性能。

为了确保执行层面的万无一失，我针对你的方案 3 细化了具体的执行逻辑和 Loss 设计：

阶段一：组件级独立蒸馏 (Component-Level Distillation)

这一步是打地基。地基不牢，地动山摇。

1. ViT 蒸馏 (0.7B Student <- SigLIP-SO400M/EVA-CLIP Teacher)

  • 目标：让你的 0.7B ViT 拥有顶级视觉表征能力。
  • 方法：
    • 输入：相同的图片（经过相同的 Augmentation）。
    • 对齐对象：ViT 的最后一层输出特征 (Last Hidden State)。
    • Loss：$L_{ViT} = MSE(Student_Feat, Teacher_Feat)$。
    • 注意：如果维度不同（比如 Student 1024 vs Teacher 1152），加一个 Linear Projector 把 Student 映射到 Teacher 维度再算 Loss。

2. LLM 蒸馏 (1.3B Student <- Qwen2/Llama3 Teacher)

  • 目标：让你的 1.3B LLM 拥有超越参数量的语言理解和逻辑能力。
  • 方法：
    • 情况 A (Tokenizer 相同)：直接蒸馏 Logits。
      • $L_{LLM} = KL_Div(Student_Logits, Teacher_Logits)$。
    • 情况 B (Tokenizer 不同，大概率是这种情况)：
      • 隐层蒸馏 (Hidden State Distillation)：挑选 Student 和 Teacher 的关键层（如第 1, 12, 24 层），通过 Projector 对齐特征向量，计算 MSE Loss。
      • 数据蒸馏：使用 Teacher 生成的高质量合成数据（Textbooks is all you need 思路）进行训练。

阶段二：VLM 联合蒸馏 (VLM Joint Distillation)

这是你问题的核心。在把 ViT 和 LLM 拼起来后，如何利用 72B VLM 进行蒸馏？

策略：双管齐下，首尾兼顾。

1. 视觉端蒸馏 (Visual Distillation in VLM)

问题：VLM 训练时，LLM 的梯度回传会改变 ViT 的权重，有时会导致 ViT 为了迎合 LLM 而丢失纯粹的视觉分辨能力（Catastrophic Forgetting）。 解法：继续蒸馏 Stage 1 的视觉老师，而不是 72B VLM 的 ViT。 (这里要注意.)

  • 为什么？ 因为 Qwen2-VL-72B 的 ViT 是动态分辨率的，且经过了特定修改，和你固定分辨率的 ViT 特征图（Feature Map）很难空间对齐。
  • 操作：在 VLM 训练过程中，冻结一个 SigLIP-SO400M（Stage 1 的老师）在旁边。
  • Loss：$L_{Visual_Reg} = MSE(Student_ViT_Out, Frozen_SigLIP_Out)$。
  • 作用：这作为一个正则化项 (Regularization)，确保你的 ViT 在学习文本对齐的同时，不丢失原本的视觉特征。

2. 输出端蒸馏 (Logits Distillation in VLM)

对象：Student VLM (2B) vs Teacher VLM (72B)。 操作：

  • 输入：同一张图 + 同一个 Prompt。
  • 对齐：
    • 如果 Tokenizer 相同：计算 KL Divergence。这是让小模型学会 72B “思维链”和“不确定性”的神技。
    • 如果 Tokenizer 不同：你只能做 Sequence-Level Distillation。即把 72B 的输出当做 Ground Truth，用 Cross-Entropy Loss 训练。
  • 作用：直接复制 72B 的推理能力。

总结：你的终极 Loss 函数 (The Ultimate Recipe)

在 VLM 联合训练阶段，你的总 Loss 应该是这样的：

\[L_{Total} = \alpha \cdot L_{Text\_Gen} + \beta \cdot L_{Logits\_Distill} + \gamma \cdot L_{Visual\_Reg}\]
  1. $L_{Text_Gen}$ (Hard Label): 基础的 Next Token Prediction。保证模型“说人话”。
  2. $L_{Logits_Distill}$ (Soft Label): 模仿 72B VLM 的输出概率分布。保证模型“有逻辑”。
  3. $L_{Visual_Reg}$ (Feature Match): 模仿 SigLIP 的视觉特征。保证模型“看得清”。

关键提示 (Pro Tips)

  • $\gamma$ 的权重衰减：在训练初期，$\gamma$ (视觉正则化) 应该大一点，防止 ViT 被随机初始化的 Projector 带偏。随着训练进行，逐渐减小 $\gamma$，让 ViT 更多地服务于 LLM 的文本生成任务。
  • Projector 的初始化：在开始联合蒸馏前，建议先冻结 ViT 和 LLM，只训练 Projector 一个 Epoch（用简单的 Caption 数据）。这能让 Projector 先“热身”，避免一开始梯度太乱冲垮了 ViT 和 LLM 的预训练权重。

结论：“两头蒸馏，中间对齐” 是训练高性能小参数 VLM 的最佳路径。

数据端

对于 2B 这种小参数模型，数据质量决定了模型的上限（Data is the Ceiling）。

小模型容量有限，不能像 72B 那样“死记硬背”海量噪音数据。你必须喂给它经过提炼的、高密度的、逻辑清晰的数据。

既然你手里有 Qwen2-VL-72B（或者 GPT-4V 级别模型），你应该把它当成一个“数据工厂”，生产以下 4 类核心数据用于蒸馏。这是目前工业界训练 SOTA 小模型（如 MiniCPM-V, Phi-3-Vision）的标准配方：

第一类：高密度详细描述 (Dense Captions) —— 占比 ~40%

目的：让 ViT 和 LLM 彻底对齐，让模型学会“看细节”。 痛点：传统的 COCO/LAION 数据只有简短的一句话（”A cat on the grass”），这对 2B 模型来说信息量太低，学不到东西。 蒸馏策略：

  • 源图片：从 LAION-2B, COYO-700M 中筛选出的高美学评分图片。
  • Teacher 动作：让 72B 模型生成 300-500 词的超详细描述。
  • Prompt 模板：

    “Describe this image in extreme detail. Start with the main subject, then describe the background, colors, lighting, object relationships, and any text present in the image. Do not miss any small details.”

  • 效果：这能让你的 2B 模型学会把视觉特征映射到丰富的词汇上（颜色、材质、空间关系）。这是 ShareGPT4V 的核心秘诀。

第二类：视觉思维链 (Visual Chain-of-Thought, CoT) —— 占比 ~30%

目的：弥补小模型逻辑推理能力的不足。 痛点：直接给答案（Answer: “Yes”），小模型学不会推理过程，遇到稍微变一点的问题就懵了。 蒸馏策略：

  • 源图片：ScienceQA, MathVista, MMMU 等推理密集型数据集的图片。
  • Teacher 动作：让 72B 模型生成“观察-分析-结论”三段式答案。
  • Prompt 模板：

    “Question: [User Question] Answer: Let’s think step by step.

    1. Observation: First, I see [Key Elements] in the image.
    2. Reasoning: Based on [Visual Clue A], it implies [Fact B]. The relationship between X and Y suggests…
    3. Conclusion: Therefore, the answer is [Final Answer].”
  • 价值：这是蒸馏的精髓。你不是在蒸馏“答案”，而是在蒸馏“思考路径”。2B 模型一旦学会了这个套路，泛化能力会暴涨。

第三类：OCR 与文档理解 (Document/Chart Understanding) —— 占比 ~20%

目的：提升工具属性，这是 VLM 落地最常用的功能。 痛点：OCR 需要极高的分辨率和精准的坐标对应。 蒸馏策略：

  • 源图片：DocVQA, ChartQA, SynthDoG (合成文档)。
  • Teacher 动作：
    1. 全文转录：让 72B 把图里的字全部读出来，作为 Ground Truth。
    2. 结构化提取：让 72B 把表格图片转成 Markdown 格式。
    3. 坐标关联 (Grounding)：让 72B 输出物体/文字的 Bounding Box（例如：<box>200,300,400,500</box>）。
  • 注意：Qwen2-VL-72B 对坐标的处理非常强，一定要把这个能力蒸馏过来。这能让你的 2B 模型具备“指哪打哪”的能力。

第四类：纯文本指令微调 (Pure Text SFT) —— 占比 ~10%

目的：防止“灾难性遗忘” (Catastrophic Forgetting)。 痛点：VLM 训多了，模型会变笨，只会看图，不会正常聊天了。 策略：

  • 数据：高质量的 Alpaca, ShareGPT 纯文本对话数据。
  • 操作：在训练 VLM 时，随机混入纯文本数据（不需要过 Vision Encoder，直接进 LLM）。
  • 价值：保持 LLM 的语言流畅度和逻辑基座。

进阶：负样本与幻觉修正 (Negative Samples for DPO)

追求极致的话,还需要准备DPO (Direct Preference Optimization) 数据。

  • 构造方法：
    • 正样本 ($y_w$)：72B 生成的完美回答。
    • 负样本 ($y_l$)：
      1. 幻觉构造：把正样本里的物体改名（把“猫”改成“狗”），或者把颜色改错。
      2. 逻辑陷阱：构造一些“图片里没有”的问题，强迫模型回答“I cannot answer this based on the image”。
  • 价值：这能大幅减少小模型“一本正经胡说八道”的概率。

总结：数据配方表 (The Recipe)

假设你总共训练 1 Epoch（对于高质量数据，1 Epoch 足够，多了会过拟合），建议的数据混合比例如下：

数据类型 来源建议 作用 蒸馏重点
Dense Caption ShareGPT4V-Caption, Allava 基石：建立视觉-语言映射 极尽详尽的描述
Visual CoT ScienceQA, MathVista (用 72B 重写) 大脑：提升逻辑推理 Step-by-step 推理过程
OCR/Chart DocVQA, ChartQA (用 72B 修正) 工具：提升实用性 Markdown 表格, JSON 提取
Grounding RefCOCO (用 72B 生成 Box) 交互：物体定位 <box> 坐标预测
Pure Text Ultrachat, Alpaca 保底：维持语言能力 纯文本对话

一句话建议： 不要直接用开源的原始数据集（Raw Data）。把所有图片都过一遍你的 Qwen2-VL-72B，重新生成 Caption 和 QA。用 72B 的“嘴”喂出来的 2B 模型，才是最强的。

训练策略上

绝对不能只用一步（One-Stage）训练。

虽然你有高质量的蒸馏数据，但 VLM 的训练本质上是一个“课程学习”（Curriculum Learning）的过程。就像教小孩，必须先教他“认字”（对齐），再教他“写作文”（推理）。如果混在一起教，模型会因为梯度冲突（Gradient Conflict）而难以收敛，或者学成一个“四不像”。

针对你的 2B (0.7B ViT + 1.3B LLM) 架构，为了达到 SOTA 效果，必须分阶段进行。

以下是为你定制的 “三步走”蒸馏训练策略：

第一阶段：模态对齐预训练 (Stage 1: Alignment Pre-training)

“先让眼睛和大脑连通。”

  • 目标：初始化 Projector（连接层）。此时 Projector 是随机初始化的，如果直接喂复杂的推理数据，LLM 会收到一堆乱码，导致 LLM 的权重被破坏。
  • 训练参数：
    • 训练：Projector (全量)。
    • 冻结：ViT, LLM。
    • (进阶选项：如果你显存够，可以解冻 ViT 的最后 2-3 层，让视觉特征更适应 LLM)。
  • 数据配比：
    • 100% Image Captioning 数据（第一类数据）。
    • 不要放 CoT 或 OCR 数据，这时候模型只需要学会“图里有什么”。
  • 蒸馏策略：
    • 重点：$L_{Visual_Reg}$ (ViT 特征对齐)。
    • 辅助：$L_{Text}$ (Caption 生成)。
    • 不需要 Logits 蒸馏（因为 Caption 任务太简单，不需要学 Teacher 的概率分布）。
  • 耗时：很短（约 1 Epoch）。

第二阶段：视觉指令微调 (Stage 2: Visual Instruction Tuning / SFT)

“教会模型听懂指令并进行推理。” —— 这是最关键的一步。

  • 目标：让模型学会逻辑推理、多轮对话和工具使用。
  • 训练参数：
    • 全量解冻 (Full Fine-tuning)：ViT + Projector + LLM。
    • 对于 2B 小模型，千万不要用 LoRA，直接全量训，效果最好。
  • 数据配比（这就是你上一轮问的配方）：
    • 40% Visual CoT (推理)。
    • 30% OCR/Chart/Doc (工具)。
    • 20% Dense Caption (保持看图能力)。
    • 10% Pure Text SFT (保持语言能力)。
  • 蒸馏策略 (火力全开)：
    • $L_{Logits_Distill}$ (KL Div)：这是核心。让 2B 模型模仿 72B 对复杂问题的思考概率。
    • $L_{Visual_Reg}$：继续保持，防止 ViT 在学语义时忘掉视觉特征。
  • 耗时：主要训练时间（2-3 Epochs）。

第三阶段：长上下文与偏好对齐 (Stage 3: Long-Context & DPO) —— 可选，但推荐

“让模型更像人，更安全，更少幻觉。”

  • 目标：解决“复读机”问题，减少幻觉，支持高分辨率（多图输入）。
  • 训练参数：全量微调。
  • 数据配比：
    • DPO 数据：正样本（72B 生成） vs 负样本（幻觉/错误）。
    • 多图/长文数据：Interleaved Image-Text 数据（如 MMC4）。
  • 蒸馏策略：
    • 这里主要靠 DPO Loss，不再需要 Logits 蒸馏。
  • 耗时：很短（约 0.5 - 1 Epoch）。

为什么必须分阶段？（技术原理）

  1. 梯度冲突 (Gradient Conflict)：
    • 对齐阶段的任务是“把图像特征映射到词向量空间”。
    • 推理阶段的任务是“根据词向量进行逻辑推演”。
    • 如果一步到位，Projector 还没训好，传给 LLM 的就是噪声。LLM 会为了拟合这些噪声而大幅修改权重，导致它原本的语言能力（Pre-training 获得的）崩塌。
  2. 数据效率 (Data Efficiency)：
    • Caption 数据便宜且量大，适合用来“热身”Projector。
    • CoT/Reasoning 数据昂贵且难生成，应该用在刀刃上（Stage 2）。如果在 Stage 1 就用掉，模型根本学不进去（Under-fitting）。
  3. 分辨率策略 (Resolution Strategy)：
    • Stage 1 可以用低分辨率（如 224x224 或 336x336）快速跑，省钱省时间。
    • Stage 2 必须上高分辨率（如 448x448 或 512x512），因为 OCR 和推理需要看清细节。
    • 如果混在一起训，分辨率很难动态调整。

总结：你的最终训练脚本逻辑

# Stage 1: Alignment
model = VLM(vit, llm, projector)
freeze(vit); freeze(llm); unfreeze(projector)
dataset = Dense_Caption_Data
loss = MSE(vit_feat, teacher_vit_feat) + CrossEntropy(text)
train(model, dataset, loss)

# Stage 2: SFT (The Big One)
unfreeze(vit); unfreeze(llm) # Full Tuning
dataset = Mix_Data(CoT=0.4, OCR=0.3, Caption=0.2, Text=0.1)
# 核心蒸馏逻辑
loss = alpha * CrossEntropy(text) + \
       beta * KL_Div(student_logits, teacher_72b_logits) + \
       gamma * MSE(vit_feat, teacher_vit_feat)
train(model, dataset, loss)

# Stage 3: DPO (Optional)
dataset = Preference_Data
loss = DPO_Loss(student_policy, ref_policy, chosen, rejected)
train(model, dataset, loss)

结论：不要偷懒做 One-Stage。分阶段训练 + 全链路蒸馏，是目前工业界训练高性能小模型的唯一正解。
