# NaVid：基于视频 VLM 的连续环境视觉-语言导航

> **论文**：*NaVid: Video-based VLM Plans the Next Step for Vision-and-Language Navigation*  
> **作者**：Jiazhao Zhang, Kunyu Wang, Rongtao Xu, Gengze Zhou, Yicong Hong, Xiaomeng Fang, Qi Wu, Zhizheng Zhang†, He Wang†  
> **单位**：Peking University, BAAI, CASIA, University of Adelaide, ANU, Galbot  
> **发布时间**：2024-02-24（arXiv），RSS 2024  
> **arXiv**：[https://arxiv.org/abs/2402.15852](https://arxiv.org/abs/2402.15852)  
> **项目主页**：[https://pku-epic.github.io/NaVid/](https://pku-epic.github.io/NaVid/)

---

## 摘要

NaVid 是首个将**视频大视觉语言模型（Video VLM）**直接用于连续环境视觉-语言导航（VLN-CE）的工作。它只依赖单目 RGB 视频流和自然语言指令，端到端输出下一步动作，**完全不使用深度传感器、里程计或先验地图**。NaVid 基于 LLaMA-VID 构建，通过 Q-Former 提取与指令相关的 token，同时用 Grid Pool 保留全图几何信息，再经 Vicuna-7B LLM 生成带参数的自然语言动作。在 VLN-CE R2R Val-Unseen 上，NaVid 在纯 RGB 设置下达到 **37.4% SR / 35.9% SPL**，超越同期需要深度+里程计+全景图的方法；在 R2R→RxR 跨数据集零样本迁移中，SPL 从 6.3% 提升到 **21.2%**。真实世界实验中，NaVid 在 Turtlebot4 上仅使用 RGB 相机完成约 **66%** 成功率。

---

## 一、研究背景与动机

### 1.1 连续环境 VLN 的现实瓶颈

传统 VLN-CE 方法通常依赖：
- **深度图 / 里程计**：存在累积误差和 sim-to-real 域偏移；
- **先验地图 / 离散导航图**：难以直接部署到真实机器人。

这些额外传感器本身成为泛化的瓶颈：真实世界里的深度噪声、光照变化、地面纹理都会让仿真里训练的策略失效。

### 1.2 核心洞察：像人一样，只看 RGB

NaVid 的主张是：**导航可以像人类一样，仅凭单目 RGB 视频流和语言指令完成下一步决策。** 这样做的好处是：

- 消除了深度、里程计、地图等跨域差异；
- 动作输出为自然语言形式，可直接利用 VLM 的预训练常识；
- 视频流提供完整历史观测，保留时序上下文。

## 二、核心贡献

| 贡献 | 内容 |
|------|------|
| **首个视频 VLM 做连续 VLN** | 用 LLaMA-VID 处理视频流，端到端输出动作 |
| **纯 RGB-only SOTA** | 在 VLN-CE R2R Val-Unseen 上达到 37.4% SR / 35.9% SPL |
| **跨数据集泛化** | R2R→RxR 零样本迁移 SPL 从 6.3% 提升到 21.2% |
| **真实世界部署** | Turtlebot4 + 单目 RGB，200 条指令上约 66% 成功率 |
| **双 token 感知设计** | Instruction-queried token + Instruction-agnostic tokens 平衡指令跟随与几何感知 |

## 三、方法详解

### 3.1 整体架构

NaVid 的骨干是 **LLaMA-VID**（Vicuna-7B + EVA-CLIP ViT），整体流程如下：

```
单目 RGB 视频流 {x_0, ..., x_t}
           │
    ┌──────▼──────────────────────────────┐
    │  Observation Encoder                 │
    │  ├─ EVA-CLIP ViT (冻结)              │
    │  ├─ Q-Former → instruction-queried   │
    │  │   token (每帧 1 个)               │
    │  └─ Grid Pool → instruction-agnostic │
    │      tokens (当前帧 64 个, 历史帧 4个)│
    └──────┬──────────────────────────────┘
           │
    [HIS] {历史帧 tokens} [/HIS]
    [OBS] {当前帧 tokens} [/OBS]
    [NAV] {语言指令 tokens}
           │
    ┌──────▼──────────────────────────────┐
    │  Vicuna-7B LLM                      │
    └──────┬──────────────────────────────┘
           │
    "Forward 75 cm" / "Turn left 30°" / "Stop"
```

### 3.2 双 token 设计

这是 NaVid 感知编码的核心：

| Token 类型 | 生成方式 | 数量 | 作用 |
|------------|----------|------|------|
| **Instruction-queried** | Q-Former cross-attention | 每帧 1 个 | 提取与指令最相关的视觉信息 |
| **Instruction-agnostic** | Grid Pool | 当前帧 64 个，历史帧每帧 4 个 | 保留全图几何结构，支撑距离/角度估计 |

这种设计平衡了两个需求：
- **指令跟随**需要选择性关注；
- **动作参数**（走多远、转多少度）需要精确的几何信息。

消融表明，LLaMA-VID 原设计每帧仅 2 个 token 对 VLN 不够——QA 任务不需要精确距离，但导航需要。

### 3.3 动作输出

NaVid 输出**带参数的自然语言动作**：

```
FORWARD 75 cm
TURN-LEFT 30°
TURN-RIGHT 45°
STOP
```

然后通过正则表达式解析为可执行命令。论文报告该解析器在 R2R Val-Unseen 上有效动作率达到 **100%**——VLM 从未输出不可解析的格式。

## 四、训练策略

### 4.1 数据构成

| 数据类型 | 数量 | 作用 |
|----------|------|------|
| 导航动作规划（R2R oracle） | 320K | 学习标准最优轨迹 |
| 导航动作规划（Dagger 非最优） | 180K | 学习从偏离中恢复 |
| 指令推理（instruction reasoning） | 10K | 增强场景理解与指令跟随 |
| Web 视频 QA（LLaMA-VID 原有） | 763K | 保持 VLM 通用视觉理解 |

### 4.2 Dagger 数据收集

训练不只使用 ground-truth 轨迹：

1. 先在 R2R oracle 轨迹上训练基础模型；
2. 用该模型在 VLN-CE 中自主运行，采集非最优轨迹；
3. 将 180K 非最优轨迹加回训练集；
4. oracle + non-oracle 混合重新训练。

消融显示 Dagger 数据带来约 **5% SR 提升**（32.5% → 37.4%）。

### 4.3 训练规模

- 24 张 NVIDIA A100，约 28 小时，总计 672 GPU hours；
- EVA-CLIP、Q-Former、BERT、Vicuna-7B 加载预训练权重；
- 只训练 Vicuna 和 text encoder，1 个 epoch。

## 五、实验

### 5.1 VLN-CE R2R Val-Unseen

| 方法 | 观测 | TL | NE↓ | OS↑ | SR↑ | SPL↑ |
|------|------|----|-----|-----|-----|------|
| RGB-Seq2Seq | RGB | 4.86 | 10.1 | 8.1 | 0.0 | 0.0 |
| RGB-CMA | RGB | 6.28 | 9.55 | 10.8 | 5.0 | 4.43 |
| HPN+DN† | RGBD+Odo+Pano | 7.62 | 6.31 | 40.0 | 36.0 | 34.0 |
| WS-MGMap | RGBD+Odo | 10.0 | 6.28 | 47.6 | 38.9 | 34.3 |
| **NaVid** | **RGB only** | 7.63 | 5.47 | 49.1 | 37.4 | **35.9** |

在纯 RGB 设置下，NaVid 远超 RGB-Seq2Seq 与 RGB-CMA；在没有深度和里程计的情况下，SPL 甚至超越了需要深度+里程计+全景图的方法。

### 5.2 零样本跨数据集迁移：R2R → RxR

| 方法 | 设置 | TL | NE↓ | SR↑ | SPL↑ |
|------|------|----|-----|-----|------|
| RGB-Seq2Seq | RGB | 4.43 | 11.2 | 0.0 | 0.0 |
| RGB-CMA | RGB | 13.56 | 9.55 | 0.0 | 0.0 |
| A²Nav (GPT Planner, 零样本) | — | — | — | 16.8 | 6.3 |
| **NaVid** | **RGB** | 10.59 | 8.41 | **23.8** | **21.2** |

同期 RGB-only 方法在 RxR 上完全失败（0% SR）。NaVid 在 SR 上比 A²Nav 高出 41.7%，SPL 高出 236%，展示了 VLM 在未见语言分布上的强大泛化能力。

### 5.3 不同 LLM 骨子的对比

在 100 条 R2R Val-Unseen 子集上：

| LLM 骨子 | SR | SPL |
|---------|----|------|
| GPT-4V | 5.0 | 3.11 |
| LLaVA（仅 QA） | 0 | 0 |
| LLaMA-VID（仅 QA） | 0 | 0 |
| LLaVA-Nav | 10.0 | 9.43 |
| LLaMA-VID-Nav | 29.0 | 27.5 |
| **NaVid** | **38.0** | **35.4** |

核心发现：不做导航训练的大模型（GPT-4V / LLaVA / LLaMA-VID）基本失败，说明**大模型常识不能替代导航域训练**。

### 5.4 真实世界实验

- **平台**：Turtlebot4 + Kinect DK 单目 RGB；
- **场景**：Meeting_room / Office / Lab / Lounge；
- **指令**：200 条，含简单地标跟随与复杂组合指令；
- **成功率**：约 **66%**。

关键对比：需要深度+里程计的方法在 sim-to-real 迁移时 SR 下降超过 50%，而 NaVid 不存在这个问题——因为 RGB 在仿真与真实世界是同一种模态。

### 5.5 关键消融

| 消融项 | 核心发现 |
|--------|----------|
| 当前帧 token 数 | 64 个最优；2 个几何不足，128 个收益递减 |
| 历史帧 token 数 | 每帧 4 个最优；平衡时序信息与计算成本 |
| Dagger 数据 | +5% SR |
| 导航数据量 | 280K 以下进步缓慢，330K 起跃升，之后饱和 |

## 六、优势与局限

### 6.1 优势

1. **输入简洁带来强泛化**：去掉深度、里程计、地图，从根本上减少域偏移；
2. **视频流保留时序上下文**：比单帧或压缩特征更适合长程导航；
3. **双 token 设计精巧**：兼顾指令相关性与几何精度；
4. **自然语言动作输出**：直接利用 VLM 预训练能力，无需专门训练 action head；
5. **真实部署验证**：证明了纯 RGB VLN 在真实机器人上的可行性。

### 6.2 局限

1. **动作空间有限**：仅 FORWARD/TURN/STOP 原语，不能直接驱动真实机器人低层控制器；
2. **推理延迟高**：每步约 1.2–1.5 秒，实时性不足；
3. **只在 R2R 上训练**：导航域较窄，长程任务缺少显式规划；
4. **单 RGB 缺乏几何约束**：对遮挡、快速移动物体敏感；
5. **无显式错误恢复**：一旦偏离轨迹，主要靠 Dagger 数据中学到的恢复行为。

---

## 七、历史意义与后续影响

NaVid 是第一个证明“视频 VLM + 纯 RGB 可以做好 VLN 且泛化更强”的工作，为后续 VLA/VLN 系统提供了重要范式：

| 后续工作 | 与 NaVid 的关系 |
|----------|----------------|
| **NaVILA** | 继承视频 VLM 思路，扩展到足式机器人并输出连续速度指令 |
| **Qwen-RobotNav** | 扩大数据规模到 15.6M 样本，加入专门 action head，降低推理延迟 |
| **ABot-N0** | 引入 Planner + Topo-Memory + Self-Reflector，解决长程任务与错误恢复 |
| **StreamVLN** | 针对流式视频导航优化上下文建模与推理效率 |

---

## 八、总结

NaVid 通过将视频 VLM 引入连续环境 VLN，展示了“减少输入依赖”本身就能带来更强的泛化能力。它的双 token 感知设计、Dagger 训练策略和自然语言动作输出，为后续 Qwen-RobotNav、ABot-N0、NaVILA 等 VLA/VLN 系统奠定了基础。尽管存在动作空间有限、推理慢等局限，NaVid 首次证明了：仅凭单目 RGB 视频和自然语言指令，机器人可以在仿真与真实环境中完成可靠的视觉-语言导航。
