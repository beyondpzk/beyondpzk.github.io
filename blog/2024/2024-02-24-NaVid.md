# NaVid：基于视频 VLM 的视觉-语言导航——不用地图、不用深度、不用里程计，只看 RGB 视频

> **论文**: *NaVid: Video-based VLM Plans the Next Step for Vision-and-Language Navigation*  
> **作者**: Jiazhao Zhang, Kunyu Wang, Rongtao Xu, Gengze Zhou, Yicong Hong, Xiaomeng Fang, Qi Wu, Zhizheng Zhang†, He Wang†  
> **机构**: Peking University, BAAI, CASIA, University of Adelaide, ANU, Galbot  
> **发布时间**: 2024-02-24 (arXiv:2402.15852)  
> **项目主页**: https://pku-epic.github.io/NaVid/

---

## 一、一句话总结

NaVid 是**第一个基于视频 VLM 做连续环境 VLN 的工作**。它只靠单目 RGB 相机拍到的视频流和人类自然语言指令，端到端输出机器人的下一步动作——不需要深度传感器、不需要里程计、不需要地图。在 VLN-CE R2R 上达到 RGB-only SOTA，在 RxR 和真实世界场景上展现显著更强的泛化能力。

---

## 二、定位与核心贡献

### 2.1 为什么需要 NaVid

在 NaVid 之前，VLN 在连续环境（VLN-CE）里面临两个顽固问题：

1. **输入依赖导致泛化差**：大多数方法依赖深度图、里程计或先验地图。里程计有累积误差，深度感知在真实世界和仿真之间有域偏移——这些依赖本身就成为 Sim-to-Real 的瓶颈。
2. **离散环境里的 LLM Planner 方案做不了真实部署**：LLM 在离散图（R2R）上做路径规划效果不错，但这些方法输出的是"跳到下一个导航点"，无法直接控制真实机器人。

NaVid 的主张：**像人一样——只看 RGB，只输出下一步动作，没有其他传感器负担。** 这样就从根上消除了里程计噪声和深度/地图域偏移的问题。

### 2.2 核心贡献

1. **首个视频 VLM 做连续 VLN**——以视频流编码完整历史观测，不损失时序信息
2. **纯 RGB-only 达到 VLN-CE R2R SOTA**（SR 37.4%, SPL 35.9%），超越同期需要深度/里程计的方法
3. **跨数据集泛化能力极强**——仅在 R2R 上训练，零样本迁移到 RxR，SR 从同期最佳 16.8% 提升到 23.8%（相对提升 41.7%）
4. **真实世界部署**——在 4 个不同场景、200 条指令上达到约 66% 成功率，只用 RGB 视频

## 三、架构

NaVid 的骨干是一个通用视频 VLM——**LLaMA-VID**（Vicuna-7B + EVA-CLIP ViT），在此基础上加入 VLN 专项设计：

```
单目 RGB 视频流 {x_0, ..., x_t}
           │
    ┌──────▼──────────────────────────────┐
    │  Observation Encoder                 │
    │  ├─ EVA-CLIP ViT (冻结)              │
    │  ├─ Q-Former → instruction-queried   │
    │  │   token (每帧 1 个)               │
    │  └─ Grid Pool → instruction-agnostic │
    │      tokens (当前帧 64 个, 历史帧 4个)│
    └──────┬──────────────────────────────┘
           │
    [HIS] {历史帧 tokens} [/HIS]
    [OBS] {当前帧 tokens} [/OBS]
    [NAV] {语言指令 tokens}
           │
    ┌──────▼──────────────────────────────┐
    │  Vicuna-7B LLM                      │
    └──────┬──────────────────────────────┘
           │
    "Forward 75 cm" / "Turn left 30°" / "Stop"
```

### 3.1 关键设计：每帧两种 token

这是 NaVid 整个感知编码的灵魂。

**Instruction-queried token（橙色，每帧 1 个）**：通过 Q-Former 让视觉特征和指令文本做 cross-attention，提取"跟当前指令最相关的视觉信息"。比如指令是"走到白色桶前停下"，这个 token 会主动关注画面里白色物体的位置。

**Instruction-agnostic tokens（蓝色，数量可变）**：通过 grid pooling 压缩全图视觉信息，保留几何结构。当前帧给 64 个（动作规划的主依据），历史帧只给 4 个（保留空间上下文但不让 LLM 被历史淹死）。

这个设计平衡了两个需求：**指令跟随需要选择性关注，而动作参数（走多远、转多少度）需要保留全图几何信息。** LLaMA-VID 原始设计每帧只给 2 个 token，NaVid 的消融证明了对 VLN-CE 来说不够——单纯的 QA 任务不需要知道精确距离，但导航需要。

### 3.2 动作输出——自然语言形式

NaVid 输出的是**带参数的自然语言动作**，不是数值回归：

```
FORWARD 75 cm
TURN-LEFT 30°
TURN-RIGHT 45°
STOP
```

然后通过正则表达式解析出动作类型和参数。论文提到这个简单解析器在 R2R Val-Unseen 上达到 100% 的有效动作率——VLM 从来没输出过解析不了的格式。

这个设计的好处是：**保留了 VLM 的语言生成能力，所有 pretrain 的常识和推理都能自然迁移。**

## 四、训练策略

### 4.1 数据构成

| 数据类型 | 数量 | 作用 |
|---------|------|------|
| 导航动作规划 (R2R oracle) | 320K | 学习标准最优轨迹 |
| 导航动作规划 (Dagger 非最优) | 180K | 学习从偏离中恢复 |
| 指令推理 (instruction reasoning) | 10K | 场景理解 + 指令跟随 |
| Web 视频 QA (LLaMA-VID 原有) | 763K | 保持 VLM 的通用视觉理解 |

### 4.2 Dagger——从最优到非最优的轨迹采集

NaVid 的训练不只用 ground-truth 轨迹。标准做法只用最优路径训练——但真实世界机器人不可能每次都走在理想路径上。NaVid 的做法：

1. 先在 R2R oracle 轨迹（61 个 MP3D 场景，320K 步）上训练基础模型
2. 用这个模型在 VLN-CE 环境里自己跑，采集非最优轨迹
3. 把非最优轨迹（180K 步）加回训练集
4. 用 oracle + non-oracle 混合数据重新训练

消融显示：非最优数据贡献了约 5% 的 SR 提升（从 32.5% 到 37.4%）。

### 4.3 两个辅助任务

**Instruction Reasoning**：给 NaVid 看一段导航轨迹的视频，让它反推这段轨迹对应的自然语言指令。这个任务强迫模型学会把视觉行为和语言描述对齐——对 VLN 的 instruction following 能力有帮助。

**Web Video QA**：继承 LLaMA-VID 原有的视频问答数据，保持 VLM 的通用推理能力不被导航训练冲掉。这和 Qwen-RobotNav 的 15% VL 推理混入逻辑一致。

### 4.4 训练规模

- 24 张 NVIDIA A100，约 28 小时，总计 672 GPU hours
- 全部模块（EVA-CLIP, Q-Former, BERT, Vicuna-7B）加载预训练权重
- 只训练 Vicuna 和 text encoder，1 个 epoch

## 五、实验

### 5.1 VLN-CE R2R (Val-Unseen)

| Method | 观测 | TL | NE↓ | OS↑ | SR↑ | SPL↑ |
|--------|------|----|-----|-----|-----|------|
| RGB-Seq2Seq | RGB | 4.86 | 10.1 | 8.1 | 0.0 | 0.0 |
| RGB-CMA | RGB | 6.28 | 9.55 | 10.8 | 5.0 | 4.43 |
| HPN+DN† | RGBD+Odo+Pano | 7.62 | 6.31 | 40.0 | 36.0 | 34.0 |
| WS-MGMap | RGBD+Odo | 10.0 | 6.28 | 47.6 | 38.9 | 34.3 |
| **NaVid** | **RGB only** | 7.63 | 5.47 | 49.1 | 37.4 | **35.9** |

**关键**：在同为 RGB-only 的设置下，NaVid 碾压 RGB-Seq2Seq 和 RGB-CMA。在没有深度和里程计的情况下，它的 SR 甚至接近、SPL 超越了当时需要深度+里程计+全景图的最强方法 WS-MGMap。

### 5.2 零样本跨数据集：R2R → RxR

| Method | 设置 | TL | NE↓ | SR↑ | SPL↑ |
|--------|------|----|-----|-----|------|
| RGB-Seq2Seq | RGB | 4.43 | 11.2 | 0.0 | 0.0 |
| RGB-CMA | RGB | 13.56 | 9.55 | 0.0 | 0.0 |
| A2 Nav (GPT Planner, 零样本) | — | — | — | 16.8 | 6.3 |
| **NaVid** | **RGB** | 10.59 | 8.41 | **23.8** | **21.2** |

**同时期的 RGB-only 方法在 RxR 上完全失败（0% SR）。** A2 Nav 用了 GPT 做 zero-shot planning，NaVid 在 SR 上高出 41.7%，SPL 高出 236%。这证明了 VLM 在泛化到未见语言分布上的能力远超传统方法——RxR 是多语言、长指令、密集路标描述的 benchmark，传统模型在这上面直接崩了。

### 5.3 不同 LLM 做骨子的对比

论文在 100 条 R2R Val-Unseen 子集上替换了 LLM 骨子做对比：

| LLM 骨子 | SR | SPL |
|---------|----|------|
| GPT-4V | 5.0 | 3.11 |
| LLaVA (仅 QA) | 0 | 0 |
| LLaMA-VID (仅 QA) | 0 | 0 |
| LLaVA-Nav | 10.0 | 9.43 |
| LLaMA-VID-Nav | 29.0 | 27.5 |
| **NaVid** | **38.0** | **35.4** |

**核心发现**：不做导航训练的大模型（GPT-4V / LLaVA / LLaMA-VID）在这个子集上基本失败——GPT-4V 只有 5% SR。说明**光有大模型的常识推理不够，必须加导航领域的训练数据**。但一旦加了（LLaMA-VID-Nav vs NaVid），VLM 骨子的设计差异（两种 token、grid pooling、历史帧编码策略）开始显著拉开差距。

### 5.4 真实世界实验

- 平台：Turtlebot4 + Kinect DK 单目 RGB
- 4 个室内场景（Meeting_room / Office / Lab / Lounge）
- 200 条指令，分两类：简单地标跟随（25 条/场景）+ 复杂组合指令（25 条/场景）
- 成功率约 **66%**

论文没有逐个场景报数字，但给出了一个关键对比：**同时期需要深度+里程计的方法在 Sim-to-Real 迁移时 SR 下降超过 50%，而 NaVid 不存在这个问题——因为 RGB 在仿真和真实世界里是同一个模态。**

### 5.5 消融大结论

| 消融项 | 核心发现 |
|-------|---------|
| 当前帧 token 数 | 64 tokens 最优；太少 (2) 几何信息不足, 太多 (128) 增加延迟收益递减 |
| 历史帧 token 数 | 4 tokens/帧 最优；平衡了时序信息和计算成本 |
| Dagger 数据 | +5% SR 的增量贡献 |
| 导航数据量 | 280K 以下进步缓慢, 330K 起有明显跃升, 之后饱和 |

## 六、和后续工作的关系

NaVid 是第一个证明"VLM + 视频 + 纯 RGB 可以做 VLN 且泛化能力更强"的工作，但它的局限在后续工作中逐步被解决：

| NaVid 的局限 | 后续解决方案 |
|------------|------------|
| 动作空间是 VLN-CE 原语（FORWARD/TURN/STOP），不能部署在真实机器人上 | **NaVILA** 继承了 NaVid 的视频 VLM 思路，扩展到轮式+腿式底座并输出连续速度指令 |
| 推理慢（每步 1.2-1.5 秒） | 量化加速（ABot-N0 / Qwen-RobotNav 的 TensorRT 部署）；小模型蒸馏 |
| 只在 R2R 上训练，导航域窄 | Qwen-RobotNav 扩展为 15.6M 样本覆盖 5 大任务族；ABot-N0 扩展为 16.9M + 5 大任务 |
| 单一 RGB 输入缺乏深度/几何约束的辅助 | 后续工作大多在 planner 层加 LIDAR 安全护栏或 Neural Controller |
| 纯仿真训练，长程任务缺少 Agent | ABot-N0 引入 Planner + Topo-Memory + Self-Reflector 闭环 |

## 七、对后续论文设计者的启示

1. **VLM 的预训练知识是有用的——但不是免费的。** GPT-4V 在 VLN 上直接崩（5% SR），必须经过导航域训练。骨子的大模型提供了更好的 start point，但不能替代 domain-specific data。

2. **输入简洁就是泛化。** NaVid 最大的贡献不是任何一个 benchmark 数字，而是证明了"扔掉深度、里程计、地图"本身就能带来更好的泛化。后续 Qwen-RobotNav 和 ABot-N0 虽然在架构上有很大改进，但"减少输入依赖"这个方向是一致的。

3. **自然语言动作输出是一种"穷人版 VLA"。** NaVid 通过正则表达式把 VLM 的语言输出解析成动作，绕过了专门训练一个 action head 的需要。这在当时是巧妙的，但现在 Qwen-RobotNav 和 ABot-N0 都证明了——训一个专用的 action head 在精度和延迟上都更好。

4. **Dagger 对于恢复行为很重要，但纯仿真扩展有限。** NaVid 用 Dagger 在 R2R 训练集上采了 180K 非最优轨迹，SR 提升了 5%。但 ABot-N0 的 Agentic 系统展示了更根本的解法——在真实世界用 Self-Reflector + Planner 重规划来处理失败，而不是只靠仿真里的数据扩充。
