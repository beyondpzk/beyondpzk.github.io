MAR (Diffusion Loss)

paper link

高级深度学习与生成模型研讨课讲义：连续空间下的自回归图像生成与扩散损失 (Diffusion Loss)

核心文献： Autoregressive Image Generation without Vector Quantization (Tianhong Li, Yonglong Tian, He Li, Mingyang Deng, Kaiming He)

模块一：引言与背景梳理 (Setting the Stage)

在进入本期研讨课的核心内容之前，我们需要对当前的视觉生成模型范式进行一次全景式的回顾。近年来，图像生成领域主要由两大流派主导：扩散模型 (Diffusion Models)（如 DDPM, Latent Diffusion/Stable Diffusion）和自回归模型 (Autoregressive Models)（如 VQGAN, DALL-E, 乃至近期的 LlamaGen 等）。

1.1 传统自回归图像生成的“执念”：离散化与向量量化 (VQ)

在自然语言处理 (NLP) 领域，自回归模型（如 Transformer/GPT）取得了统治地位。语言天生是离散的（单词或子词组合），因此，语言模型在预测下一个 Token 时，本质上是在解决一个多分类问题。模型输出一个 Logits 向量，通过 Softmax 转化为概率，然后使用交叉熵损失 (Cross-Entropy Loss) 进行优化。

受到 NLP 成功的启发，研究者们在将自回归模型迁移到图像生成时，形成了一个根深蒂固的思维定势：自回归模型必须作用于离散的 Token 空间。

典型的传统流程如下：

  1. 训练分词器 (Tokenizer)：使用 VQ-VAE 或 VQGAN，将连续的图像像素压缩成一个低维特征图，然后通过查表（Codebook matching），强制将连续特征映射为有限词表（如 $K=8192$）中的离散索引 (Discrete Tokens)。
  2. 序列建模：将这些离散索引展平成一维序列，训练一个 Transformer 以预测下一个索引的概率分布。

1.2 向量量化 (Vector Quantization) 的痛点

尽管 VQ + AR 的范式取得了显著成果，但向量量化本身存在不可忽视的缺陷：

  1. 信息有损压缩：强制的离散化不可避免地会丢失连续特征中的细粒度信息，导致重构图像的细节模糊或出现伪影。实验表明，连续值自编码器（如 KL-regularized Autoencoder）的重构质量远超离散的 VQ-VAE。
  2. 优化困难 (Gradient Approximation)：量化操作 (Argmin) 是不可导的，通常需要使用 Straight-Through Estimator (STE) 来近似梯度。这使得 VQ Tokenizer 的训练极度敏感且不稳定，容易出现 Codebook 崩塌（部分 Token 从未被使用）等问题。
本课核心反思： 自回归建模的本质是“基于历史信息预测未来信息”——即条件概率 $p(x_i x_{<i})$。这一数学本质与变量 $x_i$ 是离散还是连续毫无关系。我们之所以依赖 VQ，仅仅是因为在离散空间中计算分类交叉熵非常方便。如果我们能找到一种优雅的方式来建模连续变量的条件概率分布，我们就能彻底抛弃向量量化！

这正是何恺明团队这篇论文的切入点：提出一种名为 Diffusion Loss（扩散损失） 的机制，使得序列模型能够直接在连续值空间中进行自回归生成。

模块二：理论基础与核心创新——Diffusion Loss

既然要放弃离散词表，我们就必须回答一个核心数学问题：给定前序连续 Token 提供的信息，如何建模下一个连续 Token 的任意复杂概率分布？

2.1 重新审视离散 Token 与分类分布

在离散设定下，设 $x$ 为要预测的下一个目标 Token（取值范围 $0 \leq x < K$）。自回归模型输出一个 $D$ 维向量 $z \in \mathbb{R}^D$（这是对前序上下文的编码总结）。随后，通过一个权重矩阵 $W \in \mathbb{R}^{K \times D}$ 进行线性映射，概率分布被建模为： \(p(x|z) = \text{softmax}(Wz)\) 这里有两个关键组件：

  1. 损失函数：度量估计分布与真实分布差异的交叉熵损失。
  2. 采样器 (Sampler)：在推理阶段，可以通过温度系数 $\tau$ 调节的多项式采样：$p(x z) = \text{softmax}(Wz / \tau)$。

2.2 走向连续空间：引入局部扩散过程

在连续空间中，$x \in \mathbb{R}^d$ 是一个连续向量。我们要建模 $p(x z)$。高斯混合模型 (GMM) 是一个选项（如前沿工作 GIVT 所尝试的），但其表达能力受限于预设的混合数量。

论文提出，去噪扩散概率模型 (DDPM) 已经被证明是拟合高维任意分布的最强工具。因此，作者巧妙地将扩散过程嵌套入自回归的每一个 Token 预测中。

注意这里的概念转变：在典型的 Latent Diffusion Model (LDM) 中，扩散过程用于联合建模所有 Token 的全局分布；而在本方法中，扩散过程仅仅用于建模单个 Token 的局部条件分布 $p(x z)$。

2.3 Diffusion Loss 的数学表达

基于 DDPM 的标准范式，目标条件分布 $p(x z)$ 的损失函数被定义为一个去噪准则：
\[L(z, x) = \mathbb{E}_{\epsilon, t} \left[ \|\epsilon - \epsilon_\theta(x_t | t, z)\|^2 \right]\]

我们来详细拆解这个公式中的物理与数学含义：

  • $x$：当前位置的真实连续值 Token（目标值）。
  • $z$：自回归网络（Transformer）对前序上下文编码后输出的条件向量。它包含了“网络对当前位置应该出现什么内容的预测和理解”。
  • $\epsilon \sim \mathcal{N}(0, I)$：采样的标准高斯噪声。
  • $t$：扩散过程的时间步（Noise Schedule 的步数）。
  • $x_t$：向真实目标 $x$ 添加噪声后的加噪版本，定义为 $x_t = \sqrt{\bar{\alpha}_t}x + \sqrt{1 - \bar{\alpha}_t}\epsilon$。
  • $\epsilon_\theta$：一个小型的去噪网络（参数为 $\theta$，在本文中具体为一个轻量级 MLP）。该网络的任务是：在条件 $z$ 的指导下，从加噪的 $x_t$ 中预测并剥离出噪声 $\epsilon$。

深刻理解 $z$ 的作用： 在这个公式中，$z = f(x_1, …, x_{i-1})$，它是自回归主干网络 $f(\cdot)$ 的输出。计算 $L(z, x)$ 的梯度时，梯度不仅会更新微型 MLP $\epsilon_\theta$ 的参数，还会反向传播回向量 $z$，进而更新整个自回归 Transformer 的参数。 换言之，这个小型的扩散模型在概念上充当了一个“高阶的、可参数化的损失函数”，用来评判 Transformer 输出的 $z$ 的质量。这就是它被称为 Diffusion Loss 的原因。

2.4 推理阶段的连续采样器 (Continuous Sampler)

有了损失函数进行训练，推理时我们需要对应的采样器从 $p(x|z)$ 中生成具体的连续 Token $x$。 在给定条件 $z$ 后，我们从纯高斯噪声 $x_T \sim \mathcal{N}(0, I)$ 开始，执行逆向扩散过程：

\[x_{t-1} = \frac{1}{\sqrt{\alpha_t}} \left( x_t - \frac{1-\alpha_t}{\sqrt{1-\bar{\alpha}_t}} \epsilon_\theta(x_t|t, z) \right) + \sigma_t \delta\]
其中 $\delta \sim \mathcal{N}(0, I)$，$\sigma_t$ 是对应时间步的噪声水平。最终得到的 $x_0$ 便是服从目标分布 $p(x z)$ 的生成 Token。

温度采样 (Temperature Sampling) 的连续版本： 自回归生成中，调节温度参数 $\tau$ 以平衡生成的多样性 (Diversity) 和保真度 (Fidelity) 是必不可少的。对于扩散采样器，作者采用了基于分数的缩放技巧。从本质上讲，调整温度意味着从重整化分布 $p(x|z)^{\frac{1}{\tau}}$ 中采样。在实践中，作者通过将采样公式中的噪声注入项 $\sigma_t \delta$ 缩放为 $\tau \sigma_t \delta$ 来实现这一点。$\tau$ 控制了采样过程中的随机性方差。

模块三：模型架构深度剖析 (MAR 框架)

除了 Diffusion Loss，本文的另一大贡献是将传统的因果自回归模型 (Causal AR) 和掩码生成模型 (Masked Generative Models, 如 MaskGIT, MAGE) 统一到了一个通用的掩码自回归 (Masked Autoregressive, MAR) 框架中。我们对整个系统的结构进行解构。

3.1 突破因果掩码的限制：从 AR 到 MAR

传统的 AR 模型（比如按从左上到右下的光栅扫描顺序 Raster-scan）使用严格的下三角因果掩码 (Causal Mask)。即第 $i$ 个 Token 只能看到 $1$ 到 $i-1$ 的 Token。

为什么要改？ 图像没有明确的时间先后顺序，光栅扫描不仅低效，而且强迫模型忽略了图像的二维空间双向依赖。

MAR 的核心设计： MAR 仍然保持了“基于已知的 Token 预测未知的 Token” 这一自回归本质，但它允许：

  1. 随机预测顺序：不再局限于从左上到右下，而是随机掩码。
  2. 双向注意力 (Bidirectional Attention)：借用 MAE (Masked Autoencoder) 的架构思想来实现。

3.2 详细网络结构：数据流向全解析

整个网络由三大核心组件构成。我们以生成 $256 \times 256$ 图像为例进行推演。

组件 A：连续值分词器 (Continuous-Valued Tokenizer)

  • 结构：采用类似 Latent Diffusion Models (LDM) 中的 KL-regularized Autoencoder（例如文中使用的 KL-16 Tokenizer）。
  • 输入：$256 \times 256 \times 3$ 的 RGB 图像。
  • 处理：经过 Encoder 下采样 16 倍，输出特征图大小为 $16 \times 16 \times d$（其中 $d$ 为通道数）。
  • 输出：我们将这 $16 \times 16$ 个空间位置展平，得到 $256$ 个连续值 Token，每个 Token 是一个 $d$ 维向量。注意：这里不进行任何 VQ 量化！

组件 B：掩码自回归主干 (MAR Transformer)

采用非对称的 MAE 风格的 Encoder-Decoder 结构：

  • 掩码策略：在训练阶段，从 256 个连续 Token 中随机选择一部分作为“掩码 (Mask)”，剩下的作为“已知 (Visible)”。
  • Encoder：只接收“已知 Token”加上它们的位置编码 (Positional Embeddings)，进行全注意力 (Full Attention) 计算。这使得已知区域的 Token 之间可以充分交流，不受因果限制。
  • Decoder：接收 Encoder 的输出，并与“掩码 Token”的占位符（同样加上了目标位置的位置编码）拼接。在 Decoder 内部，掩码 Token 可以通过交叉/自注意力机制看到所有已知 Token 提供的高级语义特征。
  • 输出条件向量 $z$：Decoder 最终在掩码位置上输出 $D$ 维向量序列，即我们前文定义的条件向量 $z_i$。

组件 C：扩散损失预测头 (Diffusion Head / Denoising MLP)

这是一个极轻量级的多层感知机 (MLP)。

  • 输入：
    1. 被加噪的真实连续 Token $x_t \in \mathbb{R}^d$。
    2. 时间步 $t$（映射为 Time Embedding）。
    3. Transformer 在对应位置输出的条件向量 $z_i \in \mathbb{R}^D$。
  • 结构：通常仅由 3 个 MLP 模块组成，通道宽度约为 1024。在 MAR-L 配置下，这个头部的参数量仅为约 21M（相对于 Transformer 主干 400M 的参数量，增加计算开销不到 5%）。
  • 计算机制：在训练时，对于一张图像的 $z$，系统会随机采样 4 次不同的时间步 $t$，以充分利用前馈计算出的条件 $z$，提高优化效率。然后执行均方误差回归，计算梯度的同时更新 MLP 和 Transformer。

3.3 推理阶段的并行解码 (Parallel Decoding)

在推理（生成）阶段，MAR 展现了极大的效率优势：

  1. 初始状态：输入一个全掩码序列（256 个占位符）。
  2. 迭代预测：模型不需要像传统 AR 那样进行 256 步串行预测。它可以分步进行（例如 64 步）。在每一步中，Transformer 结合当前已知 Token 并行输出多个掩码位置的条件向量 $z$。
  3. 局部扩散生成：对于这些选定的位置，微型扩散 MLP 根据对应的 $z$ 并行运行 100 步逆向采样，生成对应的连续 Token 值。将这些新生成的 Token 填入已知序列中，进行下一轮迭代。
  4. 图像解码：得到完整的 $16 \times 16$ 连续 Token 后，送入 KL-Autoencoder 的图像 Decoder，还原回 $256 \times 256$ 的 RGB 图像。

这种机制打破了序列生成必须一步一 Token 的性能瓶颈。

模块四：实验论证与深入分析 (Empirical Analysis)

论文通过大量详实的消融实验证明了机制的有效性。我们需要重点解读以下几个核心实验。

4.1 Diffusion Loss 对比交叉熵损失 (Cross-Entropy)

实验控制变量：相同的 Transformer 规模 (MAR-L, ~400M)，相同的训练 Epoch。

  • 使用 VQ-16 Tokenizer + Cross-Entropy (随机掩码顺序，双向注意力)： FID = 3.69
  • 使用 KL-16 Tokenizer + Diffusion Loss (相同架构设定)： FID = 1.98

分析：FID（Fréchet Inception Distance）越小图像质量越高。Diffusion Loss 使得生成质量取得了飞跃（FID 下降约 50%）。原因有二：

  1. 上限更高：KL-16 连续 Tokenizer 本身的重构误差 (rFID 约为 1.20~1.43) 远小于离散 VQ-16 (rFID=5.87)。连续空间保留了更多高频细节。
  2. 分布建模更优：扩散模型拟合高维复杂分布的能力强于离散的分类分布。

4.2 极高的 Tokenizer 灵活性 (Flexibility)

因为不依赖分类词表，Diffusion Loss 理论上可以对接任何连续值表征。论文展示了惊人的兼容性：

  • 兼容 VQ Tokenizer：在 VQ 步骤之前截取其连续的 Latent 特征进行学习，同样获得了比直接在量化后做 Cross-Entropy 更好的效果 (FID 3.64 vs 3.69)。这反证了 VQ 操作本身的破坏性。
  • 兼容步幅不匹配的 Tokenizer：针对步幅为 8 (输出 $32 \times 32$ 序列) 的 KL-8 Tokenizer，作者直接将空间上 $2 \times 2$ 的连续 Token “打包拼接”成一个更粗粒度的长序列，在不增加序列长度的情况下实现了 FID 2.05 的优异表现。
  • 兼容异构架构：连 OpenAI 的 Consistency Decoder（并非设计给自回归使用的非 VQ 架构）也能直接拔插使用。

4.3 效率与精度的完美权衡 (Speed/Accuracy Trade-off)

在当前的生成模型领域，效率是一个痛点。让我们将本方法（MAR + Diff Loss）与当红炸子鸡 DiT (Diffusion Transformer) 进行对比。

  • DiT 范式：在 Latent 空间全局做扩散。每一级去噪步（假设 250 步），巨大的 Transformer 网络（如 DiT-XL 675M 参数）都需要进行完整的前向计算。推理极慢。
  • MAR 范式：Transformer 主干仅执行少量的自回归步数（例如 64 次迭代）。每次迭代确定的条件向量 $z$ 后，需要运行 100 步扩散的是那个只有 21M 参数的极小 MLP。

结论：在生成 256x256 图像时，MAR-L 以单图 <0.3 秒 的推理时间，达到了极高的图像保真度。相比之下，传统的全局扩散模型在同等质量下往往需要数秒时间。

4.4 其他重要消融发现

  • 局部 MLP 的大小：MLP 极小（如 2M 参数，宽度 256）时模型依然工作正常（FID 2.45）。增加到 21M（宽度 1024）能进一步稳定提升质量。这证明了 Diffusion Loss 主要利用了 Transformer 提取的高阶语义，去噪过程本身由于作用空间狭小（针对单 Token），不需要极深的网络。
  • CFG 的作用：Classifier-Free Guidance 在 Diffusion Loss 中同样适用且至关重要。训练时将 10% 的类别标签设为空；推理时将有条件预测与无条件预测的外插作为最终梯度。这显著提升了生成多样性与指令服从性（IS 指数大幅提升）。

模块五：总结、局限性与开放问题

5.1 本质理念的升华 (Takeaways)

这篇论文最大的贡献不仅仅是刷高了 FID，而是提供了一种模块化解耦的哲学：

  • Token 之间的宏观关联 (Inter-token dependence)：交由擅长处理长序列、捕捉全局注意力依赖的 Transformer (自回归建模) 来处理。
  • Token 内部的微观分布 (Per-token distribution)：交由擅长拟合复杂高维数据概率密度的 Diffusion Model 来建模。

这种将自回归框架与扩散局部化相结合的思路，彻底解开了视觉生成领域长久以来“序列模型绑定离散 VQ”的枷锁。

5.2 局限性探讨 (Limitations)

尽管模型在 ImageNet 上取得了当前 Token-based 模型的顶尖结果 (MAR-H FID 1.55)，但我们作为研究者必须客观审视其局限性：

  1. 学术数据集的偏置：ImageNet 图像多为中心主体的物体类别。在零样本或开放域 (Open-domain/Text-to-Image) 场景下的扩展性如何？目前尚需海量非结构化图文数据的验证。
  2. 不可忽视的伪影 (Artifacts)：对于复杂场景的生成，模型偶尔会产生局部结构不合理、纹理拼凑的伪影。这可能是局部扩散未能完全融合全局上下文，也可能是自编码器的固有缺陷。
  3. 对 Tokenizer 的深度依赖：本框架无法脱离预训练的连续值 Tokenizer。Tokenizer 压缩比的上限决定了生成质量的上限。未来是否可以直接在原像素空间 (Pixel Space) 做 MAR + Diff Loss？（论文附录 D.1 初步尝试了 $64 \times 64$ 像素空间的直接建模并取得了成功，但计算代价过于庞大，属于有待攻克的难点。）

详细的生成过程

首先tokenizer就是 KL-regularized 的VAE. 先说训练的过程,

  1. 输入一个img, 得到对应的latent: z; z是(16, 16, d), 即256个tokens;
  2. 随机选择一部分tokens进行mask, 标记为位置, 用transformer来训练,逻辑是用看得到tokens来预测看不到的tokens;
  3. 对于某个位置的token,它经过transformer的输出记为y,
  4. 然后经过一个小的MLP,用来学习去noise,它的输入是 带noise的y, 这一步用的是DDPM的逻辑;
  5. 这样最终loss就能够回传回来

推理的时候有一些像maskgit, 不断地填充.
