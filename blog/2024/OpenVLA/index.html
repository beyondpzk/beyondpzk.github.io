深入解析 OpenVLA：开源视觉-语言-动作大模型

一、 引言与研究背景

在当前的具身智能（Embodied AI）研究中，如何让机器人像人类一样具备强大的泛化能力，能够处理未见过的物体、场景并听懂多样的自然语言指令，是核心痛点之一。过去的机器人策略往往从零开始训练（From-scratch Imitation Learning），存在数据依赖性强、泛化能力弱的问题。

近期，业界开始将互联网规模数据上预训练的视觉-语言模型（VLMs，如 CLIP, Llama, SigLIP）引入机器人领域，构建了视觉-语言-动作模型（VLA, Vision-Language-Action Models）。然而，诸如谷歌的 RT-2-X 等前沿 VLA 模型完全闭源，且参数量巨大（55B），严重阻碍了学术界对 VLA 的进一步探索。

本讲义将重点剖析斯坦福大学与加州大学伯克利分校等机构联合推出的 OpenVLA。这是一个 70 亿参数（7B）的完全开源 VLA 模型，不仅在性能上超越了参数量大其七倍的闭源模型 RT-2-X，更为学术界提供了一套完整的 VLA 架构设计、高效微调与量化部署的标准化范式。

二、 OpenVLA 的核心学术与工程贡献

作为本领域的重要基石，OpenVLA 提出的核心贡献可以总结为以下四个维度：

  • 确立了开源 VLA 的性能新标杆：模型在涵盖多种机器人形态的 970,000 条真实世界机器人演示数据（Open X-Embodiment 数据集）上进行训练。在 29 个跨形态测试任务中，其绝对任务成功率超越了 55B 参数的 RT-2-X 达 16.5%。
  • 创新的混合视觉表示学习（Fused Vision Representation）：在网络架构上，摒弃了单一的视觉编码器，创造性地融合了 DINOv2（低级空间/几何特征）与 SigLIP（高级语义特征），显著提升了机器人的空间推理和细粒度操作能力。
  • 验证了参数高效微调（PEFT）在 VLA 中的可行性：首次系统性地证明了低秩适配（LoRA, Low-Rank Adaptation）技术在 VLA 上的有效性。仅微调 1.4% 的参数即可在全新任务上达到与全参微调（Full Fine-tuning）相媲美的效果，使得在消费级 GPU 上适配 VLA 成为可能。
  • 解决了 VLA 的高显存推理瓶颈：通过 4-bit 量化（Quantization）技术，将 7B 模型的显存占用从 15GB（bfloat16）压缩至 7GB，并在真实机器人闭环控制中保持了零性能折损。

三、 核心网络结构深度剖析

OpenVLA 的整体架构基于 Prismatic-7B VLM 主干进行深度改造，采用端到端（End-to-End）的自回归生成范式。网络主要由三大部分构成：双路视觉编码器、跨模态投影层，以及大型语言模型基座。

1. 双路混合视觉编码器 (Dual Fused Vision Encoder)

机器人的操作既需要理解“这是什么”（语义），又需要精确感知“它在哪里、姿态如何”（空间几何）。传统的 VLM 多采用 CLIP 或 SigLIP，偏重于语义特征，而牺牲了空间细粒度。

  • 结构设计：OpenVLA 采用了并行的双路视觉编码器。
    • SigLIP：负责提取全局的高级语义特征，理解自然语言指令与图像内容的对应关系。
    • DINOv2：基于自监督学习（Self-Supervised Learning），不依赖文本标签，能够提取极具判别力的低级空间特征和物体边缘/几何结构信息。
  • 特征融合：将输入图像（分辨率设为 224×224 像素）分别输入到这两个编码器中，得到两个 Patch Embeddings 序列。随后，网络在通道维度（Channel-wise）将这两个特征进行拼接（Concatenation）。
  • 关键训练发现：在标准 VLM 训练中，视觉编码器通常是冻结（Frozen）的。但 OpenVLA 的消融实验表明，对于 VLA 而言，解冻并微调视觉编码器是极其关键的。预训练的视觉主干可能缺乏对场景中控制相关细节（如微小缝隙、物体姿态）的敏锐度，必须通过端到端的动作监督梯度来更新。

2. 跨模态投影层 (Vision-Language Projector)

  • 功能：弥合视觉特征空间与 LLM 文本嵌入空间之间的模态鸿沟（Modality Gap）。
  • 结构：采用了一个简单但高效的 两层多层感知机（2-layer MLP）。通过非线性映射，将拼接后的高维视觉 Token 转换到 Llama 2 的输入维度。这部分结构保持轻量，使得主要的处理算力集中在语言模型中。

3. 语言模型基座 (LLM Backbone)

  • 选型：采用了 Llama 2 7B 作为核心的大型语言模型。
  • 输入序列重构：LLM 接收的输入序列是拼接好的上下文：[System Prompt/Instruction Tokens] + [Projected Vision Tokens] + [Action Tokens]。
  • 采用“图像块即词元”（Patch-as-Token）的理念，将视觉信息作为一种特殊的外语词汇送入 Transformer 中处理。

4. 动作离散化与词表重构 (Action Discretization & Tokenizer Overwriting)

这是 VLA 将机器人连续控制问题转化为自然语言处理任务（Next-token prediction）的核心机制。

“To enable the VLM’s language model backbone to predict robot actions, we represent the actions in the output space of the LLM by mapping continuous robot actions to discrete tokens used by the language model’s tokenizer.”

  • 分位数离散化 (Quantile-based Discretization)：机器人输出通常是连续的 7 自由度（7-DoF）动作（末端执行器的 6-DoF 相对位姿 + 1-DoF 夹爪开合度）。OpenVLA 采用独立的分箱策略，将每个动作维度离散化为 256 个 Bin（区间）。
  • 剔除离群值：为防止数据中的极端离群值导致有效动作区间被过度压缩，区间边界未采用单纯的 Min-Max，而是取训练数据动作分布的 1% 分位数到 99% 分位数 进行均匀划分。
  • 词表替换策略：Llama 2 的默认 Tokenizer 只预留了较少的“特殊标记”。为解决 256 个动作 Bin 的词表扩充问题，OpenVLA 直接采用了一种极简策略：覆盖 Llama 词汇表中最不常用的最后 256 个 Token。这意味着网络实际上是在输出这 256 个“生僻字”，随后通过反向映射（De-Tokenizer）将其解码为连续的机械臂动作。
  • 损失函数设计：与标准的 Causal Language Modeling 相同，使用标准的交叉熵损失（Cross-Entropy Loss）。需要特别注意的是，Loss 仅对预测的动作 Token（Action Tokens）进行计算，不对输入的图像和指令 Token 计算梯度。

四、 核心实验发现与系统部署策略

在设计和部署具身大模型时，如何在有限算力下进行微调适配（Adaptation）是工程界极其关注的问题。OpenVLA 在此给出了详尽的实证参考。

1. 微调策略：三明治微调与 LoRA

当我们在全新的机器人硬件或新任务上部署 OpenVLA 时，如何进行参数更新？

  • Frozen Vision（冻结视觉层）与 Last layer only（仅微调最后一层）：实验证明这两种方式性能极差。机器人换了新环境后，必须重新学习视觉特征。
  • Sandwich Fine-tuning（三明治微调）：解冻视觉编码器、Token Embedding 矩阵和最后一层，冻结 LLM 主体。此方法能够在节省显存的同时获得尚可的性能。
  • LoRA (Rank=32)：对网络中所有的线性层注入低秩旁路。结果表明，LoRA 是性价比最优的微调策略，仅更新 1.4% (约 97.6M) 的参数量，即可在下游任务上达到与全参微调（7B）完全一致的成功率（约 68.2%），且单张 A100 GPU 在十余小时内即可完成微调。

2. 量化推理 (Quantized Inference)

具身模型需要在机器人端侧进行低延迟、高频率的控制（实时性要求）。

  • bfloat16 (半精度)：基础设置，需要 16.8 GB 显存，可在单张 RTX 4090 上以 ~6Hz 运行。
  • 8-bit 量化：由于增加了量化算子的开销，导致推理频率大幅下降（在某些测试机型上降至 1.2Hz）。控制频率的下降直接破坏了非阻塞系统中的动力学一致性，导致了任务成功率的暴跌。
  • 4-bit 量化 (int4)：最优解。由于减少的显存带宽开销抵消了量化计算开销，推理速度不仅没有下降，反而略有提升。最重要的是，4-bit 量化完美维持了与 bfloat16 相同的物理任务成功率（约 71.9%），但显存占用缩减至不到一半（7.0 GB）。

为什么要用两个Vit来提取feature

这是一个非常敏锐且切中要害的问题！在深度学习和具身智能的网络架构设计中，“为什么要增加模型的复杂度（引入双视觉主干）”以及“这种复杂度是否真的带来了收益”，永远是我们进行架构审视时必须要问的。

OpenVLA 确实做了只用一个 ViT 的消融实验。使用双 ViT 的核心逻辑在于“高级语义理解”与“低级空间几何感知”的互补性。

下面我为你详细拆解这背后的设计哲学以及论文中对应的实验支撑：

一、 为什么要用两个 ViT？（理论动机）

在传统的视觉-语言模型（VLM）中，大家通常只使用一个视觉编码器（比如 CLIP 或 SigLIP）。因为对于传统的 VQA（视觉问答）或图像描述任务，模型只需要知道“图里有什么”、“它们大概在什么位置”即可。

但在机器人操作（Robotic Manipulation）中，这种单一的视觉特征是远远不够的。OpenVLA 选择同时拼接 SigLIP 和 DINOv2 这两个 ViT 的特征序列，是为了解决机器人独有的痛点：

  1. SigLIP（负责“高级语义”）：
    • 机制：它是基于图文对齐（对比学习）训练出来的。
    • 优势：它能极好地理解人类的自然语言指令，知道什么是“红色的碗”、什么是“海绵”。它为机器人提供了强大的语义基础（Semantic Grounding）。
    • 劣势：对比学习往往会忽略图像中的底层纹理、物体边缘和细粒度的三维空间关系。
  2. DINOv2（负责“低级空间与几何”）：
    • 机制：它是基于自监督学习（如掩码图像建模、自蒸馏）训练的，完全没有文本标签的干预。
    • 优势：研究表明，DINOv2 能够在没有任何显式监督的情况下，学到极好的物体深度信息、边缘结构、零件级别的对应关系（Part-level features）。
    • 在机器人中的价值：机器人要抓取一个杯子，不仅要知道那是杯子，还必须极其精确地判断杯把手的空间姿态、边缘轮廓。DINOv2 恰好弥补了 SigLIP 在细粒度空间感知（Spatial Reasoning）上的短板。

因此，OpenVLA 采用“双剑合璧”的策略：把图像分别送入两个 ViT，然后将输出的特征在通道维度拼接（Concatenate）起来，让后面的语言模型既能“听懂话”，又能“看准形”。

二、 论文中做了单 ViT 的消融实验吗？（实验证明）

做了。 在论文的附录 D.2 节 (Dual vs. Single Vision Encoder Experiments) 中，作者专门针对这个问题设计了详尽的消融实验。

为了公平对比并排除海量数据带来的干扰，作者在小规模的 BridgeData V2 数据集上训练了两个变体模型进行对决：

  1. OpenVLA-Bridge（双 ViT Baseline）：保留了原有的双视觉编码器架构（SigLIP + DINOv2），但只在 Bridge 机器人数据集上训练。
  2. OpenVLA-Bridge-SigLIP（单 ViT 变体）：直接去掉了 DINOv2 编码器，视觉主干只剩下 SigLIP，同样只在 Bridge 数据集上训练。

核心实验数据对比：

在包含视觉泛化、物理泛化、语言定位等 8 个极具挑战性的下游任务测试中：Ablation

  • OpenVLA-Bridge（双 ViT）：平均任务成功率为 45.6%
  • OpenVLA-Bridge-SigLIP（单 ViT）：平均任务成功率下降至 40.6%

教授视角的实验解读：

从这 5% 的绝对成功率下降中，我们可以得出几个重要的学术结论：

  1. DINOv2 确实有效：单靠 SigLIP 这种图文对比学习模型，在处理需要精细物理交互的泛化任务（如将胡萝卜精确放入盘子中、举起细小的电池等）时显得力不从心。加上 DINOv2 后，底层空间特征确实帮助模型做出了更精准的动作决策。
  2. 但它的影响权重次于“海量数据预训练”：作为对比，如果去掉 Open X-Embodiment (OpenX) 这个包含 97 万条轨迹的超大规模先验数据集，模型的成功率会从 76.3% 暴跌 30% 直接掉到 45.6%。Ablation 这告诉我们，在当前的具身智能范式下，“数据多样性”提供的收益仍然远大于“模型架构微调（如增加一个 ViT）”带来的收益。

三、 思考

虽然双 ViT 架构在 OpenVLA 中被证明是成功的，但在工程落地时，这不可避免地带来了计算代价：

  • 两个 ViT 意味着在处理每帧图像时，前端的计算量翻倍，同时输入给大语言模型（LLM）的视觉 Token 维度也变得更长，这会增加推理延迟。

思考： 如果我们要将这个模型部署到算力极度受限的小型无人机或机器狗上，我们是否可以通过知识蒸馏（Knowledge Distillation）的技术，将“包含高级语义和低级几何的双 ViT 知识”，压缩到一个单一的、轻量级的 ViT 中？这正是目前学术界（如高效多模态表征学习领域）非常前沿的研究方向。
