RDT1B

paper link

注意注意, Dit预测的不是noise,而是直接就是机器人的动作; 原因是动作是有范围的,比如[-1, 1] 从预测noise反推动作的话,会得到比较奇怪的值.

一、 训练样本的准备（The Raw Training Sample）

假设我们从重放缓冲区（Replay Buffer）中随机抽取了一条机器人执行“倒水”任务的真实人类演示轨迹。我们截取时间戳 $t$ 的一帧，构建出当前的前向传播样本。

这个 Raw Sample 包含以下五个核心部分：

  1. 语言指令 $\ell$：一段变长的自然语言，例如："Pour water from the bottle into the mug until about one-third."（把瓶子里的水倒进马克杯，直到大约三分之一满）。
  2. 图像历史序列 ${X_{t-T_{\text{img}}+1:t+1}}^{\text{img}}$：为了捕捉动态信息，模型需要过去的视觉上下文。设定 $T_{\text{img}} = 2$，系统包含 3 个摄像头（全局外部、左腕、右腕）。因此，输入是 $2 \times 3 = 6$ 张 RGB 图像。
  3. 当前本体感觉 $z_t$：记录当前 $t$ 时刻机器人的物理状态（如各个关节角度、末端位姿等）。
  4. 控制频率 $c$：一个标量，指示当前数据集的控制频率（例如 $15\text{Hz}$），用于对齐不同机器人的时间颗粒度。
  5. 目标动作块 $a_{t:t+T_a}$（Ground Truth）：从 $t$ 时刻开始，未来 $T_a = 64$ 步的真实连续动作序列。这是我们要让模型学会预测的目标标签（Label）。

二、 扩散前向过程（Diffusion Forward Process：构造网络输入）

在生成模型中，网络不直接接收干净的动作，而是接收被加噪的动作，并试图预测出干净的动作。

  1. 随机采样时间步：从 $1$ 到 $K$（总扩散步数，例如 100 步）中随机均匀采样一个时间步 $k$。假设我们抽到了 $k = 50$。
  2. 注入高斯噪声：从标准正态分布中采样一个与动作块维度完全一致的噪声张量 $\epsilon \sim \mathcal{N}(0, I)$。
  3. 构造含噪动作：根据扩散模型的闭式（Closed-form）加噪公式，计算第 $k$ 步的含噪动作序列 $\tilde{a}_{t:t+T_a}$： \(\tilde{a}_{t:t+T_a} = \sqrt{\bar{\alpha}_k} a_{t:t+T_a} + \sqrt{1 - \bar{\alpha}_k} \epsilon\) 其中 $\bar{\alpha}_k$ 是预先定义好的标量噪声调度系数。

注意：这里的 $\tilde{a}{t:t+T_a}$ 就是即将喂给神经网络的“残缺/含噪”的动作数据。此时，去噪网络 $f\theta$ 的任务是：看着被污染的动作 $\tilde{a}{t:t+T_a}$，结合指令 $\ell$ 和画面 $o_t$，还原出干净的动作 $a{t:t+T_a}$。

三、 多模态分词与编码（Multi-modal Tokenization）

由于 Transformer 只能处理 Token 序列（维度假设为 $D$），我们需要将上述异构的物理世界数据翻译入同一高维隐空间（Latent Space）。

1. 条件输入编码（Condition Encoding）

这部分提供上下文“指导”，它们不参与自回归或全量自注意力，而是作为 Cross-Attention 的 Key 和 Value。

  • 语言指令 $\ell$：送入冻结权重的 T5-XXL 语言模型，提取出变长的高维特征，再通过一个 MLP 投影到 $D$ 维。得到文本 Tokens 矩阵，形状为 $(N_{\text{text}}, D)$。
  • 图像序列 $X$：6 张图像送入冻结权重的 SigLIP 视觉大模型。为防止时空信息混乱，加上多维时空网格位置编码（区分哪张图是第几帧、哪个相机）。最后通过 MLP 投影，得到视觉 Tokens 矩阵，形状为 $(N_{\text{img}}, D)$。
  • 正则化技巧：在这一步，模型有 $10\%$ 的概率随机将文本或视觉 Token 替换为可学习的掩码（Mask），这是为了强迫模型不要在不同传感器之间“走捷径（Shortcut Learning）”。

2. 主序列输入编码（Main Sequence Encoding）

这是流经 Transformer 骨干的“主体数据”，它们将作为 Attention 的 Query。

  • 统一动作空间与掩码（Padding Mask）：本体感觉 $z_t$ 和含噪动作序列 $\tilde{a}_{t:t+T_a}$ 中的每一步，都被塞入预设的 128 维“物理可解释统一动作空间”。由于双臂机器人可能没有移动底座，这些缺失的维度补 $0$。为了区分“物理上的速度0”和“空缺补0”，再拼接一个 128 维的 0-1 掩码向量。于是，每个动作步变为 256 维向量。
  • MLP 投影与拼接：
    • $z_t$ (256维) $\xrightarrow{\text{MLP}}$ 1 个 Token
    • $\tilde{a}_{t:t+T_a}$ ($64 \times 256$维) $\xrightarrow{\text{MLP}}$ 64 个 Tokens
    • 控制频率 $c$ $\xrightarrow{\text{MLP}}$ 1 个 Token
    • 扩散时间步 $k$ $\xrightarrow{\text{MLP}}$ 1 个 Token
  • 这些特征在序列长度维度上拼接（Concatenation），形成长度为 $1 + 64 + 1 + 1 = 67$ 的主序列。
  • 最后加上一维的位置编码（Positional Embeddings），得到主序列 Tokens 矩阵，形状为 $(67, D)$。

四、 Transformer 骨干网络（The DiT Backbone）

现在，形状为 $(67, D)$ 的主序列将依次穿过 $L$ 层的 DiT Block。每一层 Block 内部发生着极其复杂的交互，这是 RDT-1B 架构的灵魂所在。

针对每一个 DiT Block，数据流经如下三个子模块：

1. RMSNorm 与 自注意力层（Self-Attention）

  • 首先，$(67, D)$ 的张量通过 RMSNorm。同学们要记住，这里不用标准的 LayerNorm，因为 LayerNorm 的均值去中心化会破坏机器人时序数据（比如动作轨迹）的物理对称性。
  • 接着进行自注意力计算（Self-Attention）。在这个过程中，当前的动作、历史本体感觉、频率和时间步互相交换信息，建立时序一致性。
  • 为了防止在 12 亿参数规模下计算 Attention Score 时梯度爆炸，我们在计算 Query 和 Key 点积之前，强制插入了 QKNorm。

2. 交替条件注入（Alternating Condition Injection, ACI）- 核心创新！

经过自注意力层后，主序列需要向图像和文本“寻求指导”。模型通过交叉注意力（Cross-Attention）来实现。主序列 $(67, D)$ 作为 Query。 但在传统的模型中，几十个文本 Token 往往会被数千个高分辨率视觉 Token 完全淹没。RDT 是怎么做的？

  • 如果是奇数层（Odd Layers）：主序列只与视觉 Tokens $(N_{\text{img}}, D)$ 计算 Cross-Attention。模型在这一层专心“看图”，理解空间几何与障碍物。
  • 如果是偶数层（Even Layers）：主序列只与文本 Tokens $(N_{\text{text}}, D)$ 计算 Cross-Attention。模型在这一层专心“听话”，核对当前动作是否符合“倒三分之一水”的语义约束。 这种“交替注入”极其优雅地解决了多模态信息密度不平衡的灾难。

3. 前馈神经网络（FFN）与残差连接

经过交叉注意力后，张量通过全连接的 FFN 层进行非线性特征升维再降维，并辅以残差连接（Residual Connection）加到输入张量上，输出给下一层。

经过 $L$ 层这样的提纯和加工，网络深刻理解了“在给定的指令和画面下，第 $k$ 步的噪声应该长什么样，干净动作应该长什么样”。

五、 解码输出（Decoding to Physical Action）

从第 $L$ 层 DiT Block 出来的张量形状依然是 $(67, D)$。

  1. 提取动作位点：我们丢弃掉对应 $z_t$、$c$、$k$ 的 3 个 Token，只提取对应未来动作的那 64 个 Tokens（形状为 $64 \times D$）。
  2. MLP 解码器（MLP Decoder）：传统 DiT 在这里只用一个简单的线性层（Linear Layer）将 $D$ 维降至物理维度。但是，机器人面对的摩擦力、刚体碰撞等物理动态具有极高的非线性。因此，RDT 在这里使用了一个深层的 MLP Decoder，将其非线性映射回统一物理动作空间。
  3. 网络预测值：网络最终输出的是形状为 $64 \times 128$ 的张量。这就是去噪网络 $f_\theta$ 对未来 64 步干净动作的预测值，我们记作 $\hat{a}_{t:t+T_a}$。

(注：在一些早期的扩散模型中，网络预测的是注入的噪声 $\epsilon$，但在连续控制领域，由于动作通常是有界的物理量，现代策略如 Chi et al. (2023) 证明，直接预测原始的干净样本 $a_0$ 表现更稳定且更好约束物理边界。RDT 采用的正是直接预测动作本身。)

六、 损失函数的计算与反向传播（Loss Calculation）

到现在为止，我们在前向传播中获得了一个极其宝贵的输出：网络基于含噪输入，竭尽全力还原的干净动作预测值 $\hat{a}_{t:t+T_a}$。

而我们在第一步（The Raw Sample）中，手里正握着真实的物理世界标签——人类专家在 $t$ 到 $t+64$ 步内实际打出的动作绝对真理：$a_{t:t+T_a}$。

所以，损失函数（Loss Function）的计算大道至简。针对这个特定样本，我们计算预测动作块与真实动作块之间的均方误差（Mean Squared Error, MSE）：

\[\mathcal{L}(\theta) := \text{MSE}\left(a_{t:t+T_a}, \hat{a}_{t:t+T_a}\right)\]

或者写成论文中期望的形式： \(\mathcal{L}(\theta) := \mathbb{E}_{k, \epsilon, (\ell, o_t, a)} \left[ \left\| a_{t:t+T_a} - f_\theta\left(\ell, o_t, \sqrt{\bar{\alpha}_k}a_{t:t+T_a} + \sqrt{1 - \bar{\alpha}_k}\epsilon, k\right) \right\|_2^2 \right]\)

反向传播： 基于这个标量 Loss 梯度将沿着 MLP 解码器 $\rightarrow$ $L$ 层 DiT Block (及其内部的 RMSNorm, QKNorm, ACI) $\rightarrow$ 状态编码 MLP 一路反传（Backpropagation）。 请注意，由于视觉的 SigLIP 和文本的 T5-XXL 都是冻结的（Frozen），梯度传到它们那里就停止了，这样为我们节省了极其可观的显存开销。

总结回顾

同学们，我们刚刚完整走完了 RDT-1B 内部的生命周期。从一段带噪声的、高频波动的、甚至是异构双臂的生数据，经过 Padding 掩码机制 转化为统一格式；在 DiT 骨干 内部，通过 RMSNorm 保持时序物理特征，通过 交替注意力（ACI） 均衡了庞大视觉画面和微小文本指令的话语权；最后通过 MLP 解码器 还原出具备高度非线性的物理动作。

正是这样一层层针对机器人物理世界特性的严谨工程改造，才让单纯的 Diffusion Model 在具身智能领域真正落地生根，并展现出了对未见物体和复杂精细操作的惊人零样本泛化能力。

大家可以梳理一下刚才推导的张量维度变化，有不清楚的地方我们随时探讨。

整体结构理解

非常好！这位同学，你的技术直觉非常敏锐，能够迅速在庞杂的网络结构中抓住了它的骨架。

你对大框架的理解完全正确：RDT-1B 的底层生成范式，确实就是标准的 DDPM（Denoising Diffusion Probabilistic Models）逻辑。 它接收带噪声的动作序列作为主要输入，并以视觉图像和自然语言指令作为条件（Condition）来引导去噪过程。

但是！在这里，我必须纠正你总结里的一个极其关键的技术偏差。这也是很多刚从计算机视觉（CV）跨界到具身智能（Embodied AI）领域的同学最容易踩坑的地方：

在这篇 RDT 论文中，DiT 模型预测的绝对不是噪声 $\epsilon$，而是直接预测“干净的动作本身” $a_{t:t+T_a}$（即 $a_0$）！

为了让大家在未来的科研中避开这个坑，我们有必要深入探讨一下：为什么在机器人连续控制中，大模型要抛弃经典的“预测噪声”，转而“预测原图（动作）”？

一、 回归公式：证据就在 Loss 函数中

大家回看我们刚刚讲义里的去噪均方误差公式（原论文公式2）：

\[\mathcal{L}(\theta) := \text{MSE}\left(a_t, f_\theta(\ell, o_t, \sqrt{\bar{\alpha}_k}a_t + \sqrt{1 - \bar{\alpha}_k}\epsilon, k)\right)\]

请看 MSE 函数的第一个参数：它是 $a_t$（在这个公式中代表真实的、干净的专家动作，对应我们之前讲的 $a_{t:t+T_a}$）。而第二个参数 $f_\theta(\dots)$ 是我们 DiT 网络的输出。

这意味着，网络 $f_\theta$ 的优化目标是无限逼近那个纯净的物理动作，而不是逼近正态分布里采出来的那个高斯噪声 $\epsilon$。

二、 深度思考：为什么要直接预测动作（Predict $a_0$）？

在经典的图像生成（比如早期的 DDPM 或 Stable Diffusion）中，预测噪声 $\epsilon$ 往往在数学上更稳定，效果更好。但在机器人领域，这套逻辑行不通。原因有三：

  1. 物理边界的绝对约束： 机器人的动作（比如机械臂关节角度、移动速度）具有极其严格的物理边界和安全死区。如果预测的是噪声 $\epsilon$，在扩散的早期阶段（$k$ 很大时，方差极大），从预测的 $\epsilon$ 逆推回来的当前状态 $a_0$ 会产生极其荒谬的极端值，轻易就会突破机器人的物理极限范围 $[-1, 1]$。 而如果是直接预测 $a_0$，网络在每一次去噪步中，都会被迫输出一个“在物理上看起来合理”的绝对动作值。我们甚至可以在代码层面，在每一步推理后对预测出的 $a_0$ 施加截断（Clipping），这在工程上提供了极大的安全性保障。

  2. 多峰分布下的平滑过渡： 正如我们开头讲过的，双臂操作具有强烈的“多峰性（Multi-modality）”。如果我们预测噪声，模型在权衡两种不同的抓取策略时，稍微的误差波动就会导致噪声分布崩溃。而预测动作本身，即使模型在早期的 $k$ 步产生了一定程度的动作模糊或平均，在后续的去噪步中也能更平稳地收敛到一个确定的、物理可行的动作模式上（这种范式在学术界最初是由哥伦比亚大学的 Diffusion Policy 论文发扬光大的，RDT 继承了这一优良传统）。

  3. 对齐动作分块（Action Chunking）： RDT 预测的是未来 64 步的动作序列。如果预测噪声，时序上的高频抖动会被放大；直接预测连续的动作轨道，结合最后那一层 MLP 解码器，模型能更有效地保证动作在时间轴上的平滑与连贯。

三、 重新校准：你该如何向同行精准描述 RDT 的结构？

所以，如果今后在学术会议上有人问你 RDT-1B 到底是个什么网络，你可以用这段话给出极其专业的回答：

“RDT-1B 在宏观上沿用了 DDPM 的迭代去噪逻辑。在输入端，它接收一段带噪声的轨迹 $\tilde{a}_{t:t+T_a}$，同时利用预训练的 T5-XXL 和 SigLIP 提取出语言和视觉的 Token 作为条件。

它的核心替换发生在主干网络：它彻底抛弃了 CV 中常用的 UNet，使用了具有交替条件注入（ACI）和 RMSNorm 设计的 DiT（Diffusion Transformer）作为 Backbone。

最关键的是，DiT 的最后经过一个多层感知机（MLP Decoder），直接回归预测出未来 64 步的干净动作绝对值 $a_{t:t+T_a}$，并通过对干净动作算 MSE Loss 来更新梯度。”

大家在读顶会论文时，一定要像这样敏锐地察觉到：看似通用的框架（如 Diffusion），在跨界到一个具有强物理属性的新领域（如 Robotics）时，其内部的输入输出目标、归一化方式、甚至损失函数的参数位，必定会发生适应物理直觉的“重构”。这正是科研创新的落脚点。
