# RoboScape-R——基于世界模型的具身智能泛化训练框架


---

## 第一讲：引言与问题背景

### 1.1 具身智能的核心挑战

具身智能（Embodied Intelligence）作为实现通用人工智能（AGI）的关键路径之一，其核心在于使智能体能够在物理世界中执行复杂的操作任务。然而，当前具身智能体面临的最根本挑战是**泛化能力不足**。

**定义：泛化能力**
- **领域内泛化**：在同一环境但不同初始状态下的表现
- **领域外泛化**：面对未见过的物体、场景或任务组合时的表现
- **任务级泛化**：跨不同任务类型的迁移能力

### 1.2 现有训练范式的局限性

#### 1.2.1 模仿学习（Imitation Learning, IL）

**基本原理**：
- 通过监督学习优化策略
- 依赖专家轨迹数据
- 目标函数：$\min_{\theta} \mathbb{E}_{(s,a)\sim \mathcal{D}_{\text{expert}}} [\mathcal{L}(\pi_\theta(s), a)]$

**核心局限性**：
1. **分布偏移问题**：策略在训练分布之外的表现不可控
2. **过拟合**：过度拟合专家轨迹的特定模式
3. **数据效率低**：需要大量高质量专家数据
4. **探索不足**：缺乏自主探索机制

#### 1.2.2 强化学习（Reinforcement Learning, RL）

**基本原理**：
- 通过与环境交互优化策略
- 依赖奖励信号引导学习
- 目标函数：$\max_{\theta} \mathbb{E}_{\tau \sim \pi_\theta} [\sum_{t=0}^{T} \gamma^t r(s_t, a_t)]$

**核心局限性**：
1. **奖励函数设计困难**：跨任务场景难以设计统一的奖励函数
2. **泛化瓶颈**：在任务族间的泛化能力有限
3. **样本效率**：需要大量环境交互

### 1.3 世界模型：新的解决方案

**世界模型的定义**：能够基于当前状态和动作预测下一观测的生成模型

**形式化表示**：
$$\hat{x}_{t+1}, \hat{d}_{t+1} = WM(x_{0:t}, c)$$

其中：
- $x_t$：t时刻的观测（RGB图像）
- $\hat{d}_t$：预测的任务完成信号
- $c$：控制信号（动作或指令）

**世界模型的核心优势**：
1. **无监督学习**：可以从大量无标签数据中学习环境动态
2. **环境代理**：可作为真实环境的替代进行策略训练
3. **可扩展性**：通过大规模预训练获得泛化能力

### 1.4 RoboScape-R的动机与贡献

**研究问题**：如何将世界模型作为可交互的RL环境，同时解决奖励信号缺失和控制鲁棒性问题？

**核心贡献**：
1. **内生通用奖励机制**：首次提出从世界模型自身导出的通用奖励信号
2. **双世界模型架构**：动作世界模型与文本世界模型的协同设计
3. **实证验证**：在OOD场景下实现37.5%的性能提升

## 第三讲：RoboScape-R方法论

### 3.1 整体架构

#### 3.1.1 双世界模型设计

**架构图解析**：
```
┌─────────────────────────────────────────────────────┐
│                  RoboScape-R Pipeline                │
├─────────────────────────────────────────────────────┤
│  ┌──────────────┐      ┌─────────────────────────┐  │
│  │    Policy    │─────▶│  World Model Environment│  │
│  │   π_θ(·|s)   │◀────│  with General Reward    │  │
│  └──────────────┘      └─────────────────────────┘  │
│         │                           │                 │
│         │  action                   │ observation     │
│         ▼                           ▼ reward         │
│  ┌─────────────────────────────────────────────┐    │
│  │         Dual World Model Structure          │    │
│  │  ┌─────────────────┐  ┌─────────────────┐   │    │
│  │  │  Action WM      │  │  Text WM        │   │    │
│  │  │  WM_act         │  │  WM_txt          │   │    │
│  │  │  - action input │  │  - instruction  │   │    │
│  │  │  - next frame   │  │  - goal frame   │   │    │
│  │  └─────────────────┘  └─────────────────┘   │    │
│  └─────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────┘
```

**两个世界模型的职责分离**：
1. **动作世界模型（WM_act）**：负责与策略交互，生成下一帧观测
2. **文本世界模型（WM_txt）**：负责生成目标观测，提供奖励先验

#### 3.1.2 信息流

**前向传播**：
$$\hat{x}_t, \hat{d}_t = WM(x_{0:t-1}, c)$$

其中：
- 对于动作世界模型：$c = a_{0:t} \in \mathbb{R}^{d \times t}$（d维机器人动作）
- 对于文本世界模型：$c = i$（文本指令）

### 3.2 世界模型的详细架构

#### 3.2.1 视觉Tokenizer：MAGVIT-2

**编码过程**：
$$g_{0:t}, s_{0:t} = \text{Tokenizer}(x_{0:t})$$

- 输入：$x_{0:t} \in \mathbb{R}^{T \times H \times W \times 3}$
- 输出：离散潜变量 $g_{0:t}, s_{0:t} \in \mathbb{R}^{T \times H' \times W' \times D}$
- 下采样因子：$\alpha$，满足 $H' = H/\alpha$, $W' = W/\alpha$

**技术特点**：
- 基于向量量化（VQ）的离散表示
- 时空联合压缩
- 重建保真度高

#### 3.2.2 控制信号编码

**文本编码器（T5）**：
$$c_i = \text{T5}(i) \in \mathbb{R}^{d_{\text{text}}}$$

- 使用预训练的T5编码器
- 捕获指令的语义信息

**动作编码器（MLP）**：
$$c_a = \text{MLP}(a) \in \mathbb{R}^{d_{\text{action}}}$$

- 处理7-DoF机器人动作
- 维度：7（位置+旋转）

#### 3.2.3 条件化的时空模块

**STBlock-T（文本条件）**：
- 输入：视觉token序列 + 文本嵌入
- 机制：交叉注意力注入文本信息

**STBlock-A（动作条件）**：
- 输入：视觉token序列 + 动作嵌入
- 机制：交叉注意力注入动作信息

**模块结构**：
```
输入token序列
    │
    ▼
┌─────────────────────┐
│  Spatial Attention  │←─── 控制信号嵌入
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  Temporal Attention │
└─────────────────────┘
    │
    ▼
 输出预测token
```

#### 3.2.4 完成信号预测

**完成信号头**：
$$\hat{d}_t = \text{MLP}_{\text{done}}(x_t)$$

- 从当前帧预测任务完成概率
- 用于稀疏奖励和轨迹终止判断

### 3.3 训练损失函数

#### 3.3.1 视觉损失

**交叉熵损失**：
$$\mathcal{L}_{\text{vis}} = -\sum_{t=0}^{T} z_t \log p(\hat{z}_t)$$

其中：
- $z_t$：对于动作世界模型为$s_t$，对于文本世界模型为$g_t$
- $\hat{z}_t$：预测的token分布

#### 3.3.2 完成信号损失

**RMSE损失**：
$$\mathcal{L}_d = \text{RMSE}(\hat{d}_t, d_t)$$

- 监督完成信号的预测
- $d_t$为真实完成标签

#### 3.3.3 总损失

$$\mathcal{L} = \mathcal{L}_{\text{vis}} + \mathcal{L}_d$$

### 3.4 基于世界模型的通用奖励机制

#### 3.4.1 目标观测生成

**生成过程**：
$$x_{\text{goal}} = \text{Dec}(g_{t^*}), \quad t^* = \min\{t+1 | \hat{d}_t^{\text{txt}} \geq \theta\}$$

- $\theta$：完成信号阈值
- 文本世界模型自回归生成直到完成信号超过阈值
- 取第一帧超过阈值的帧作为目标观测

#### 3.4.2 稠密奖励

**LPIPS相似度**：
$$\mathcal{R}_{\text{den}} = \text{LPIPS}(x_t, x_{\text{goal}})$$

- 基于感知相似度
- 提供细粒度的引导信号
- 加速策略收敛

**LPIPS的特点**：
- 基于深度特征的感知度量
- 比L2更符合人类视觉感知
- 对像素级噪声鲁棒

#### 3.4.3 稀疏奖励

$$\mathcal{R}_{\text{sps}} = \begin{cases}
1 & \text{if } \hat{d}_t^{\text{act}} \geq \theta_{\text{sps}} \\
0 & \text{otherwise}
\end{cases}$$

- 基于动作世界模型的完成信号
- 避免陷入局部最优
- 增强泛化能力

#### 3.4.4 总奖励

$$\mathcal{R} = \mathcal{R}_{\text{sps}} + \mathcal{R}_{\text{den}}$$

**奖励设计的关键创新**：
- **内生性**：来自世界模型自身，而非手工设计
- **无监督**：无需人工标注奖励标签
- **通用性**：可跨不同任务迁移

### 3.5 基于世界模型的RL训练流程

#### 3.5.1 算法伪代码解析

**算法1：RoboScape-R训练框架**

```
1: 通用环境初始化
2:  E ← {e_0}
3:  for each environment n ∈ {1,...,N} do
4:      E ← E ∪ Init(e_n)  # 初始化额外环境
5:  end for
6:  for each environment n ∈ {0,...,N} do
7:      x_n ← x_n^0, t ← 0, d ← 0
8:      while d_t^txt < θ do
9:          (x_t, d) ← WM_txt(x_{0:t-1}, i)  # 递归生成直到完成
10:         t ← t + 1
11:     end while
12:     x_{goal,n} ← x_t  # 环境n的目标帧
13: end for
14: for each environment n ∈ {0,...,N} do
15:     x_n ← x_n^0  # 重置到初始观测
16: end for
17: for k = 1 to K do  # 总训练迭代
18:     D ← ∅  # 初始化轨迹缓冲区
19:     # 环境中的策略 rollout
20:     for t = 1 to T do
21:         for each environment n ∈ {0,...,N} do
22:             a_i ← π_θ(·|s_i)  # 策略输出动作
23:             (x_{n,t}, R_sps) ← WM_act(x_{n,0:t-1}, a_{0:t-1})
24:             R_den ← sim(x_{n,t}, x_{goal,n})
25:             R_i ← α·R_sps + β·R_den  # 总奖励
26:             D ← D ∪ {(x_n, a_n, R_n)}
27:             x_n ← x_t  # 更新当前状态
28:         end for
29:     end for
30:     # 策略优化
31:     Θ ← Θ + λ·∇_Θ J(Θ; D)
32: end for
33: return π_Θ
```

#### 3.5.2 关键设计解析

**通用环境初始化**：
- 支持多个环境并行交互
- 每个环境都有独立的目标观测
- 利用文本世界模型生成任务特定的目标

**策略Rollout**：
- 动作世界模型响应策略动作
- 实时生成下一帧观测
- 结合稠密和稀疏奖励

**策略优化**：
$$\Theta \leftarrow \Theta + \gamma \cdot \nabla_{\Theta} J(\Theta; \mathcal{D}(X, R))$$

- $\Theta$：策略参数
- $\gamma$：学习率
- $J$：目标函数（如PPO的clip目标）
- $\mathcal{D}$：收集的轨迹数据


## 目标观测生成逻辑

# 目标观测生成机制详解

这是RoboScape-R中最精妙的设计之一。目标观测是通过**文本世界模型**自动生成的，整个过程完全自动化，无需人工标注或手工指定目标状态。

---

## 一、目标观测生成的直观理解

### 1.1 基本思想

**核心思想**：给定一个任务的文本指令（如"pick up the green bell pepper and place it on the plate"）和初始观测，让文本世界模型"想象"出任务完成时的画面。

这类似于：
- 人类听到指令后，会在脑海中想象"成功时应该是什么样子"
- 这个想象画面成为后续执行的参照标准

### 1.2 输入输出

```
输入：
    - 初始观测 x₀ (任务开始时的画面)
    - 文本指令 i ("pick up the green bell pepper and place it on the plate")
    
输出：
    - 目标观测 x_goal (任务完成时的预测画面)
```

---

## 二、生成过程的数学形式

### 2.1 核心公式

论文中的公式(6)给出了目标观测的生成方式：

$$x_{\text{goal}} = \text{Dec}(g_{t^*}), \quad t^* = \min\{t+1 | \hat{d}_t^{\text{txt}} \geq \theta\}$$

其中：
- $\text{Dec}$：视觉解码器（将token解码为图像）
- $g_{t^*}$：第$t^*$时刻的视觉token
- $\hat{d}_t^{\text{txt}}$：文本世界模型预测的t时刻完成信号
- $\theta$：完成信号阈值（论文中设为0.8）

### 2.2 直观解释

这个公式表达了一个**自回归生成直到任务完成**的过程：

1. 从初始观测$x_0$开始
2. 让文本世界模型一步步预测后续帧
3. 在每一帧，模型同时预测完成信号$\hat{d}_t^{\text{txt}}$
4. 当$\hat{d}_t^{\text{txt}}$首次超过阈值$\theta$时，认为任务已完成
5. 取这一帧作为目标观测$x_{\text{goal}}$

---

## 三、详细生成流程

### 3.1 步骤分解

#### 步骤1：初始化

```
输入：
    x₀: 初始观测（RGB图像）
    i: 文本指令
    
状态：
    t = 0
    history = [x₀]
    done = False
```

#### 步骤2：自回归生成循环

```
while not done:
    # 文本世界模型前向传播
    (ˆx_{t+1}, ˆd_{t+1}) = WM_txt(history, i)
    
    # 检查完成信号
    if ˆd_{t+1} ≥ θ:
        # 任务已完成
        x_goal = ˆx_{t+1}
        done = True
    else:
        # 继续生成
        history.append(ˆx_{t+1})
        t = t + 1
```

#### 步骤3：输出目标观测

```
return x_goal  # 任务完成时的预测画面
```

### 3.2 可视化示例

以"pick up the green bell pepper and place it on the plate"为例：

```
时间步 t=0: [初始画面] 辣椒在桌上，盘子空着，夹爪在起始位置
                ↓ (WM_txt预测)
时间步 t=1: [接近阶段] 夹爪移向辣椒
                ↓ (WM_txt预测)
时间步 t=2: [抓取阶段] 夹爪抓住辣椒
                ↓ (WM_txt预测)
时间步 t=3: [抬起阶段] 辣椒被抬起
                ↓ (WM_txt预测)
时间步 t=4: [移动阶段] 辣椒移向盘子
                ↓ (WM_txt预测)
时间步 t=5: [放置阶段] 辣椒放入盘子
                ↓ (WM_txt预测)
时间步 t=6: [完成] 辣椒在盘子中，夹爪离开
                ˆd₆ = 0.85 ≥ θ → 停止
                ↓
目标观测 x_goal = 时间步6的画面
```

## 四、文本世界模型的前向过程

### 4.1 单步预测的详细流程

在每一步，文本世界模型的完整前向过程：

```
输入: history = [x₀, x₁, ..., x_t], 指令 i

1. 视觉tokenization（对历史所有帧）
   g₀, g₁, ..., g_t = MAGVIT-2(history)
   # g ∈ R^{H'×W'×D}，是离散token

2. 指令编码
   c_i = T5(i)  # 文本嵌入

3. 条件化时空建模
   for each STBlock-T:
       # 空间注意力（融合当前帧的空间信息）
       g' = SpatialAttention(g)
       
       # 交叉注意力（注入指令信息）
       g'' = CrossAttention(g', c_i)
       
       # 时序注意力（融合历史信息）
       g''' = TemporalAttention(g'', context)
   
   输出: 预测的下一帧token ˆg_{t+1}

4. 完成信号预测
   ˆd_{t+1} = MLP_done(当前特征)

5. 视觉解码
   ˆx_{t+1} = Decoder(ˆg_{t+1})

输出: 预测的下一帧 ˆx_{t+1}，预测的完成信号 ˆd_{t+1}
```

### 4.2 完成信号的意义

完成信号$\hat{d}_t$是文本世界模型预测的**任务完成概率**，范围[0,1]。

**训练时**：
- 从真实轨迹中，任务完成帧的$d_t=1$，未完成帧$d_t=0$
- 通过RMSE损失监督模型学习何时任务完成

**推理时**：
- 模型基于对任务的理解，预测每一步的完成概率
- 当概率超过阈值$\theta$，认为任务已达成

## 五、目标观测的数学性质

### 5.1 作为最优轨迹的终点

文本世界模型生成的轨迹实际上是**"黄金轨迹"**（golden trajectory）的预测：

$$\tau_{\text{gold}} = \{x_0, x_1, ..., x_{t^*}\}$$

其中：
- $x_0$：初始观测
- $x_{t^*}$：目标观测
- 中间帧：完成任务所需的中间状态

### 5.2 为什么取第一帧超过阈值的？

取$\min\{t+1 | \hat{d}_t^{\text{txt}} \geq \theta\}$而非最后一帧，是因为：

- **效率**：任务完成后继续生成可能引入多余动作
- **准确性**：完成时刻的画面最能代表"成功状态"
- **简洁性**：避免因生成过长轨迹导致的累积误差

### 5.3 目标观测的唯一性

对于同一任务的不同初始状态，目标观测可能不同：

```
任务: "pick up the pepper and place it on the plate"

初始状态A: 辣椒在左边，盘子右边
    → 目标观测A: 辣椒在盘子里，但夹爪可能在不同位置

初始状态B: 辣椒在右边，盘子左边
    → 目标观测B: 同样是辣椒在盘子里，但夹爪轨迹不同
```

目标观测是**任务完成的视觉表征**，而非固定的图像。

---

## 六、为什么需要生成整个轨迹而非直接预测目标？

### 6.1 自回归的必要性

**问题**：为什么不直接训练一个模型从$x_0$和$i$直接预测$x_{\text{goal}}$？

**原因**：

1. **物理合理性**：任务完成的过程必须符合物理规律。通过逐步生成，模型确保中间状态是物理可行的。

2. **因果性**：最终状态是中间状态的因果结果。自回归生成保持了这种因果链。

3. **完成信号**：需要在过程中判断何时任务完成，这无法从端到端的预测中获得。

4. **训练数据一致性**：模型训练时就是逐帧预测，推理时也应保持一致。

### 6.2 与动作世界模型的关系

| 世界模型 | 生成轨迹的用途 | 条件 | 是否用于目标生成 |
|---------|---------------|------|------------------|
| 文本世界模型 | 生成黄金轨迹 | 指令 | **是**（提供x_goal）|
| 动作世界模型 | 响应策略动作 | 动作 | 否 |

两者**共享相同的物理知识**，但**条件不同**：
- 文本世界模型：知道"要做什么"，生成理想轨迹
- 动作世界模型：知道"如果这样做会怎样"，响应实际动作

---

## 七、目标观测的存储与复用

### 7.1 在RL训练中的使用

在环境初始化阶段（Algorithm 1第6-13行）：

```
for each environment n:
    # 生成该环境的目标观测
    x_n = x_n^0, t = 0, d = 0
    while d_t^txt < θ:
        (x_t, d) = WM_txt(x_{0:t-1}, i)
        t = t + 1
    x_{goal,n} = x_t  # 保存目标观测
    
    # 重置环境到初始状态
    x_n = x_n^0
```

在策略Rollout阶段（Algorithm 1第20-29行）：

```
for each time step t:
    for each environment n:
        # 策略执行
        a = π_θ(·|s_n)
        
        # 动作世界模型响应
        (x_{n,t}, R_sps) = WM_act(x_{n,0:t-1}, a)
        
        # 使用预先计算的目标观测计算稠密奖励
        R_den = LPIPS(x_{n,t}, x_{goal,n})
        
        # 总奖励
        R = α·R_sps + β·R_den
```

**关键点**：
- 目标观测在环境初始化时**一次性生成**并存储
- 整个RL训练过程中**重复使用**同一目标观测
- 稠密奖励计算只是简单的LPIPS相似度，**无需再调用文本世界模型**

### 7.2 存储格式

每个环境存储：
- 当前观测 $x_t$（动态更新）
- 目标观测 $x_{\text{goal}}$（固定）
- 文本指令 $i$（固定）
- 历史轨迹（可选）

## 八、目标观测生成的质量保证

### 8.1 训练数据的覆盖

文本世界模型在大量轨迹上训练：
- 80个场景
- 4种任务
- 每种任务包含最优和次优轨迹
- 总计超过30万条轨迹

这确保了模型学会了：
- 不同物体的抓取方式
- 不同容器的放置位置
- 任务完成的视觉特征

### 8.2 完成信号阈值的设定

阈值$\theta$的选择影响目标观测的质量：

| θ值 | 效果 | 风险 |
|-----|------|------|
| 过高 (0.95) | 目标观测更准确 | 可能永远达不到，生成过长轨迹 |
| 适中 (0.8) | 平衡准确性和效率 | 论文中使用的值 |
| 过低 (0.5) | 容易达到 | 可能任务未真正完成就停止 |

### 8.3 多模态目标

对于某些任务，可能存在多个"正确"的目标状态。文本世界模型会生成**最典型的完成状态**，基于训练数据中学到的分布。

## 九、总结

### 9.1 目标观测生成的关键要点

1. **自动生成**：无需人工标注，仅需初始观测和指令
2. **自回归过程**：逐帧预测直到任务完成
3. **完成信号驱动**：由模型预测的完成概率决定停止点
4. **一次性计算**：在环境初始化时生成，RL训练中复用
5. **任务特定**：不同任务、不同初始状态生成不同的目标观测

### 9.2 与Dreamer范式的对比

| 方面 | Dreamer | RoboScape-R |
|------|---------|-------------|
| **目标定义** | 奖励函数（数值） | 目标观测（图像） |
| **目标来源** | 手工设计 | 模型自动生成 |
| **泛化机制** | 奖励函数泛化 | 视觉特征泛化 |
| **监督信号** | 奖励值 | LPIPS相似度 |

### 9.3 直观理解

想象你教一个机器人做菜：
- **传统方法**：你给它一个"分数函数"（比如"辣椒离盘子越近分数越高"）
- **Dreamer**：它学习预测"如果做这个动作，会得多少分"
- **RoboScape-R**：你给它看一张"菜做好了"的照片，它自己判断"我现在的画面和这张照片像不像"

**目标观测就是那张"菜做好了"的参考照片**，而文本世界模型就是那个能根据菜谱想象出成品照片的大厨。


## 和Dreamer系列工作的对比

# RoboScape-R vs Dreamer：深度对比分析

你的直觉非常准确！RoboScape-R和Dreamer确实共享**“学习世界模型→基于世界模型进行规划/强化学习”**的核心思想。但两者在**技术实现、设计目标和应用场景**上存在显著差异。让我为你详细剖析。

---

## 一、核心思想对比

### 1.1 共同点

两者都遵循**“学习环境动态模型，然后在学习到的模型中进行策略优化”**的范式：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  收集数据   │───▶│  训练世界   │───▶│  在模型中   │
│  (真实环境) │    │    模型     │    │  优化策略   │
└─────────────┘    └─────────────┘    └─────────────┘
```

### 1.2 本质区别

| 维度 | Dreamer | RoboScape-R |
|------|---------|-------------|
| **世界模型形式** | 潜在空间动态模型 | 像素空间生成模型 |
| **奖励来源** | 真实环境奖励（需手工设计） | 内生奖励（从模型自身导出） |
| **训练数据** | 在线交互数据 | 离线预训练数据集 |
| **多任务支持** | 单任务优化 | 多任务通用策略 |
| **泛化能力** | 依赖奖励设计 | 通过内生奖励实现泛化 |

## 二、Dreamer框架详解

### 2.1 Dreamer的核心架构

Dreamer（Dream to Control）是DeepMind提出的基于世界模型的强化学习框架，主要版本包括DreamerV1、V2、V3。

**基本流程**：

```
┌─────────────────────────────────────────────────────┐
│                     Dreamer                          │
├─────────────────────────────────────────────────────┤
│  循环迭代：                                           │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐        │
│  │ 收集数据 │───▶│ 训练模型 │───▶│ 行为学习 │───┐    │
│  │ (环境)   │    │ (世界模型)│    │ (策略)   │   │    │
│  └─────────┘    └─────────┘    └─────────┘   │    │
│      ▲                                          │    │
│      └──────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────┘
```

#### 2.2 Dreamer的世界模型

Dreamer使用**潜在动态模型（Latent Dynamics Model）**：

```python
# Dreamer的世界模型组件
1. 表示模型：z_t ~ q(z_t | z_{t-1}, a_{t-1}, x_t)  # 编码观测到潜在状态
2. 转移模型：ˆz_t ~ p(ˆz_t | z_{t-1}, a_{t-1})      # 预测下一潜在状态
3. 奖励模型：ˆr_t ~ p(ˆr_t | z_t)                   # 预测奖励
4. 观测模型：ˆx_t ~ p(ˆx_t | z_t)                   # 重建观测（可选）
```

**关键特点**：
- **潜在空间建模**：不在像素空间预测，而在压缩的潜在空间
- **联合训练**：所有组件一起训练，优化ELBO
- **奖励预测**：显式学习奖励函数（需要真实奖励作为监督）

#### 2.3 Dreamer的策略学习

**在想象中学习（Learning in Imagination）**：

```
在训练好的世界模型中展开想象轨迹：
1. 从当前潜在状态z_t开始
2. 用策略网络π(·|z_t)采样动作a_t
3. 用转移模型预测下一潜在状态ˆz_{t+1}
4. 用奖励模型预测奖励ˆr_t
5. 重复直到想象轨迹结束

然后用这些想象轨迹更新策略和价值网络
```

## 三、RoboScape-R与Dreamer的详细对比

### 3.1 世界模型的形式差异

#### Dreamer的世界模型

```
输入：x_t (观测), a_{t-1} (上一动作)
      ↓
编码器：z_t = Encoder(x_t)
      ↓
RNN/GRU：h_t = f(h_{t-1}, z_t, a_{t-1})
      ↓
输出：z_{t+1}的分布, ˆr_t (奖励预测)
```

**特点**：
- 在**低维潜在空间**中建模
- 需要显式学习奖励模型
- 观测重建是可选的

#### RoboScape-R的世界模型

```
输入：x_{0:t-1} (历史观测), c (控制信号)
      ↓
视觉Tokenizer：g_{0:t-1} = Tokenizer(x_{0:t-1})
      ↓
条件化时空模块：融合控制信号c
      ↓
自回归预测：ˆg_t = Transformer(g_{0:t-1}, c)
      ↓
视觉Decoder：ˆx_t = Decoder(ˆg_t)
      ↓
输出：ˆx_t (像素空间预测), ˆd_t (完成信号)
```

**特点**：
- 在**像素空间**中直接生成
- **不学习奖励模型**，而是内生奖励
- 必须生成高质量视觉观测

### 3.2 奖励机制的差异

这是**最核心的区别**：

| 方面 | Dreamer | RoboScape-R |
|------|---------|-------------|
| **奖励来源** | 外生（需手工设计） | 内生（从模型导出） |
| **奖励形式** | 标量值 | LPIPS相似度 + 完成信号 |
| **训练监督** | 需要真实奖励标签 | 无需奖励标签 |
| **泛化能力** | 依赖奖励函数设计 | 通过视觉相似性泛化 |

**形式化对比**：

Dreamer的奖励预测：
$$\hat{r}_t = f_{\text{reward}}(z_t)$$
需要真实奖励$r_t$作为监督：
$$\mathcal{L}_{\text{reward}} = \text{MSE}(\hat{r}_t, r_t)$$

RoboScape-R的奖励计算：
$$\mathcal{R}_{\text{den}} = \text{LPIPS}(x_t, x_{\text{goal}})$$
$$\mathcal{R}_{\text{sps}} = \mathbb{1}[\hat{d}_t \geq \theta]$$
**无需学习**，直接从模型输出计算

### 3.3 训练流程的差异

#### Dreamer的训练循环

```
for episode in 1...M:
    # 1. 在真实环境中收集数据
    while not done:
        a_t = π(z_t)  # 策略推理
        x_{t+1}, r_t, done = env.step(a_t)
        store (x_t, a_t, r_t, done)
    
    # 2. 训练世界模型
    for step in 1...K:
        sample batch from buffer
        update world model (representation, transition, reward)
    
    # 3. 在想象中学习策略
    for step in 1...L:
        sample start state from buffer
        imagine trajectory in latent space
        update actor and critic
```

**关键点**：
- **在线交互**：不断在真实环境中收集新数据
- **循环迭代**：数据收集、模型训练、策略学习交替进行
- **单任务**：针对当前任务优化

#### RoboScape-R的训练流程

```
# 第一阶段：离线预训练世界模型
collect large dataset from simulator (300k+ trajectories)
train WM_act and WM_txt separately (offline, 5 epochs)

# 第二阶段：基于世界模型的RL训练
initialize N parallel environments
for each environment:
    x_goal = WM_txt(x_0, i)  # 生成目标观测

for iteration in 1...K:
    # 在世界模型中收集数据
    for t in 1...T:
        for n in 1...N:
            a = π_θ(x_n)
            x_n', d = WM_act(x_n, a)
            R = LPIPS(x_n', x_goal) + 1[d≥θ]
            store (x_n, a, R)
    
    # 更新策略
    update π_θ with PPO using collected data
```

**关键点**：
- **离线预训练**：世界模型在大规模数据上预训练
- **无需真实环境**：RL阶段完全在世界模型中进行
- **多任务并行**：同时与多个环境交互
- **内生奖励**：无需奖励模型

### 3.4 目标函数的差异

#### Dreamer的目标函数

世界模型训练（ELBO）：
$$\mathcal{L} = \mathbb{E}[\sum_t (\underbrace{-\log p(x_t|z_t)}_{\text{重建}} + \underbrace{\beta \cdot \text{KL}[q(z_t|...)||p(z_t|...)]}_{\text{正则化}} + \underbrace{-\log p(r_t|z_t)}_{\text{奖励预测}})]$$

策略学习（想象中）：
$$J(\pi) = \mathbb{E}_{\tau \sim p_\phi}[\sum_t \gamma^t \hat{r}_t]$$

#### RoboScape-R的目标函数

世界模型训练：
$$\mathcal{L} = \underbrace{-\sum_t z_t\log p(\hat{z}_t)}_{\text{视觉预测}} + \underbrace{\text{RMSE}(\hat{d}_t, d_t)}_{\text{完成信号}}$$
**无奖励预测项**

策略学习（真实轨迹）：
$$J(\pi) = \mathbb{E}_{\tau \sim \text{WM}}[\sum_t \gamma^t (\underbrace{\text{LPIPS}(x_t, x_{\text{goal}})}_{\text{内生奖励}} + \underbrace{\mathbb{1}[d_t \geq \theta]}_{\text{稀疏奖励}})]$$

|------|---------|-------------|---------------------|
| **世界模型空间** | 潜在空间 | 像素空间 | 生成可视觉检查的轨迹 |

## 五、为什么RoboScape-R不采用Dreamer的方式？

### 5.1 Dreamer的局限性

1. **奖励依赖**：Dreamer需要真实的奖励信号作为监督。在多任务场景下，需要为每个任务设计不同的奖励函数，且这些奖励函数必须**对齐**（aligned）才能训练通用策略。

2. **泛化瓶颈**：Dreamer的策略泛化能力高度依赖奖励函数的设计。如果奖励函数无法泛化到新场景，策略也无法泛化。

3. **单任务优化**：Dreamer的设计本质上是单任务的。虽然可以在多任务数据上训练世界模型，但策略学习仍需要统一的奖励信号。

### 5.2 RoboScape-R的解决方案

RoboScape-R通过**内生奖励**解决了这些问题：

1. **奖励统一**：LPIPS相似度对所有任务都是相同的计算方式，不需要任务特定的设计
2. **无需对齐**：不同任务的目标观测自动提供任务特定的奖励形状
3. **语义泛化**：LPIPS基于深度特征，对像素级变化鲁棒，自然泛化到新物体

### 5.3 实验结果验证

论文表2的数据直接证明了这一点：

| 环境 | P&P (pepper) | P&P (peach) | M.A. (pepper) | M.A. (peach) |
|------|--------------|-------------|---------------|--------------|
| RL w. ManiSkill | 95.0 (↑.27) | 60.5 (↑.38) | 64.0 (↓.16) | 62.5 (↓.21) |
| RL w. World Model | 93.1 (↑.25) | 64.6 (↑.47) | 85.9 (↑.11) | 84.5 (↑.06) |

**解读**：
- 使用物理模拟器（类似Dreamer范式）训练时，P&P任务性能提升，但M.A.任务反而下降 → **奖励不对齐**
- 使用RoboScape-R训练时，所有任务均衡提升 → **内生奖励自动对齐**

## 六、总结

| 类比 | Dreamer | RoboScape-R |
|------|---------|-------------|
| **世界模型** | 类似人的**抽象思维**（在脑海想象） | 类似人的**视觉想象**（在脑海"看到"） |
| **奖励** | 外部给的**分数** | 自己对"成功画面"的**感知相似度** |
| **学习方式** | 边做边学 | 先大量观察，再在想象中练习 |
| **泛化能力** | 需要重新设计分数 | 自动适应新场景 |

**一句话总结**：
- **Dreamer**：学习"如何在给定奖励下做得好"
- **RoboScape-R**：学习"成功应该是什么样子"，然后自己判断做得好不好

这正是RoboScape-R在多任务泛化上超越Dreamer范式的根本原因。
